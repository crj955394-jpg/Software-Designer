脚本语言属于动态语言，其程序结构可以在运行时改变
    运行时改变：指的是在代码编辑的时候不是强类型约束 这种的
    编译型语言的运行效率一般高于解释型，因为编译型时编译为目标程序后执行，不需要每行解释
程序语言分类
    按照语言级别：低级语言和高级语言，低级语言面向机器：汇编，高级语言面向开发者：c c++ java python
    按照翻译方式：编译型和解释型，编译型是将源程序全部翻译成目标程序后执行，解释型是逐句翻译执行，不产生目标程序
    按照编程思维：命令式语言，面向对象语言，函数式语言，逻辑型语言（Prolog，只给出目标是什么，没有步骤，全靠推理得出）
程序设计语言的四个组成 
    数据，运算，控制，传输
    数据类型：基本数据类型，用户自定义类型，构造类型（基本数据类型和自定义组合的，数组这种），指针类型，空类型
        变量具有五个属性：名，地址，值，作用域，生存期
    运算成分:
        算术运算：加减乘除 这种数据运算
        关系运算:比较运算，大于等于，小于等于 这种的 ，结果是布尔值
        逻辑运算：逻辑与，或，非 && || ！
        位运算：& | >>  << 对二进制进行操作
        赋值运算：= += -= 赋值操作
        优先级是：括号的优先级最高，多层括号，先算最里面的
                单目运算> 算数运算> 移位运算> 关系运算> 位逻辑运算> 逻辑运算> 赋值运算
                单目运算：自增，自减 这种 ++，-- ，！，自增分为 ++a 和a++，区别是++a 是先算a自增，之后拿a的结果，a++就不一样了，是先拿a的结果，再将a自增
                移位运算高于关系运算，高于位逻辑运算 比如：a & b > c ，先算 b > c，再算 a & 结果
                逻辑运算短路特性：就是 && 的意思是同真位真，一假则假，当左侧运算结果是假的，那么就没必要算右侧的了,int a=0,b=1; if(a++ && ++b)，执行后a=1，b仍为1——&&左侧a++先取值为 0（假），右侧++b被短路，不执行
                赋值运算是从右向左，a = b = c = 5，c=5，再算b=5，最后算a=5
                == 和= 的区别，== 是判断，=是赋值
            若有定义int a=2,b=3,c=4;，则表达式a+b>c && b==c的值是？ 答案：0
    控制：顺序，分支（if，switch），循环（for）
    传输：赋值运算 = ，IO输入输出，printf() 和 scanf(),函数调用中的参数

汇编
    汇编三类语句：指令语句（产生机器代码），伪指令语句（不产生机器代码，是符号赋值，分配存储单元这种），宏指令语句（替换后产生机器代码）
    汇编程序的基本工作：将每一条可执行的汇编语句转成响应的机器指令，处理源程序中的伪指令
        汇编程序的扫描次数：一般需要两次扫描源程序才能完成翻译工作
            第一遍扫描：确定符号的地址，建立符号表
            第二遍扫描：将符号替换为实际地址，生成机器代码
    
编译程序的原理 ⭐
    功能：将高级语言编写的 源程序 翻译为 目标程序
    编译的六个阶段：源程序 ->词法分析 ->语法分析 ->语义分析 ->中间代码生成 -> 代码优化->目标代码生成 ->目标程序
    各阶段详解 
        | 阶段               | 输入                 | 输出                     | 主要任务                                                           |
        |--------------------|----------------------|--------------------------|------------------------------------------------------------------|
        | **①词法分析**      | 源程序字符串         | 单词符号序列             | 从左到右逐个字符扫描，识别出关键字、标识符、常数、运算符、界符等单词符号  |
        | **②语法分析**      | 单词符号序列         | 语法树                   | 根据语法规则将单词序列分解成各类语法单位，检查程序语法结构是否合法       |
        | **③语义分析**      | 语法树               | 带语义信息的语法树       | 检查程序是否存在语义错误（如类型不匹配），收集类型信息用于后续代码生成    |
        | **④中间代码生成**  | 语义分析结果         | 中间代码                 | 生成与具体机器无关的中间表示（后缀式、三元式、四元式等）                |
        | **⑤代码优化**      | 中间代码             | 优化后的中间代码         | 对中间代码进行等价变换，提升目标代码的执行效率、减少资源占用             |
        | **⑥目标代码生成**  | 优化后的中间代码     | 目标机器代码             | 将中间代码转换为特定硬件平台可执行的目标机器代码                        |
    重点：词法分析和语法分析都是将代码结构进行拆分，语义分析是对代码的正确性进行校验
          词法分析：分析单词使用的是否正确，识别关键字，标识符，
          语法分析：分析语句结构是否合法
          语义分析：分析语句含义是否合法（类型检查）
    错误：词法错误 词法分析阶段发现，非法字符，拼写错误
          语法错误 语法分析阶段发现，括号不匹配，缺少分号
          静态语义错误 语义分析阶段发现，类型不匹配，未声明变量
          动态错误，逻辑错误 运行时发现  除0，数组下标越界

          词法分析阶段发现所有词法层面的错误，但是不是所有拼写错误
          语法阶段发现所有语法层面的错误
          语义分析阶段只能发现静态语义错误，动态语义错误需要运行时才能发现
    符号表：存储源程序中各符号的必要信息，用于 辅助语义的正确性检查和代码生成
            需要支持，查询 插入 修改 删除，最常用哈希表实现，且是最优实现，查找效率最高
正规式和有限自动机
    正规式运算：（Σ={a,b}，x=ab，y=ba），连接运算：两个符号串前后拼接，ε 与任何串连接等于串本身	xy=abba，εx=xε=x
                                       幂运算	xⁿ = n 个 x 连续连接，规定 x⁰=ε	x¹=ab，x²=abab，a⁰=ε
                                       闭包运算	正闭包Σ+：Σ¹∪Σ²∪Σ³∪...，所有长度≥1 的串
                                       克林闭包Σ*：{ε}∪Σ+，所有长度≥0 的串（含空串）	Σ*={ε,a,b,aa,ab,ba,bb,...}
                                       【超级重点】克林闭包*表示 0 次或多次重复，包含空串 ε，
    对于正规式 A B ，有对应正规集 L(A),L(B)
        A|B 也写作A+B 对应正规集L(A)∪L(B) 含义是 或 ， 选A或选B
        AB 也写作 （A・B） 对应正规集 L(A)L(B) 含义是连接 先A后B
        （A）*    对应正规集 （L（A））*，含义是克林闭包 A重复0次或者多次
        优先级：括号>闭包>连接 >或 |，也就是说 a|b*  先算b*  再算a|
    正规式等价：当且仅当两个正规式表示的正规集完全相等的时候，而非形式相似
        (a|b)* = (a*b*)* = (a|b*)* = (a*|b)*，均能表示所有 a、b 组成的串
        a*|b* ≠ (a|b)*，前者只能表示全 a 或全 b 的串，无法表示 ab、ba 等混合串
    自动机：
        确定有限自动机：DFA
        不确定有限自动机：NFA
语法分析：大部分结构比如if for循环等都是采用上下文无关文法： 只要所有规则左部都是单个大写字母（非终结符），就是上下文无关文法
    上下文无关文法的产生式左边必须是非终结符，说白了就是左边不能是不能拆的东西，最基本的组成单位得在最右面
    上下文无关文法是 2 型文法
    最左推导和最右推导：其本质就是按照题目给出的规则，从左侧开始 一个字符一个字符的拼出题目想要的字符串
    如果题目要看那个是上下文无关文法，就看最左侧字符是不是单个的大写字符即可，原因就是，上下文无关文法就决定要拆成不能拆分的字符
    语法分析还有自顶向上和自底向上文法，其中 只要带LL就是自顶向下。带LR就是自底想上
    LALR (1) 是实际编译器最常用的方法，平衡了分析能力和空间开销
    LR 分析的核心是找句柄进行规范归约
    移进 - 归约冲突、归约 - 归约冲突是 LR 分析中常见的冲突类型
    词法分析：检查单个单词（标识符）是否合法 
    语法分析：检查语句结构是否合法 
    语义分析：检查含义、类型是否合法
    方法	名称	核心特点	冲突处理能力	适用场景
    LR(0)	零向前看 LR 分析	无需向前看符号，构造最简单	最弱，极易出现移进 - 归约冲突	几乎无实用价值，仅作为理论基础
    SLR(1)	简单 LR 分析	基于 FOLLOW 集处理冲突，构造简单	中等，能解决部分 LR (0) 的冲突	入门级实用方法
    LR(1)	规范 LR 分析	每个项带 1 个向前看符号，构造最复杂	最强，能处理绝大多数冲突	理论最完善，但分析表体积大
    LALR(1)	向前看 LR 分析	合并 LR (1) 的同心项集，平衡能力和复杂度	中等偏强，比 SLR 强，比 LR (1) 简单	工业级标准，Yacc/Bison 工具使用的方法
表达式求值与后缀表达式
    中缀表达式：人类常用的表达式:a + b*c
    后缀表达式求值:a b c * +` | 运算符在操作数后面
        1. 从左到右扫描后缀表达式
        2. 遇到**操作数**：入栈
        3. 遇到**运算符**：从栈顶弹出两个操作数，进行运算，将结果入栈
        4. 重复步骤 2-3 直到表达式结束
        5. 栈顶元素即为结果
    前缀表达式求值:+ a * b c` | 运算符在操作数前面 
        1. 从右到左扫描前缀表达式
        2. 遇到**操作数**：入栈
        3. 遇到**运算符**：从栈顶弹出两个操作数，进行运算，将结果入栈
        4. 栈顶元素即为结果
