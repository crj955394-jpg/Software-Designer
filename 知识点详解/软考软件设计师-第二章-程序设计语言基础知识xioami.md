## 1. 程序设计语言概述

### 1.1 基本概念

程序设计语言的定义涉及三个方面：

| 方面 | 含义 | 举例 |
|------|------|------|
| **语法** | 程序的构成规则（词法+语法规则） | 标识符命名规则、语句结构 |
| **语义** | 程序的含义（各语法单位的含义） | `if`语句表示条件判断 |
| **语用** | 程序与使用者的关系（上下文环境） | 变量在不同作用域的含义 |

### 1.2 程序设计语言的分类

#### 按语言级别分类

- **低级语言**：面向机器的语言
  - **机器语言**：由0和1组成的指令序列，计算机可以直接执行。可读性差，难以修改和维护
  - **汇编语言**：用助记符代替机器指令，需要汇编程序翻译成机器语言后执行

- **高级语言**：面向各类应用的程序设计语言，接近人类语言
  - 如 Java、C、C++、Python、Pascal 等
  - 需要编译程序或解释程序翻译后才能执行

#### 按翻译方式分类

| 特性 | 编译型 | 解释型 |
|------|--------|--------|
| **工作方式** | 将源程序全部翻译成目标程序后再执行 | 逐句翻译执行，不产生目标程序 |
| **执行效率** | 高（翻译一次，多次执行） | 低（每次执行都需重新解释） |
| **灵活性** | 差 | 好 |
| **可移植性** | 差 | 好 |
| **代表语言** | C、C++、Pascal | Python、JavaScript、早期 BASIC |

> **关键区别**：编译方式下，机器上运行的是与源程序等价的**目标程序**，源程序和编译程序都**不再参与**目标程序的执行过程。解释方式下，源程序和解释程序**始终参与**程序的执行过程。

#### 按编程思维特点分类（重点！）

| 类别 | 特点 | 代表语言 |
|------|------|----------|
| **命令式（结构化）语言** | 基于动作，描述"怎么做"；自顶向下、逐步精化；程序只含顺序、选择、循环三种结构，每种构造单入口单出口 | FORTRAN、Pascal、C |
| **面向对象语言** | 支持封装、继承、多态；始于 Simula；数据隐藏、数据抽象 | Simula、C++、Java、Smalltalk |
| **函数式语言** | 以 λ 演算为基础；始于 LISP；程序和数据形式等价；大量使用递归 | LISP、Scheme、Haskell |
| **逻辑型语言** | 以形式逻辑为基础；关系模型和一阶谓词理论；强推理功能 | PROLOG |

#### 脚本语言

PHP、JavaScript、Python 等。特点是：通常为解释执行，语法简洁灵活，适合快速开发。

### 1.3 程序设计语言的发展历程（了解）

| 语言 | 特点 |
|------|------|
| **FORTRAN** | 第一个被广泛使用的高级语言，用于科学计算。由一个主程序和若干子程序组成 |
| **ALGOL** | 分程序结构的语言；用 BNF 描述语法；支持过程递归调用 |
| **COBOL** | 面向事务处理；用于商业数据处理；语法规则严格 |
| **Pascal** | 结构化程序设计语言；从 ALGOL 60 衍生；过程可以嵌套和递归 |
| **C** | 通用程序设计语言；高效、可直接访问底层硬件；系统级应用和嵌入式开发 |
| **C++** | 在 C 基础上增加类机制，面向对象 |
| **Java** | 保留 C++ 基本语法，删除不好的特征；跨平台 |
| **Python** | 面向对象的解释型语言；可翻译成字节码在虚拟机上运行 |

### ⏰ 练习题

**1.**（2016年真题）将高级语言源程序翻译成目标程序的是（）。
A. 解释程序  B. 编译程序  C. 汇编程序  D. 调试程序

**答案：B**

**解析**：编译程序将源程序翻译成目标语言程序。解释程序是逐句翻译执行不产生目标程序。汇编程序是将汇编语言翻译成机器语言。

**2.**（2019年真题）以下关于程序设计语言的叙述中，错误的是（）。
A. 脚本语言属于动态语言，其程序结构可以在运行时改变
B. 脚本语言一般不具有复杂的语法和严格的类型检查
C. 函数式语言的代表有 LISP、Haskell 等
D. 编译型语言的执行效率一般低于解释型语言

**答案：D**

**解析**：编译型语言先编译成目标代码再执行，执行效率**高于**解释型语言，因为解释型语言每次执行都需要逐句翻译。

## 2. 程序设计语言的基本成分

程序设计语言的基本成分包括四部分：**数据、运算、控制、传输**。

### 2.1 数据成分

数据是程序操作的对象，具有存储类别、类型、名称、作用域和生存期等属性。

#### 数据类型分类

| 类型 | 说明 | 举例 |
|------|------|------|
| **基本类型** | 不可再分解的最小数据单元 | 整型(int)、实型(float)、字符型(char)、布尔型(bool) |
| **用户定义类型** | 用户自定义的枚举类型等 | `enum Color {Red, Green, Blue}` |
| **构造类型** | 由基本类型组合而成 | 数组、结构体(struct)、联合体(union) |
| **指针类型** | 存储变量地址的类型 | `int *p` |
| **空类型** | 表示无返回值 | `void` |

#### 变量的属性

变量具有五个属性：**名、地址、值、作用域、生存期**。

- **作用域**：变量在程序中可以被引用的范围
  - 块级作用域（局部变量）
  - 函数作用域
  - 文件作用域（全局变量）
- **生存期**：变量从分配内存到释放内存的时间段
  - 静态生存期（程序运行期间一直存在）
  - 自动生存期（进入作用域时创建，退出时销毁）

### 2.2 运算成分

指明允许使用的运算符号及运算规则。

| 运算类型 | 符号举例 | 说明 |
|----------|----------|------|
| 算术运算 | `+` `-` `*` `/` `%` | 基本数学运算 |
| 关系运算 | `>` `<` `>=` `<=` `==` `!=` | 比较运算，结果为布尔值 |
| 逻辑运算 | `&&` `\|\|` `!` | 逻辑与、或、非 |
| 位运算 | `&` `\|` `^` `~` `<<` `>>` | 对二进制位操作 |
| 赋值运算 | `=` `+=` `-=` | 赋值操作 |

**运算符的优先级和结合性**是重要的考试考点。

### 2.3 控制成分

指明语言允许表述的控制结构。**任何可计算问题的程序都可以用以下三种控制结构描述：**

```
1. 顺序结构：按语句书写顺序依次执行
2. 选择（分支）结构：根据条件选择执行路径
   - if-else
   - switch-case
3. 循环结构：重复执行某些操作
   - while 型（先判断后执行）
   - do-while 型（先执行后判断）
   - for 型
```

### 2.4 传输成分

指明语言允许的数据传输方式：
- **赋值**：`x = 10`
- **I/O**：`printf()`、`scanf()` 等
- **函数调用**：参数传递（传值、传引用）

### ⏰ 练习题

**3.**（真题改编）程序设计语言的基本成分不包括（）。
A. 数据  B. 运算  C. 控制  D. 存储

**答案：D**

**解析**：程序设计语言的基本成分包括数据、运算、控制和传输四个部分，不包括"存储"。

## 3. 汇编程序基本原理

### 3.1 汇编语言源程序的三类语句

| 类型 | 功能 | 产生机器代码？ |
|------|------|---------------|
| **指令语句**（机器指令语句） | 汇编后产生被 CPU 直接识别并执行的机器代码 | ✅ 是 |
| **伪指令语句** | 指示汇编程序在汇编时完成某些工作（如分配存储单元、给符号赋值） | ❌ 否 |
| **宏指令语句** | 宏的引用，在相应位置使用宏名相当于使用了一段程序 | 替换后产生 |

### 3.2 汇编程序的工作

- **功能**：将汇编语言编写的源程序翻译成机器指令程序
- **基本工作**：
  1. 将每一条可执行汇编语句转换成对应的机器指令
  2. 处理源程序中出现的伪指令
- **扫描次数**：一般需要**两次扫描**源程序才能完成翻译工作

> 第一遍扫描：确定符号的地址（建立符号表）
> 第二遍扫描：将符号替换为实际地址，生成机器代码

### ⏰ 练习题

**4.** 汇编语言中，指示汇编程序在汇编时完成某些工作但不产生机器代码的语句是（）。
A. 指令语句  B. 伪指令语句  C. 宏指令语句  D. 注释语句

**答案：B**

**解析**：伪指令语句用于指示汇编程序在汇编源程序时完成某些工作（如分配存储单元），但不产生机器代码。

## 4. 编译程序基本原理（⭐重点中的重点）

### 4.1 编译程序的功能

把用高级语言书写的**源程序**翻译成与之等价的**目标程序**。

### 4.2 编译的六个阶段

```
源程序 → [词法分析] → [语法分析] → [语义分析] → [中间代码生成] → [代码优化] → [目标代码生成] → 目标程序
```

#### 各阶段详解

| 阶段 | 输入 | 输出 | 主要任务 |
|------|------|------|----------|
| **①词法分析** | 源程序字符串 | 单词符号序列 | 从左到右逐个字符扫描，识别出关键字、标识符、常数、运算符、界符等单词符号 |
| **②语法分析** | 单词符号序列 | 语法树 | 根据语法规则将单词序列分解成各类语法单位（表达式、语句等），检查结构是否合法 |
| **③语义分析** | 语法树 | 带语义信息的语法树 | 检查是否存在语义错误（如类型不匹配），收集类型信息 |
| **④中间代码生成** | 语义分析结果 | 中间代码 | 生成与具体机器无关的中间表示（后缀式、三元式、四元式等） |
| **⑤代码优化** | 中间代码 | 优化后的中间代码 | 对中间代码进行等价变换，提高目标代码的执行效率 |
| **⑥目标代码生成** | 优化后的中间代码 | 目标代码 | 将中间代码转换为特定机器上的目标机器代码 |

> **核心理解**：词法分析和语法分析本质上都是对源程序的**结构**进行分析。语义分析关注的是**含义**的正确性。

#### 各阶段的辨析（⭐高频考点）

| 考查点 | 答案 |
|--------|------|
| 分析单词是否合法 | 词法分析 |
| 分析语句结构是否合法（语法结构） | 语法分析 |
| 分析语句含义是否合法（类型检查等） | 语义分析 |
| 识别关键字、标识符、常数等 | 词法分析 |
| 确定整个语句串是否构成语法正确的程序 | 语法分析 |

### 4.3 词法分析的构造过程（了解）

```
正规式 → 构造 NFA → 转换为 DFA → DFA 最小化 → 构造词法分析器
```

### 4.4 错误类型

| 错误类型 | 发现阶段 | 举例 |
|----------|----------|------|
| **词法错误** | 词法分析 | 非法字符、拼写错误 |
| **语法错误** | 语法分析 | 括号不匹配、缺少分号 |
| **静态语义错误** | 语义分析 | 类型不匹配、未声明变量 |
| **动态错误（逻辑错误）** | 运行时 | 除以0、数组下标越界 |

### 4.5 符号表

符号表是编译过程中非常重要的数据结构，用于记录源程序中各个符号的必要信息，以**辅助语义的正确性检查和代码生成**。

符号表需要支持的操作：**查找、插入、修改和删除**。

常用实现方式：**哈希表**（最优选择，查找效率最高）。

### ⏰ 练习题

**5.**（2017年真题）编译过程中进行的语法分析主要是分析（）。
A. 源程序中的标识符是否合法
B. 程序语句的含义是否合法
C. 程序语句的结构是否合法
D. 表达式的类型是否合法

**答案：C**

**解析**：A属于词法分析，B和D属于语义分析。语法分析检查的是语句的结构（语法）是否合法。

**6.**（2019年真题）对高级程序语言进行编译的过程中，使用（）来记录源程序中各个符号的必要信息，以辅助语义的正确性检查和代码生成。
A. 决策表  B. 符号表  C. 广义表  D. 索引表

**答案：B**

**解析**：符号表用于记录源程序中各个符号（变量名、函数名等）的必要信息，是编译过程中最重要的数据结构之一。

**7.**（2016年真题）以下关于编译器各阶段的叙述中，正确的是（）。
A. 词法分析阶段可以发现程序中所有的拼写错误
B. 语法分析阶段可以发现程序中所有的语法错误
C. 语义分析阶段可以发现程序中所有的语义错误
D. 中间代码生成阶段生成的目标代码效率最高

**答案：B**

**解析**：语法分析阶段可以发现程序中所有的语法错误。词法分析只能发现词法层面的错误，语义分析通常只能做静态语义检查，动态语义错误在运行时才能发现。

## 5. 正规式与有限自动机（⭐必考）

### 5.1 正规式（正则表达式）

正规式是用来描述单词构成规则的表达式形式。三种等价的描述方式：
- **正规式**：用表达式形式描述单词构成
- **正规文法**：从产生单词的角度描述
- **有限自动机**：从识别的角度描述

#### 正规式的定义

在字母表 Σ 上：
1. **ε** 和 **∅** 都是正规式，表示的正规集分别为 {ε} 和 ∅
2. 对于任何 a ∈ Σ，**a** 是正规式，表示的正规集为 {a}
3. 若 U 和 V 都是正规式，表示的正规集为 L(U) 和 L(V)，则：
   - **(U|V)** 是正规式，表示 L(U)∪L(V)（或运算）
   - **(U·V)** 是正规式，表示 L(U)L(V)（连接运算）
   - **U\*** 是正规式，表示 (L(U))\*（闭包运算）

#### 运算优先级

```
*（闭包） > ·（连接） > |（或）
```

> 连接符 · 通常省略不写。

#### 常用正规式模式（⭐重要）

| 正规式 | 含义 |
|--------|------|
| `a*` | 由零个或多个 a 组成的串 |
| `a+` (= aa*) | 由一个或多个 a 组成的串 |
| `(a|b)*` | 由 a 和 b 组成的任意串 |
| `a(a|b)*` | 以 a 开头的由 a、b 组成的串 |
| `(a|b)*a` | 以 a 结尾的由 a、b 组成的串 |
| `(0|1)*` | 由 0 和 1 组成的任意串（任意二进制串） |
| `1(0|1)*` | 以 1 开头的二进制串 |
| `(0|1)*0` | 以 0 结尾的二进制串 |

#### 正规式的等价

若两个正规式表示的正规集相同，则称它们**等价**。
例如：`b(ab)*` = `(ba)*b`

### 5.2 确定有限自动机（DFA）

**定义**：DFA 是一个五元组 M = (S, Σ, f, s₀, F)

| 符号 | 含义 |
|------|------|
| S | 有穷状态集 |
| Σ | 输入符号的字母表 |
| f | 状态转换函数：S × Σ → S（一对一） |
| s₀ | 唯一的初始状态 |
| F | 终态集（F ⊆ S） |

**DFA 的特征**：
- 每个状态对每个输入符号，**最多只有一个**转换
- 不允许 ε 转换

#### DFA 如何识别字符串

从初始状态 s₀ 开始，逐个读取输入符号，按照状态转换函数 f 进行状态转移。若读完所有输入符号后，DFA 处于某个终态，则称该字符串**被接受**；否则**被拒绝**。

### 5.3 非确定有限自动机（NFA）

**定义**：NFA 也是一个五元组 M = (S, Σ, f, s₀, F)，但与 DFA 有区别：

| 区别 | DFA | NFA |
|------|-----|-----|
| 初始状态 | 唯一 | 可以有多个 |
| 状态转换 | 对每个输入符号最多一个转换 | 对同一个输入符号可以有**多个**转换 |
| ε 转换 | 不允许 | 允许 |
| f 的定义 | S × Σ → S | S × (Σ∪{ε}) → 2^S |

### 5.4 NFA 转化为等价的 DFA（子集法）⭐

**核心思想**：DFA 的每个状态对应 NFA 的一组状态集合。

**步骤**：

**第一步**：将 NFA 规范化（如果需要）
- 增加唯一的初态和终态
- 将弧上的字符串改为单个字符或 ε

**第二步**：子集构造法
1. 计算初始状态的 ε-闭包 ε_closure(s₀)，作为 DFA 的初始状态
2. 对于当前状态集合 I 和输入符号 a，计算 move(I, a) 的 ε-闭包
3. 将新的状态集合加入 DFA
4. 重复直到没有新的状态集合产生

**ε-闭包的定义**：从状态集合 I 中的任一状态出发，只经过 ε 转换能到达的所有状态的集合。

### 5.5 DFA 的化简（最小化）

**目标**：找到一个状态数最少的 DFA M'，使得 L(M) = L(M')

**方法——分割法**：
1. 将所有状态分为**终态集**和**非终态集**两个子集
2. 检查每个子集中的状态是否等价（对每个输入符号的转换目标是否在同一子集中）
3. 若不等价，则分割该子集
4. 重复直到所有子集中的状态都等价
5. 从每个子集中选一个代表状态

### 5.6 正规式与 DFA 的转换

- **正规式 → NFA**：Thompson 构造法
- **NFA → DFA**：子集构造法
- **DFA → 最小 DFA**：分割法

## 6. 上下文无关文法与语法分析（⭐必考）

### 6.1 文法基础

#### 文法的分类（乔姆斯基体系）

| 类型 | 名称 | 产生式形式 | 说明 |
|------|------|-----------|------|
| 0 型 | 短语文法 | α → β（α 至少含一个非终结符） | 图灵机可识别 |
| 1 型 | 上下文有关文法 | αAβ → αγβ（\|αAβ\| ≤ \|αγβ\|） | 线性有界自动机 |
| **2 型** | **上下文无关文法** | **A → β**（A 为非终结符） | 下推自动机，**考试重点** |
| 3 型 | 正规文法/正则文法 | A → aB 或 A → a（右线性）；A → Ba 或 A → a（左线性） | 有限自动机 |

> **考试重点**：程序设计语言的绝大多数语法规则采用**上下文无关文法（2 型文法）**进行描述。

#### 上下文无关文法的定义

产生式形式为 **A → β**，其中：
- A 是**非终结符**（必须单独出现在产生式左边）
- β 是由终结符和非终结符组成的符号串

**示例**：描述算术表达式的文法 G[E]：
```
E → E + T | T
T → T * F | F
F → (E) | id
```
其中：E、T、F 是非终结符；+、*、(、)、id 是终结符。

#### 关键概念

| 概念 | 定义 |
|------|------|
| **终结符** | 组成语言的基本符号（不可再分解），用小写字母或符号表示 |
| **非终结符** | 可以被进一步分解的语法单位，用大写字母表示 |
| **产生式** | 定义非终结符如何被替换的规则，形式为 A → β |
| **句型** | 从开始符号出发，经过若干步推导得到的符号串（可以含非终结符） |
| **句子** | 仅由终结符组成的句型 |
| **语言** | 文法产生的所有句子的集合 |
| **开始符号** | 推导的起点（特殊非终结符） |

### 6.2 推导方式

#### 最左推导与最右推导（规范推导）

- **最左推导**：每一步替换句型中**最左边**的非终结符
- **最右推导（规范推导）**：每一步替换句型中**最右边**的非终结符

**示例**：对于文法 `E → E+T | T, T → T*F | F, F → (E) | id`

推导 id + id * id：

**最左推导**：
```
E ⇒ E+T ⇒ T+T ⇒ F+T ⇒ id+T ⇒ id+T*F ⇒ id+F*F ⇒ id+id*F ⇒ id+id*id
```

**最右推导**：
```
E ⇒ E+T ⇒ E+T*F ⇒ E+T*id ⇒ E+F*id ⇒ E+id*id ⇒ T+id*id ⇒ F+id*id ⇒ id+id*id
```

### 6.3 语法树与二义性

#### 语法树（推导树）

语法树是推导过程的图形表示：
- 根节点：开始符号
- 内部节点：非终结符
- 叶子节点：终结符或非终结符
- 从左到右读取叶子节点得到句型

#### 文法的二义性

**定义**：若一个句子存在**两棵不同的语法树**（或两种不同的最左推导/最右推导），则称该文法是**二义性**的。

> 注意：二义性是针对**文法**而言的，不是针对语言。同一个语言可以有不同的文法描述，其中一些可能是二义性的，另一些不是。

**常见二义性示例**：

文法 `E → E+E | E*E | id` 对于句子 `id+id*id` 存在两棵语法树：
- 一种理解为 `(id+id)*id`
- 另一种理解为 `id+(id*id)`

消除二义性的方法：引入优先级层次
```
E → E+T | T
T → T*F | F
F → (E) | id
```

### 6.4 语法分析方法

#### 自顶向下分析

从开始符号出发，尝试构造输入串的推导。

- **LL(1) 分析法**：最常用的自顶向下分析方法
  - L：从左到右扫描输入
  - L：产生最左推导
  - 1：向前看一个符号

要求文法**无二义性、无左递归、无公共前缀**。

#### 自底向上分析

从输入串出发，尝试归约到开始符号。

- **LR 分析法**：最常用的自底向上分析方法
  - L：从左到右扫描输入
  - R：产生最右推导的逆过程

LR(0)、SLR(1)、LALR(1)、LR(1) 等变体。

## 7. 表达式求值与后缀表达式（⭐必考）

### 7.1 表达式的三种表示方式

以表达式 `a + b*c` 为例：

| 表示方式 | 形式 | 说明 |
|----------|------|------|
| **中缀表达式** | `a + b*c` | 运算符在操作数中间（人最习惯的写法） |
| **前缀表达式**（波兰式） | `+ a * b c` | 运算符在操作数前面 |
| **后缀表达式**（逆波兰式） | `a b c * +` | 运算符在操作数后面 |

### 7.2 中缀转后缀（⭐高频考点）

#### 方法一：根据语法树

**后缀表达式 = 语法树的后序遍历**（左子树 → 右子树 → 根节点）

#### 方法二：直接转换规则

1. 确定运算符的优先级和结合性
2. 加括号标明运算顺序
3. 将运算符移到对应的右括号后面
4. 去掉所有括号

**示例**：`a + b * (c - d) / e`

步骤：
```
原式：a + b * (c - d) / e
加括号：(a + ((b * (c - d)) / e))
移运算符：a b (c d - ) * e / +
去括号：a b c d - * e / +
```

后缀表达式为：`a b c d - * e / +`

### 7.3 后缀表达式求值

**算法**（使用栈）：
1. 从左到右扫描后缀表达式
2. 遇到**操作数**：入栈
3. 遇到**运算符**：从栈顶弹出两个操作数，进行运算，将结果入栈
4. 重复步骤 2-3 直到表达式结束
5. 栈顶元素即为结果

**示例**：求后缀表达式 `2 3 4 + * 5 -` 的值
```
扫描 2：栈 = [2]
扫描 3：栈 = [2, 3]
扫描 4：栈 = [2, 3, 4]
扫描 +：弹出 4 和 3，3+4=7，栈 = [2, 7]
扫描 *：弹出 7 和 2，2*7=14，栈 = [14]
扫描 5：栈 = [14, 5]
扫描 -：弹出 5 和 14，14-5=9，栈 = [9]
结果 = 9
```

等价中缀表达式：`(2 * (3 + 4)) - 5 = 9` ✓

### 7.4 前缀表达式求值

**算法**（使用栈，从右到左扫描）：
1. 从右到左扫描前缀表达式
2. 遇到**操作数**：入栈
3. 遇到**运算符**：从栈顶弹出两个操作数，进行运算，将结果入栈
4. 栈顶元素即为结果

### 7.5 从语法树获取表达式

| 表达式类型 | 遍历方式 |
|------------|----------|
| 前缀表达式 | 前序遍历（根 → 左 → 右） |
| 中缀表达式 | 中序遍历（左 → 根 → 右） |
| 后缀表达式 | 后序遍历（左 → 右 → 根） |

### ⏰ 练习题

## 8. 函数调用与参数传递（⭐必考）

### 8.1 参数传递方式

#### 传值调用（Call by Value）

- **机制**：将实参的**值**复制给形参，形参和实参各占独立的存储空间
- **特点**：
  - 函数内部对形参的修改**不影响**实参
  - 实参可以是常量、变量、表达式
  - 安全性高，但需要额外的存储空间和复制开销

```
void swap(int a, int b) {
    int t = a; a = b; b = t;  // 不影响实参
}
int x = 1, y = 2;
swap(x, y);
// x 仍然为 1，y 仍然为 2
```

#### 传引用调用（Call by Reference）

- **机制**：将实参的**地址**（引用）传递给形参，形参和实参共享同一存储空间
- **特点**：
  - 函数内部对形参的修改**就是**对实参的修改
  - 实参必须是变量（不能是常量或表达式）
  - 效率高（不需要复制），但可能产生副作用

```
void swap(int &a, int &b) {
    int t = a; a = b; b = t;  // 修改实参
}
int x = 1, y = 2;
swap(x, y);
// x 变为 2，y 变为 1
```

#### 对比总结

| 特性 | 传值调用 | 传引用调用 |
|------|----------|-----------|
| 传递的内容 | 实参的**值** | 实参的**地址** |
| 形参变化影响实参？ | ❌ 不影响 | ✅ 影响 |
| 实参要求 | 常量、变量、表达式均可 | 必须是变量 |
| 效率 | 较低（需复制） | 较高（共享存储） |
| 安全性 | 高 | 低（可能副作用） |

#### 传名调用（了解）

ALGOL 60 使用的方式。将实参的名字（而非值或地址）传给形参，在使用形参时才对实参进行求值。会产生宏替换的效果。

### 8.2 活动记录（栈帧）

函数调用时，系统会为每次调用创建一个**活动记录**（Activation Record / Stack Frame），通常包含：

```
+---------------------+
|    返回值空间        |
+---------------------+
|    实参（实际参数）   |
+---------------------+
|    控制链（访问链）   |
+---------------------+
|    返回地址          |
+---------------------+
|    形式参数          |
+---------------------+
|    局部变量          |
+---------------------+
|    临时变量          |
+---------------------+
```

### 8.3 调用方式（了解）

| 调用方式 | 特点 |
|----------|------|
| 传值（call by value） | 复制值 |
| 传地址（call by reference） | 传递地址 |
| 传结果（call by result） | 返回时将值写回实参 |
| 传名（call by name） | 使用时才求值 |

### ⏰ 练习题

## 9. 中间代码与代码优化

### 9.1 中间代码

**定义**：源程序经过语义分析后，首先转换为一种与具体机器无关的中间表示形式，称为中间代码。

**为什么要用中间代码？**
- 源程序与目标代码的逻辑结构差别大
- 一次翻译到位很困难
- 用语法制导方式机械生成的目标代码往往繁琐和低效
- 有利于进行与机器无关的优化处理

#### 中间代码的形式

| 形式 | 说明 | 示例 |
|------|------|------|
| **后缀式（逆波兰式）** | 运算符放在操作数后面 | `a b +` 表示 `a + b` |
| **树形表示** | 用树结构表示表达式 | 二叉树 |
| **三元式** | (运算符, 操作数1, 操作数2) | `(+, a, b)` 表示 `a + b` |
| **四元式** | (运算符, 操作数1, 操作数2, 结果) | `(+, a, b, t₁)` 表示 `t₁ = a + b` |

> **四元式**是最常用的中间代码形式，与三地址码等价。

#### 三地址码

每个四元式最多包含一个运算符，形如：`x = y op z`

**示例**：表达式 `a + b * c` 的四元式序列：
```
(1) (*, b, c, t1)     // t1 = b * c
(2) (+, a, t1, t2)    // t2 = a + t1
```

### 9.2 代码优化

**定义**：对中间代码进行等价变换，以提高目标代码的执行效率。

代码优化建立在对程序的**控制流**和**数据流**分析基础之上，与具体的机器无关。

#### 常见优化技术

| 类型 | 说明 |
|------|------|
| **常量折叠** | 编译时计算常量表达式，如 `3+4` → `7` |
| **公共子表达式消除** | 相同表达式只计算一次 |
| **循环优化** | 循环不变量外提（将循环中不变的计算移到循环外） |
| **强度削弱** | 用较快运算替换较慢运算，如 `x*2` → `x+x` |
| **删除归纳变量** | 消除不必要的循环控制变量 |
| **复写传播** | 用常量或变量替换已赋值的变量 |

### 9.3 目标代码生成

**功能**：把中间代码变换成特定机器上的目标代码。

需要考虑的因素：
- 计算机的体系结构
- 寄存器分配
- 指令选择
- 指令调度

### ⏰ 练习题

**17.**（2018年真题）对高级程序语言进行编译的过程中，对源程序进行语义分析之后就可以直接生成目标代码，但考虑到（），有必要先生成中间代码。
A. 便于进行代码优化
B. 便于进行寄存器分配
C. 便于进行与机器无关的优化处理
D. 便于进行目标代码的生成

**答案：C**

**解析**：源程序与目标代码的逻辑结构差别很大，一次翻译到位很困难。中间代码与具体机器无关，此时所做的优化一般建立在对程序的控制流和数据流分析的基础之上，与具体的机器无关。

**18.** 表达式 `a = b * c + d - e / f` 的四元式序列为（）。

**答案**：
```
(1) (*, b, c, t1)     // t1 = b * c
(2) (+, t1, d, t2)    // t2 = t1 + d
(3) (/, e, f, t3)     // t3 = e / f
(4) (-, t2, t3, t4)   // t4 = t2 - t3
(5) (=, t4, _, a)     // a = t4
```

## 10. 综合练习题与历年真题精选

### 选择题

**19.**（2015年真题）以下关于高级程序设计语言实现的编译和解释方式的叙述中，正确的是（）。
A. 编译程序不参与用户程序的运行控制，而解释程序则参与
B. 在编译方式下，机器上运行的是与源程序等价的目标程序
C. 在解释方式下，机器上运行的是与源程序等价的目标程序
D. 编译方式比解释方式更适合于交互式程序的开发

**答案：A、B**

（注：本题应为多选题或需判断正误）

**解析**：
- 编译程序不参与目标程序的运行控制（A对）
- 编译方式下运行的是目标程序（B对）
- 解释方式下运行的仍然是源程序（C错）
- 解释方式更适合交互式程序开发（D错）

**20.**（2019年真题）将源程序翻译成目标程序的过程中，对源程序中出现的（）进行分析属于语义分析阶段的任务。
A. 单词的构成是否合法
B. 语句的结构是否合法
C. 表达式中运算对象的类型是否匹配
D. 标识符的命名是否规范

**答案：C**

**解析**：A属于词法分析，B属于语法分析，D属于词法分析（标识符命名规则是词法层面）。类型检查是语义分析的核心任务。

**21.**（2018年真题）表达式 `a*(b+c)-d` 的后缀式为（）。
A. `a b c + * d -`  B. `a b c * + d -`  C. `- * + a b c d`  D. `a b c d - * +`

**答案：A**

**解析**：先算 `b+c` → `b c +`，再算 `a*(b+c)` → `a b c + *`，再算 `-d` → `a b c + * d -`。

**22.**（2022年真题）正规式 `(a|b)*a(a|b)` 描述的正规集是（）。
A. 所有由 a 和 b 组成的串
B. 倒数第二个字符为 a 的由 a 和 b 组成的串
C. 倒数第一个字符为 b 的由 a 和 b 组成的串
D. 以 a 开头的由 a 和 b 组成的串

**答案：B**

**解析**：(a|b)* 表示任意多个 a 或 b 的串，后面跟 a（倒数第二个是 a），再跟 (a|b)（最后一个是 a 或 b）。所以倒数第二个字符一定是 a。

**23.**（2017年真题）对于一个文法，若存在两棵不同的语法树，则说明该文法是（）的。
A. 正规  B. 二义性  C. 上下文有关  D. 递归

**答案：B**

**解析**：若一个句子存在两棵不同的语法树（或两种不同的最左/最右推导），则称该文法是二义性的。

**24.**（2020年真题）在对编译器各阶段的描述中，（）是正确的。
A. 词法分析阶段的主要任务是对各条语句的结构进行合法性分析
B. 语法分析阶段的主要任务是识别单词并输出单词符号
C. 语义分析阶段的主要任务是检查语句的含义是否合法
D. 中间代码生成阶段的主要任务是将目标代码翻译成机器指令

**答案：C**

**解析**：A是语法分析的任务，B是词法分析的任务，D是目标代码生成阶段的任务。语义分析检查含义是否合法（如类型检查）。

**25.**（2023年真题改编）设有文法 G[S]：S → aSb | ε，该文法描述的语言是（）。
A. {aⁿbⁿ | n ≥ 0}
B. {aⁿbⁿ | n ≥ 1}
C. {aⁿb²ⁿ | n ≥ 0}
D. {aⁿb²ⁿ | n ≥ 1}

**答案：A**

**解析**：S→ε 得到空串（n=0），S→aSb 递归产生 a 和 b 配对，n≥0。所以语言是 {aⁿbⁿ | n ≥ 0}。
