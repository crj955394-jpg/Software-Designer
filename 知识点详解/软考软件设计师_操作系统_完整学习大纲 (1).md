## 第一章 操作系统概述

### 1.1 操作系统的定义与作用

**官方教材定义**：
> 操作系统（Operating System, OS）是管理计算机硬件与软件资源的程序，是计算机系统的核心系统软件。它为用户和应用程序提供一个与硬件之间的接口，使用户能够方便地使用计算机。

**通俗理解**：
操作系统就是计算机的"管家"——你不需要直接操作 CPU、内存、硬盘这些硬件，OS 帮你统一管理、分配和调度。

**核心作用**（考试常考）：
- **资源管理**：管理 CPU、内存、外设等硬件资源
- **人机接口**：提供命令行、图形界面等交互方式
- **扩展机器**：将裸机扩充为功能更强、使用更方便的虚拟机

### 1.2 操作系统的特征（高频考点）

| 特征 | 官方定义 | 理解要点 |
|------|----------|----------|
| **并发性** | 两个或多个事件在同一时间间隔内发生 | ⚠️ 并发≠并行。并行是同一时刻同时发生（需多核），并发是宏观上同时、微观上交替执行 |
| **共享性** | 系统中的资源可供多个并发执行的进程共同使用 | 分为互斥共享（如打印机）和同时共享（如磁盘） |
| **虚拟性** | 通过某种技术把一个物理实体变为若干个逻辑实体 | 虚拟处理器（多道程序）、虚拟存储器、虚拟设备 |
| **不确定性/异步性** | 进程以不可预知的速度向前推进 | 同一程序、同一数据，多次运行结果和时间可能不同 |

> 🎯 **真题考点**：并发和共享是操作系统最基本的两个特征，互为存在条件。

### 1.3 操作系统的分类

| 类型 | 特点 | 典型场景 |
|------|------|----------|
| **批处理系统** | 作业成批处理，用户不干预执行过程 | 早期大型机 |
| **分时系统** | 时间片轮转，多用户同时交互 | UNIX |
| **实时系统** | 及时响应、高可靠性 | 工业控制、航空航天 |
| **网络操作系统** | 基于计算机网络，提供网络服务 | Windows Server |
| **分布式系统** | 多机协同，对用户透明 | 分布式计算集群 |
| **嵌入式操作系统** | 微型化、可定制、实时性、可靠性 | μC/OS、嵌入式 Linux |

**嵌入式 OS 特点**（近年新增考点）：
- 微型化：资源受限环境
- 可定制：按需裁剪功能
- 实时性：硬实时/软实时
- 可靠性：长时间稳定运行
- 易移植性：跨硬件平台

**初始化顺序**：片级初始化 → 板级初始化 → 系统级初始化

### 1.4 分时系统与实时系统的对比（经典考点）

| 对比项 | 分时系统 | 实时系统 |
|--------|----------|----------|
| 设计目标 | 用户交互性 | 任务及时完成 |
| 响应时间 | 时间片级别（秒级） | 毫秒/微秒级 |
| 可靠性 | 一般 | 高 |
| 交互性 | 强 | 弱（专用） |
| 典型场景 | 多用户终端 | 控制系统、航空 |

### 📝 本章典型真题

**【2022年真题改编】**
> 操作系统的四个基本特征是（）。
> A. 并发性、共享性、虚拟性、不确定性
> B. 并发性、独占性、虚拟性、异步性
> C. 并行性、共享性、虚拟性、不确定性
> D. 并行性、独占性、虚拟性、异步性

**答案：A**
**解析**：操作系统四大特征为并发性、共享性、虚拟性、不确定性（也称异步性）。注意"并发"不是"并行"，并发是时间间隔内交替，真正同时才叫并行。

**【2021年真题改编】**
> 以下关于分时系统和实时系统的叙述中，正确的是（）。
> A. 分时系统的响应时间一定优于实时系统
> B. 实时系统的交互性比分时系统强
> C. 实时系统比分时系统的可靠性更高
> D. 分时系统和实时系统都使用时间片轮转

**答案：C**
**解析**：实时系统的设计目标是高可靠性和及时响应，交互性不是其强项。分时系统才强调交互性。

## 第二章 进程管理（★★★ 核心重点）

### 2.1 进程的基本概念

**官方教材定义**：
> 进程是程序在一个数据集合上运行的过程，是系统进行资源分配和调度的基本单位。进程由程序、数据和进程控制块（PCB）三部分组成。

**进程的组成**（三要素）：

| 组成部分 | 作用 |
|----------|------|
| **程序** | 描述进程要完成的功能 |
| **数据** | 进程执行时所需的数据和工作区 |
| **PCB（进程控制块）** | 进程存在的唯一标志，包含进程标识、状态、优先级、资源清单等 |

**通俗理解**：
- 程序是静态的菜谱，进程是动态的做菜过程
- PCB 是进程的"身份证"，包含这个进程的一切信息
- OS 通过 PCB 来管理和控制进程

### 2.2 进程的状态与转换（★★★ 必考）

**三态模型**：

```
    ┌──────────────────────────┐
    │                          │
    ▼                          │
 就绪态 ──(被调度)──▶ 运行态 ──(时间片到)──▶ 就绪态
                     │                      ▲
                     │(等待事件)            │(等待事件发生)
                     ▼                      │
                   阻塞态 ──────────────────┘
```

| 状态 | 定义 |
|------|------|
| **运行态** | 进程已获得 CPU，正在执行 |
| **就绪态** | 进程已获得除 CPU 之外的所有必要资源，等待调度 |
| **阻塞态（等待态）** | 进程因等待某事件（如 I/O 完成）而暂停执行 |

**状态转换条件**（必须牢记）：

| 转换 | 触发条件 |
|------|----------|
| 就绪 → 运行 | 进程被调度程序选中（获得 CPU） |
| 运行 → 就绪 | 时间片用完 / 被更高优先级进程抢占 |
| 运行 → 阻塞 | 请求 I/O / 申请资源失败 / 等待某事件 |
| 阻塞 → 就绪 | 等待的事件发生（如 I/O 完成） |

> ⚠️ **注意**：阻塞态不能直接到运行态！必须先到就绪态，等调度后才能运行。

**五态模型**（扩展了解）：
在三态基础上增加**新建态**和**终止态**：
- 新建态：进程正在被创建
- 终止态：进程执行完毕，等待系统回收资源

### 2.3 进程的同步与互斥

**关键概念**：

| 概念 | 定义 |
|------|------|
| **同步** | 多个进程在执行次序上的协调，有明确的先后关系（如：生产者必须先生产，消费者才能消费） |
| **互斥** | 多个进程不能同时使用同一临界资源（如：打印机同一时刻只能服务一个进程） |
| **临界资源** | 一次只允许一个进程访问的资源 |
| **临界区** | 进程中访问临界资源的那段代码 |

**同步机制准则**（教材原文）：
1. 空闲让进：临界区空闲时，允许一个请求进入
2. 忙则等待：临界区被占用时，其他请求必须等待
3. 有限等待：等待进入临界区的时间必须有限
4. 让权等待：不能进入临界区的进程应释放 CPU

### 2.4 PV 操作与信号量（★★★ 超高频考点）

**信号量定义**：
> 信号量 S 是一个整型变量，用于表示系统中某类资源的数量。通过 P、V 操作实现进程同步和互斥。

**P 操作（wait 操作，申请资源）**：
```
S = S - 1;
if S < 0 then
    将该进程插入等待队列，阻塞该进程;
```

**V 操作（signal 操作，释放资源）**：
```
S = S + 1;
if S ≤ 0 then
    从等待队列中唤醒一个进程;
```

**信号量含义**：
- S > 0：表示可用资源数量
- S = 0：表示无可用资源，但也没有进程等待
- S < 0：|S| 表示等待队列中的进程数

**经典模型**：

#### ① 互斥模型（单资源）
```
Semaphore S = 1;  // 互斥信号量

进程 Pi:              进程 Pj:
  P(S);                P(S);
  临界区;              临界区;
  V(S);                V(S);
```

#### ② 生产者-消费者模型
```
Semaphore mutex = 1;  // 互斥信号量
Semaphore empty = n;  // 空缓冲区数
Semaphore full = 0;   // 满缓冲区数

生产者:                消费者:
  P(empty);             P(full);
  P(mutex);             P(mutex);
  放入产品;             取出产品;
  V(mutex);             V(mutex);
  V(full);              V(empty);
```

#### ③ 读者-写者模型
```
Semaphore rmutex = 1;  // 读者互斥
Semaphore wmutex = 1;  // 写者互斥
int readcount = 0;     // 读者计数

读者:                    写者:
  P(rmutex);              P(wmutex);
  readcount++;            写操作;
  if readcount == 1       V(wmutex);
    P(wmutex);
  V(rmutex);
  读操作;
  P(rmutex);
  readcount--;
  if readcount == 0
    V(wmutex);
  V(rmutex);
```

> 🎯 **PV操作解题技巧**：
> 1. 先确定哪些是互斥关系（信号量初始值=1）
> 2. 再确定哪些是同步关系（信号量初始值=n 或 0）
> 3. P 操作顺序很重要——互斥 P 放在同步 P 之后，否则可能死锁
> 4. V 操作顺序无所谓

### 2.5 进程调度（★★）

**调度层次**：

| 层次 | 调度对象 | 频率 |
|------|----------|------|
| 高级调度（作业调度） | 作业从外存调入内存 | 低 |
| 中级调度（内存调度） | 内外存对换 | 中 |
| 低级调度（进程调度） | 就绪队列中选进程执行 | 高 |

**常见调度算法**（考试重点）：

| 算法 | 原理 | 特点 | 考试注意 |
|------|------|------|----------|
| **FCFS（先来先服务）** | 按到达顺序调度 | 简单，但对短作业不利 | 可能出现"护航效应" |
| **SJF/SPF（短作业/进程优先）** | 选执行时间最短的 | 平均等待时间最优 | 对长作业不利，可能导致饥饿 |
| **优先级调度** | 按优先级高低调度 | 灵活 | 分为抢占式和非抢占式 |
| **时间片轮转** | 每个进程分配固定时间片 | 公平，适合分时系统 | 时间片大小是关键 |
| **多级反馈调度** | 综合优先级和时间片 | 兼顾各方面性能 | 现代 OS 最常用 |

### 2.6 死锁（★★★ 必考）

**死锁定义**：
> 两个或两个以上的进程在执行过程中，因争夺资源而造成的一种互相等待的现象，若无外力作用，它们都将无法推进下去。

**死锁产生的四个必要条件**（必须全部满足才会死锁）：

| 条件 | 含义 |
|------|------|
| **互斥条件** | 资源一次只能被一个进程使用 |
| **占有并等待（请求和保持）** | 进程持有至少一个资源，同时等待其他进程持有的资源 |
| **不可剥夺** | 已分配给进程的资源不能被强制剥夺 |
| **循环等待** | 存在一个进程-资源的环形链 |

**死锁处理策略**：

| 策略 | 方法 | 说明 |
|------|------|------|
| **预防** | 破坏四个必要条件之一 | 静态分配、可剥夺资源等 |
| **避免** | 银行家算法 | 动态检查，安全才分配 |
| **检测** | 资源分配图 | 允许死锁发生，定期检测 |
| **解除** | 剥夺资源/撤销进程 | 发现死锁后处理 |

**死锁资源计算公式**（★ 超高频计算题）：
- 系统有 **n** 个进程，每个进程需要 **R** 个资源才能完成
- **发生死锁的最大资源数**：n × (R - 1)
- **不发生死锁的最小资源数**：n × (R - 1) + 1

**推导理解**：
最坏情况——每个进程都差一个资源（各持有 R-1 个），谁也完不成 → 总共 n×(R-1) 个资源时刚好死锁。多 1 个资源就能让某个进程完成并释放资源。

**银行家算法**（经典考法）：
- 系统有 m 种资源，n 个进程
- Available：系统可用资源向量
- Max：各进程最大需求矩阵
- Allocation：各进程已分配矩阵
- Need = Max - Allocation：各进程还需要的资源

**安全性检查步骤**：
1. 计算 Work = Available
2. 找满足 Need ≤ Work 的进程
3. 执行该进程，Work = Work + Allocation
4. 重复直到所有进程完成（安全）或找不到满足条件的进程（不安全）

### 2.7 线程（★）

**进程与线程的区别**（教材对比）：

| 对比项 | 进程 | 线程 |
|--------|------|------|
| 定义 | 资源分配的基本单位 | CPU 调度的基本单位 |
| 拥有资源 | 拥有独立的内存空间 | 共享进程的资源 |
| 开销 | 创建/切换开销大 | 创建/切换开销小 |
| 通信 | 需要 IPC 机制 | 可直接读写共享数据 |
| 独立性 | 独立 | 同一进程内线程互相依赖 |

**关键结论**（考试常考）：
- 进程是**资源分配**的基本单位
- 线程是**CPU 调度**的基本单位
- 同一进程内的线程共享该进程的地址空间和资源
- 线程有自己独立的：程序计数器、寄存器、栈

### 📝 本章典型真题

**【2023年真题改编 — 进程状态】**
> 在单处理机系统中，若系统中有 n 个进程，则就绪队列中进程的个数最多为（）。
> A. n-1　　B. n　　C. 1　　D. 0

**答案：A**
**解析**：在单处理机系统中，n 个进程中最多只有 1 个在运行态，其余 n-1 个最多在就绪队列中。

**【2022年真题改编 — 死锁计算】**
> 某系统中有 3 个并发进程，每个进程都需要 4 个同类资源才能运行完成，则系统至少需要多少个资源才不会发生死锁？
> A. 9　　B. 10　　C. 11　　D. 12

**答案：B**
**解析**：代入公式 n×(R-1)+1 = 3×(4-1)+1 = 3×3+1 = 10

**【2021年真题改编 — 信号量值判断】**
> 某系统有 5 台打印机，有 3 个进程竞争使用。每个进程最多需要 2 台打印机。用信号量 S 表示可用打印机数，初始值为 5。当 S 的值为 -1 时，表示系统中（）。
> A. 有 1 台打印机空闲　　B. 有 1 个进程等待打印机
> C. 有 0 台打印机空闲，1 个进程等待　　D. 有 1 台打印机空闲，0 个进程等待

**答案：C**
**解析**：S < 0 时，|S| 即为等待队列中进程数。S = -1 说明有 1 个进程在等待，且 5 台打印机已全部分配。

**【2020年真题改编 — PV 操作】**
> 进程 P1 和 P2 共享变量 S1 和 S2，需要互斥访问。设信号量 S1 和 S2 的初值均为 1，P1 中访问 S1 和 S2 的代码如下：
> ```
> P(S1); P(S2); 访问 S1 和 S2; V(S2); V(S1);
> ```
> 如果 P2 中代码与 P1 类似但先 P(S2) 再 P(S1)，则可能导致（）。
> A. 死锁　　B. 饥饿　　C. 数据丢失　　D. 优先级反转

**答案：A**
**解析**：P1 先 P(S1) 再 P(S2)，P2 先 P(S2) 再 P(S1)，形成循环等待，满足死锁条件。**解题技巧**：所有进程的 P 操作顺序保持一致即可避免。

## 第三章 存储管理（★★★ 核心重点）

### 3.1 存储管理概述

**官方教材定义**：
> 存储管理的主要任务是为多道程序的运行提供良好的运行环境，方便用户使用存储器，提高存储器的利用率，并能从逻辑上扩充内存。

**存储管理方式分类**：

| 方式 | 特点 | 是否连续分配 |
|------|------|:------------:|
| **分区存储管理** | 将内存分为若干区域 | ✓ |
| **分页存储管理** | 将内存和程序都分成等大小的页 | ✗ |
| **分段存储管理** | 按程序逻辑结构分段 | ✗ |
| **段页式存储管理** | 先分段再分页 | ✗ |

### 3.2 分区存储管理（★★ 考试偶考选择题）

#### 3.2.1 分区方式详解

**（1）固定分区**

**教材定义**：预先将内存划分为若干个大小固定的分区，每个分区装入一个作业。分区大小可以相等也可以不等。

| 特点 | 说明 |
|------|------|
| 划分方式 | 静态划分，系统启动时确定 |
| 分区大小 | 可以相等或不等 |
| 碎片类型 | **内部碎片** — 作业小于分区时，分区内的剩余空间浪费 |
| 优点 | 实现简单 |
| 缺点 | 内存利用率低，大作业可能装不下 |

**内部碎片**：分配给进程的分区中未被使用的部分。

**（2）可变分区（动态分区）**

**教材定义**：不预先划分分区，而是在作业装入时，根据作业的实际需求动态划分分区。分区大小恰好等于作业大小。

| 特点 | 说明 |
|------|------|
| 划分方式 | 动态划分，按需分配 |
| 碎片类型 | **外部碎片** — 经过多次分配回收后，内存中出现大量不连续的小空闲区 |
| 优点 | 无内部碎片，内存利用率较高 |
| 缺点 | 产生外部碎片，需要紧凑操作 |

**外部碎片**：内存中有足够的空闲空间总量，但因为不连续而无法满足分配请求。

> **碎片对比**（考试常考辨析）：
> - 固定分区 → 内部碎片（分区内剩余）
> - 可变分区 → 外部碎片（分区间空隙）
> - 分页 → 内部碎片（最后一页填不满）
> - 分段 → 外部碎片（段间空隙）

**（3）可重定位分区**

**教材定义**：在可变分区的基础上，通过移动已分配的分区，使空闲分区合并为一个连续的大分区，解决外部碎片问题。也称为"紧凑"或"拼接"。

- 定时或在分配失败时执行紧凑
- 需要硬件支持（动态重定位寄存器）
- 开销较大（需要移动大量数据）

#### 3.2.2 可变分区分配算法（★★ 选择题常考）

当有多个空闲分区满足需求时，按什么策略选择？

| 算法 | 策略 | 优缺点 |
|------|------|--------|
| **首次适应法（First Fit, FF）** | 从空闲分区链首开始，找到**第一个**足够大的分区 | ✅ 简单快速；❌ 低地址碎片多，高地址大空闲区利用率低 |
| **循环首次适应法（Next Fit, NF）** | 从**上次分配的位置**继续向后查找第一个足够大的分区 | ✅ 分布均匀；❌ 大空闲区被分割 |
| **最佳适应法（Best Fit, BF）** | 找到能满足需求的**最小**空闲分区 | ✅ 保留大分区；❌ 产生大量极小外部碎片 |
| **最差适应法（Worst Fit, WF）** | 找到**最大**的空闲分区 | ✅ 减少小碎片；❌ 大空闲区很快被耗尽 |

> 🎯 **考试技巧**：题目通常给一段空闲分区表，要求用不同算法分配某个大小的作业，需要手动模拟。关键是记住：
> - 首次适应：从头找第一个够大的
> - 最佳适应：找"刚刚好"的（最小够用的）
> - 最差适应：找最大的

#### 3.2.3 分区分配计算示例

> **例题**：某系统采用可变分区分配方式，空闲分区按地址递增排列：100KB、300KB、50KB、200KB、400KB。现有一作业需要 150KB，分别用首次适应、最佳适应、最差适应算法求分配结果。

| 算法 | 选中分区 | 分配后剩余 |
|------|----------|------------|
| 首次适应 | 第 1 个 100KB → ❌不够，选第 2 个 300KB | 剩 150KB |
| 最佳适应 | 找最小够用的：200KB（第 4 个） | 剩 50KB |
| 最差适应 | 找最大的：400KB（第 5 个） | 剩 250KB |

### 3.3 分页存储管理（★★★ 超高频计算考点）

**核心概念**：

| 概念 | 含义 |
|------|------|
| **页（Page）** | 逻辑地址空间的等分单位 |
| **块/帧（Frame）** | 物理内存的等分单位 |
| **页大小 = 块大小** | 通常为 4KB |
| **页表** | 逻辑页号 → 物理块号的映射表 |

**逻辑地址结构**：
```
|  页号 (p)  |  页内偏移 (d)  |
|  高位部分  |   低位部分     |
```

- 若页大小为 4KB = 2^12 字节 → 页内偏移占 12 位
- 若逻辑地址为 32 位 → 页号占 32-12 = 20 位 → 最多 2^20 = 1M 个页

**地址转换公式**（★ 必考）：
```
物理地址 = 物理块号 × 页大小 + 页内偏移
```

**计算示例**：
> 某系统页大小 4KB，逻辑地址 0x3A7F 对应的物理块号为 6，求物理地址。
> - 页号 = 0x3A7F / 0x1000 = 3
> - 页内偏移 = 0x3A7F % 0x1000 = 0xA7F
> - 物理块号 = 6 → 物理地址 = 6 × 4KB + 0xA7F = 0x6000 + 0xA7F = 0x6A7F

**页表项内容**：

| 字段 | 含义 |
|------|------|
| **物理块号** | 对应的物理内存块 |
| **状态位（存在位）** | 该页是否在内存中 |
| **访问位** | 该页是否被访问过 |
| **修改位** | 该页是否被修改过 |

**淘汰原则**（页面置换时的淘汰顺序）：
1. 优先淘汰：不在内存的（状态位 = 0）
2. 其次淘汰：未被访问的（访问位 = 0）
3. 再次淘汰：未被修改的（修改位 = 0）—— 减少磁盘写回开销

**页面置换算法**（★★ 高频考点）：

| 算法 | 原理 | 特点 |
|------|------|------|
| **OPT（最佳置换）** | 淘汰将来最久不使用的页 | 理论最优，无法实现，用作参照 |
| **FIFO（先进先出）** | 淘汰最早进入内存的页 | 可能出现 **Belady 异常**（增加页框反而缺页增加） |
| **LRU（最近最少使用）** | 淘汰最近最久未使用的页 | 性能好，开销较大 |
| **CLOCK（时钟）** | 循环扫描，访问位=0则淘汰 | LRU 的近似实现 |

> **Belady 异常**：只有 FIFO 算法会出现增加物理块数反而增加缺页次数的现象。

**分页计算练习**：
> 逻辑地址 10156，页大小 4KB=4096字节，页表：[0→3, 1→5, 2→6, 3→2]
> - 页号 = 10156 / 4096 = 2
> - 页内偏移 = 10156 % 4096 = 1964
> - 查页表得块号 = 6
> - 物理地址 = 6 × 4096 + 1964 = 24576 + 1964 = 26540

### 3.4 分段存储管理（★★）

**核心概念**：

| 概念 | 含义 |
|------|------|
| **段** | 按程序逻辑结构划分，大小不等 |
| **段表** | 每段对应一个段表项，包含段长和基址 |

**逻辑地址结构**：
```
|  段号 (s)  |  段内偏移 (d)  |
```

**段表项内容**：

| 字段 | 含义 |
|------|------|
| **段基址** | 该段在物理内存中的起始地址 |
| **段长（段限长）** | 该段的长度 |

**地址转换**：
```
物理地址 = 段基址 + 段内偏移
```
⚠️ 必须检查：段内偏移 < 段长，否则越界中断！

**分段与分页对比**（★ 考试常考）：

| 对比项 | 分页 | 分段 |
|--------|------|------|
| 划分依据 | 物理等分，大小固定 | 逻辑划分，大小可变 |
| 用户可见性 | 对用户透明 | 用户可见 |
| 地址空间 | 一维 | 二维（段号+段内偏移） |
| 碎片 | 有内部碎片（最后一页） | 有外部碎片 |
| 信息共享 | 不方便（按页） | 方便（按段） |

### 3.5 段页式存储管理（★★★ 高频计算）

**核心思想**：
> 先将程序按逻辑结构分成若干段，再将每段分成大小相等的页。

**逻辑地址结构**：
```
|  段号 (s)  |  段内页号 (p)  |  页内偏移 (d)  |
```

**地址转换过程**：
1. 根据段号查段表 → 得到该段对应的页表起始地址
2. 根据段内页号查页表 → 得到物理块号
3. 物理地址 = 物理块号 × 页大小 + 页内偏移

**位数计算方法**（★ 必考题型）：
> 段页式系统中，逻辑地址共 n 位，页大小 2^k 字节。
> - 页内偏移：k 位
> - 剩余 (n-k) 位用来表示段号和页号
> - 具体看题目给的位分配方式

**计算示例**：
> 某段页式系统，逻辑地址 16 位，页大小 4KB=2^12（页内偏移 12 位），段号占 3 位。
> - 页号占：16 - 3 - 12 = 1 位 → 每段最多 2 页
> - 段号 3 位 → 最多 8 个段
> - 页内偏移 12 位 → 页大小 4KB

### 3.6 虚拟存储管理（★★★ 高频考点）

#### 3.6.1 基本概念

**教材定义**：
> 虚拟存储器是指具有请求调入功能和置换功能，能从逻辑上对内存容量加以扩充的一种存储器系统。其逻辑容量由内存容量和外存容量之和决定，其运行速度接近于内存，每位的成本又接近于外存。

**核心思想**：
- 将内存和外存（磁盘）结合，为用户提供比实际内存大得多的虚拟地址空间
- 程序不需要全部装入内存即可运行
- 按需调入：需要时才从外存调入
- 不需要的页可以换出到外存

**实现基础——局部性原理**（★ 考试常考）：

| 类型 | 含义 | 例子 |
|------|------|------|
| **时间局部性** | 最近被访问的指令和数据，很快会被再次访问 | 循环变量、反复调用的函数 |
| **空间局部性** | 访问某个地址后，很快会访问其附近的地址 | 顺序执行的指令、数组遍历 |

> 局部性原理是虚拟存储管理的理论基础，也是 Cache 能提高效率的原因。

#### 3.6.2 请求分页存储管理（★★★ 核心）

**与基本分页的区别**：

| 对比项 | 基本分页 | 请求分页 |
|--------|----------|----------|
| 装入策略 | 全部装入 | 按需调入（请求调页） |
| 淘汰策略 | 无 | 页面置换 |
| 页表项 | 块号 + 状态位 | 块号 + 状态位 + 访问位 + 修改位 + 外存地址 |
| 虚拟空间 | 不支持 | 支持（逻辑空间 > 物理空间） |

**请求调页过程**：
1. CPU 访问某页，先查页表
2. 状态位 = 1 → 该页在内存，正常访问
3. 状态位 = 0 → 该页不在内存，产生**缺页中断**
4. OS 处理缺页中断：从外存调入该页到内存
5. 若内存已满，执行页面置换算法淘汰某页
6. 更新页表，重新执行被中断的指令

**缺页中断**（★）：
> 当访问的页面不在内存时，硬件产生缺页中断，OS 从外存调入该页。缺页中断属于**内中断（异常）**，不是外部中断。

#### 3.6.3 页面置换算法详解

（在 3.3 节已有概述，此处补充详细计算过程）

**OPT（最佳置换）详细示例**：
> 访问序列：7, 0, 1, 2, 0, 3, 0, 4, 2, 3，物理块数 3

```
访问:  7   0   1   2   0   3   0   4   2   3
块1:   7   7   7   2   2   2   2   2   2   2
块2:   -   0   0   0   0   0   0   4   4   4
块3:   -   -   1   1   1   3   3   3   3   3
缺页:  ✓   ✓   ✓   ✓           ✓   ✓
```
- 淘汰 1 时看未来序列：7 和 0 都会被用到，1 最久不会用 → 淘汰 1
- 淘汰 0 时看未来序列：2 会用到，0 会用到，3 不会再用 → 淘汰 3
- 缺页次数：6 次

**FIFO 与 Belady 异常**：
> Belady 异常：分配的物理块数增加，缺页次数反而增加。

示例：访问序列 1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5
- 3 个物理块：缺页 9 次
- 4 个物理块：缺页 10 次 ← 反而增加了！

> 只有 FIFO 会出现 Belady 异常。LRU 和 OPT 不会出现。

**LRU 详细示例**：
> 访问序列：1, 2, 3, 4, 1, 2, 5, 1, 2, 3, 4, 5，物理块数 3

```
访问:  1   2   3   4   1   2   5   1   2   3   4   5
块1:   1   1   1   4   4   4   5   5   5   3   3   3
块2:   -   2   2   2   1   1   1   1   1   1   4   4
块3:   -   -   3   3   3   2   2   2   2   2   2   5
缺页:  ✓   ✓   ✓   ✓   ✓   ✓   ✓               ✓   ✓   ✓
```
- 第 5 次访问 1：当前块内为 {1,2,3}，1 最近刚被问但就在块里... 
- 实际过程：淘汰"最近最久未使用"的
- 缺页次数：10 次

> 🎯 **做题技巧**：
> - OPT：往后看未来使用顺序
> - FIFO：按进入时间排队
> - LRU：按最近使用时间淘汰最老的

#### 3.6.4 快表（TLB）（★）

**概念**：
> 快表（Translation Lookaside Buffer, TLB）是介于 CPU 和内存之间的高速缓存，存放最近使用的页表项副本。

**访问流程**：
1. 先查快表 → 命中（TLB Hit）→ 直接得到物理块号
2. 快表未命中（TLB Miss）→ 查内存中的页表 → 同时更新快表
3. 缺页 → 外存调入 → 更新页表和快表

**有效访问时间计算**（★ 选择题）：

```
有效访问时间 = 命中率 × (快表访问时间 + 内存访问时间)
            + (1 - 命中率) × (快表访问时间 + 内存访问时间 + 内存访问时间)
```
简化（忽略快表访问时间）：
```
有效访问时间 = 命中率 × (一次内存访问) + (1-命中率) × (两次内存访问)
```

**示例**：
> 快表访问 1ns，内存访问 100ns，命中率 80%
> 有效时间 = 0.8 × (1+100) + 0.2 × (1+100+100) = 80.8 + 40.2 = 121ns

#### 3.6.5 抖动（Thrashing）（★）

**定义**：
> 刚被换出的页很快又要被访问，导致频繁换入换出，CPU 利用率急剧下降的现象。

**产生原因**：
- 分配给进程的物理块数过少
- 进程数过多，每个进程的物理块不足

**解决方法**：
- 采用工作集模型，根据工作集大小分配物理块
- 减少多道程序度（减少同时运行的进程数）

#### 3.6.6 工作集（Working Set）（★）

**定义**：
> 工作集 W(t, Δ) 是指在时刻 t 的前 Δ 次内存访问中所访问的页面集合。

- Δ 是工作集窗口大小
- 工作集大小随时间变化
- 应为每个进程分配不小于其工作集大小的物理块数

**作用**：
- 防止抖动：分配足够物理块
- 指导页面置换：淘汰不在工作集中的页面

### 📝 本章典型真题

**【2023年真题改编 — 分页地址计算】**
> 某计算机系统的内存按字节编址，页大小为 4KB。某进程的页表如下：逻辑页 0→物理块 5，逻辑页 1→物理块 3，逻辑页 2→物理块 8。逻辑地址 0x25A0 对应的物理地址为（）。
> A. 0x85A0　　B. 0x55A0　　C. 0x35A0　　D. 0x25A0

**答案：A**
**解析**：
- 0x25A0 / 0x1000 = 2 → 页号为 2
- 0x25A0 % 0x1000 = 0x5A0 → 页内偏移
- 页号 2 对应块号 8
- 物理地址 = 8 × 0x1000 + 0x5A0 = 0x85A0

**【2022年真题改编 — 页面置换】**
> 某系统采用 LRU 页面置换算法，物理块数为 3。进程的页面访问序列为：7, 0, 1, 2, 0, 3, 0, 4。缺页次数为（）。
> A. 5　　B. 6　　C. 7　　D. 8

**答案：B**
**解析**：按 LRU 规则模拟——淘汰最近最久未使用的页：
```
访问: 7  0  1  2  0  3  0  4
块1: 7  7  7  2  2  2  2  4
块2: -  0  0  0  0  0  0  0
块3: -  -  1  1  1  3  3  3
缺页: ✓  ✓  ✓  ✓     ✓     ✓  → 6次
```

**【2021年真题改编 — 段页式位数计算】**
> 某计算机系统采用段页式存储管理，逻辑地址 32 位，段号 8 位，页大小 4KB。则每个段最多有（）页。
> A. 4096　　B. 65536　　C. 256　　D. 1048576

**答案：A**
**解析**：
- 逻辑地址 32 位，页内偏移 log₂(4KB) = log₂(4096) = 12 位
- 段号 8 位
- 页号占：32 - 8 - 12 = 12 位 → 最多 2^12 = 4096 页

## 第四章 文件管理（★★ 重要）

### 4.1 文件与文件系统

**文件的逻辑结构**：

| 类型 | 特点 |
|------|------|
| **有结构文件（记录式文件）** | 由若干记录组成 |
| **无结构文件（流式文件）** | 字节流，如文本文件 |

**文件的物理结构**（存储方式）：

| 结构 | 说明 | 特点 |
|------|------|------|
| **连续分配** | 文件占用连续的物理块 | 顺序访问快，不灵活 |
| **链接分配** | 每块有指向下一块的指针 | 消除外部碎片，只能顺序访问 |
| **索引分配** | 用索引块存放各数据块地址 | 支持随机访问 |

### 4.2 索引文件结构（★★★ 计算重点）

**多级索引地址项计算**（★ 高频计算题）：

**题目给出的典型参数**：
- 磁盘地址项大小：如 4B
- 磁盘索引块大小 = 磁盘数据块大小：如 1KB
- 直接地址索引项数：如 5 个
- 一级间接索引项数：如 2 个
- 二级间接索引项数：如 1 个

**计算方法**：

| 索引类型 | 计算方式 |
|----------|----------|
| **直接索引** | 直接项数 × 数据块大小 = 5 × 1KB = 5KB |
| **一级间接** | 每个索引块能装的地址项数 = 数据块大小 / 地址项大小 = 1KB / 4B = 256 个；一级间接范围 = 一级项数 × 256 × 数据块大小 = 2 × 256 × 1KB = 512KB |
| **二级间接** | 二级间接范围 = 二级项数 × 256 × 256 × 1KB = 1 × 65536 × 1KB = 64MB |

**文件最大长度** = 直接 + 一级间接 + 二级间接

**计算示例**：
> 地址项 4B，数据块 1KB，直接 5 项，一级 2 项，二级 1 项
> - 每块索引项数 = 1024/4 = 256
> - 直接索引范围：块 0~4（5 个块）
> - 一级索引范围：块 5~(5+2×256-1)=516（512 个块）
> - 二级索引范围：块 517~(517+1×256×256-1)=66052（65536 个块）
> - 最大文件大小 = (5 + 512 + 65536) × 1KB = 66053KB ≈ 64.5MB

### 4.3 文件目录

**文件控制块（FCB）**：
> 包含文件名、文件大小、文件类型、创建时间、物理位置等信息。文件目录就是 FCB 的有序集合。

**目录结构**：
- 单级目录：所有文件在同一目录
- 二级目录：主文件目录 + 用户文件目录
- 多级目录（树形）：现代 OS 通用

**路径**：
- 绝对路径：从根目录开始
- 相对路径：从当前目录开始

### 4.4 文件存储空间管理

**位示图法（★★★ 经典计算题）**

**原理**：用二进制位表示物理块的使用情况（0=空闲，1=已分配，或反过来）

**计算题型 1**：已知系统字长 w 位，求 N 号物理块在位示图中的字/位位置
```
字的位置 M = ⌊N / w⌋（从 0 开始编号）
位的位置 bit = N % w
```

**计算题型 2**：已知磁盘大小和块大小，求位示图需要的字数
```
物理块总数 = 磁盘大小 / 块大小
位示图字数 = 物理块总数 / 字长
```

**计算示例**：
> 系统字长 32 位，N = 16385 号物理块
> M = 16385 / 32 = 512.03 → 字位置为 512
> bit = 16385 % 32 = 1 → 第 512 号字的第 1 位

> 系统字长 32 位，块大小 4MB，磁盘 1000GB
> 块数 = 1000GB / 4MB = 1000×1024 / 4 = 256000 个
> 位示图字数 = 256000 / 32 = 8000 个

**其他管理方法**：
| 方法 | 说明 |
|------|------|
| **空闲表法** | 记录每个空闲区的起始块号和长度 |
| **空闲链表法** | 将空闲块用链表串起来 |
| **成组链接法** | UNIX 采用，空闲块分组，组间链接 |

### 📝 本章典型真题

**【2022年真题改编 — 位示图】**
> 某文件管理系统采用位示图法管理磁盘空间，字长为 32 位，物理块编号从 0 开始。假设位示图的第 0 字从高位到低位分别对应 0\~31 号物理块。第 512 号字的第 17 位（从高位数起）对应的物理块号为（）。
> A. 16385　　B. 16401　　C. 16417　　D. 16433

**答案：B**
**解析**：块号 = 字号 × 字长 + 位号 = 512 × 32 + 17 = 16384 + 17 = 16401

**【2020年真题改编 — 索引结构】**
> 某文件系统采用索引分配方式，磁盘块大小 1KB，磁盘地址占 4B。若文件索引节点中有 5 个直接地址项和 1 个一级间接地址项，则文件最大长度为（）。
> A. 5KB　　B. 261KB　　C. 256KB　　D. 2560KB

**答案：B**
**解析**：
- 直接：5 × 1KB = 5KB
- 一级间接：每个索引块 1KB/4B = 256 个地址项 → 256 × 1KB = 256KB
- 合计：5KB + 256KB = 261KB

## 第五章 设备管理（★★ 重要）

### 5.1 I/O 系统概述

**教材定义**：
> I/O 系统是负责管理和控制所有输入输出设备（包括设备、控制器、通道等）以及数据传输的系统软件。

**I/O 系统组成**：

| 组件 | 功能 |
|------|------|
| **设备** | 实际执行 I/O 操作的硬件（键盘、磁盘、打印机等） |
| **设备控制器** | 接收 CPU 命令，控制设备操作（又称适配器/接口） |
| **通道** | 专用 I/O 处理机，代替 CPU 管理 I/O |
| **总线** | 连接各部件的数据通路 |

### 5.2 设备分类

| 分类角度 | 类型 | 说明 |
|----------|------|------|
| 按组织方式 | **块设备** | 以数据块为单位传输（磁盘），可寻址 |
| | **字符设备** | 以字符为单位传输（键盘、鼠标），不可寻址 |
| 按分配角度 | **独占设备** | 同一时刻只能被一个进程使用（打印机） |
| | **共享设备** | 可被多个进程交替使用（磁盘） |
| | **虚拟设备** | 通过 Spooling 将独占设备虚拟为共享设备 |
| 按传输速率 | **低速设备** | 键盘、鼠标（几 B/s ~ 几百 B/s） |
| | **中速设备** | 行式打印机（几百 B/s ~ 几 KB/s） |
| | **高速设备** | 磁盘（几百 KB/s ~ 几 MB/s） |

### 5.3 I/O 控制方式（★★★ 必考）

**四种 I/O 控制方式**的演进：程序查询 → 中断驱动 → DMA → 通道

#### ① 程序查询方式（程序轮询方式）

**原理**：CPU 主动、不断地查询设备状态，直到设备就绪后再进行数据传送。

**工作流程**：
```
CPU 向设备发读命令
    ↓
CPU 不断查询设备状态寄存器："完成了吗？"
    ↓  （循环等待...）
设备就绪 → CPU 从设备读取数据
```

| 特点 | 说明 |
|------|------|
| CPU 参与度 | **全程参与**，轮询期间 CPU 不能做其他工作 |
| 数据传送 | 经过 CPU（CPU 从设备读，再写入内存） |
| 优点 | 实现简单，硬件需求少 |
| 缺点 | CPU 利用率极低，大量时间浪费在等待 |
| 适用场景 | 低速设备、简单系统 |

#### ② 中断驱动方式（中断方式）

**原理**：CPU 发出 I/O 命令后继续执行其他任务。设备就绪后通过**中断**通知 CPU，CPU 再处理数据传送。

**工作流程**：
```
CPU 向设备发读命令
    ↓
CPU 继续执行其他进程 ← 与程序查询的本质区别
    ↓
设备就绪 → 向 CPU 发中断信号
    ↓
CPU 响应中断 → 从设备读取一个字 → 写入内存
    ↓
若数据未传完，CPU 继续其他工作，等待下一次中断
```

| 特点 | 说明 |
|------|------|
| CPU 参与度 | 发命令和处理中断时参与，**中间可做其他事** |
| 数据传送 | 每次中断传送**一个字/字节**，仍经过 CPU |
| 优点 | CPU 利用率提高 |
| 缺点 | 每传送一个字就要中断一次，大量数据时中断频繁 |
| 适用场景 | 中低速设备 |

#### ③ DMA 方式（直接存储器访问）（★★★ 高频考点）

**教材定义**：
> DMA（Direct Memory Access）方式下，数据在内存和 I/O 设备之间直接传送，不需要 CPU 的干预。只有在数据传送的开始和结束时才需要 CPU 参与。

**工作流程**：
```
CPU 向 DMA 控制器发命令：
  - 传送方向（读/写）
  - 内存起始地址
  - 传送字节数
    ↓
CPU 继续执行其他进程
    ↓
DMA 控制器直接在内存和设备之间传送数据（不经过 CPU）
    ↓
全部传送完成 → DMA 向 CPU 发中断
    ↓
CPU 处理中断（善后）
```

| 特点 | 说明 |
|------|------|
| CPU 参与度 | **仅在开始和结束时参与**，传送过程不干预 |
| 数据传送 | 在设备和内存之间**直接传送**，不经 CPU |
| 传送单位 | **块（数据块）**，一次可传送整块数据 |
| 优点 | CPU 利用率大幅提高，适合高速批量数据传输 |
| 硬件 | 需要专用 DMA 控制器 |
| 适用场景 | 磁盘、高速网络等块设备 |

**DMA 与中断方式的对比**（★ 考试常考）：

| 对比项 | 中断方式 | DMA 方式 |
|--------|----------|----------|
| 数据是否经过 CPU | 是（CPU 参与每次传送） | 否（直接内存到设备） |
| 传送单位 | 字/字节 | 数据块 |
| 中断触发时机 | 每传送一个字中断一次 | 整块传送完成才中断 |
| CPU 干预程度 | 高 | 低 |
| 适用设备 | 中低速设备 | 高速块设备 |
| 硬件复杂度 | 较低 | 较高（需要 DMA 控制器） |

#### ④ 通道方式（I/O 处理机方式）

**教材定义**：
> 通道是一种专用的处理机（I/O 处理机），它有自己的指令系统和程序，能独立地执行通道程序来控制 I/O 操作，进一步减轻 CPU 的负担。

**工作流程**：
```
CPU 向通道发出 I/O 指令（启动通道）
    ↓
CPU 继续执行其他进程 ← 完全不管 I/O
    ↓
通道执行通道程序，独立控制数据传送
    ↓
通道程序执行完毕 → 通道向 CPU 发中断
```

| 特点 | 说明 |
|------|------|
| CPU 参与度 | **仅启动和结束时参与**，中间完全自治 |
| 数据传送 | 通道独立完成，CPU 完全不干预 |
| 硬件 | 需要专用 I/O 通道处理机 |
| 优点 | CPU 负担最轻，并行度最高 |
| 缺点 | 硬件成本最高 |
| 适用场景 | 大型机、高性能计算 |

**四种 I/O 控制方式对比总结**（★★★ 必背表格）：

| 对比项 | 程序查询 | 中断驱动 | DMA | 通道 |
|--------|----------|----------|-----|------|
| CPU 介入频率 | 持续 | 每字一次 | 开始+结束 | 开始+结束 |
| 数据是否经 CPU | 是 | 是 | **否** | **否** |
| 传送单位 | 字 | 字 | 数据块 | 数据块组 |
| 并行性 | 无 | 有 | 有 | 最强 |
| 硬件成本 | 最低 | 低 | 中 | 最高 |
| CPU 利用率 | 最低 | 较低 | 高 | 最高 |

> 🎯 **考试答题口诀**：
> - 程序查询：CPU 全程盯着 → 效率最低
> - 中断方式：CPU 每次被打断处理一个字 → 效率提高
> - DMA：数据直接走内存，CPU 只管开头结尾 → 效率高
> - 通道：专用处理机，CPU 彻底放手 → 效率最高

### 5.4 I/O 软件层次结构（★）

**从上到下**：

| 层次 | 功能 | 考试要点 |
|------|------|----------|
| **用户层 I/O 软件** | 提供库函数接口（如 printf） | 用户可见 |
| **设备独立性软件（设备无关软件）** | 统一命名、缓冲管理、设备分配与回收、差错控制 | 逻辑设备到物理设备的映射 |
| **设备驱动程序** | 将抽象 I/O 请求翻译为具体设备操作，检查设备状态 | 每类设备一个驱动 |
| **中断处理程序** | I/O 完成时唤醒驱动程序，善后处理 | 保存现场→中断处理→恢复现场 |
| **硬件** | 执行实际 I/O 操作 | 设备控制器、DMA 控制器 |

### 5.5 缓冲管理（★）

**缓冲的作用**（教材）：
1. 缓和 CPU 与 I/O 设备速度不匹配的矛盾
2. 减少对 CPU 的中断频率
3. 提高 CPU 与 I/O 设备之间的并行性

**缓冲类型**：

| 类型 | 说明 |
|------|------|
| **单缓冲** | 一个缓冲区，CPU 和外设交替使用 |
| **双缓冲（缓冲对换）** | 两个缓冲区交替使用，提高并行性 |
| **循环缓冲** | 多个缓冲区首尾相连 |
| **缓冲池** | 多个缓冲区组成公共缓冲区，统一管理 |

**单缓冲处理时间计算**（★）：
> 设从磁盘输入一块数据到缓冲区时间为 T，从缓冲区传到用户区时间为 M，CPU 处理时间为 C。
> - 单缓冲：max(T, C) + M（取较大的加上串行部分）
> - 双缓冲：max(T, C + M)（更并行）

### 5.6 Spooling 技术（假脱机）（★★ 常考）

**教材定义**：
> Spooling（Simultaneous Peripheral Operation On-Line，外围设备联机同时操作）技术是在多道程序环境下，利用一道程序来模拟脱机输入/输出中的外围控制机功能，以"预输入"和"缓输出"方式把独占设备改造为共享设备。

**工作原理**：
```
                    ┌─────────┐
多个进程 → 提交任务 → │ 输入井  │ → 输入进程 → 独占设备
                    │ (磁盘)  │
多个进程 ← 获取结果 ← │ 输出井  │ ← 输出进程 ← 独占设备
                    └─────────┘
```

**核心组成部分**：
| 组件 | 功能 |
|------|------|
| **输入井** | 磁盘上模拟的输入缓冲区，预存输入数据 |
| **输出井** | 磁盘上模拟的输出缓冲区，暂存输出数据 |
| **输入进程** | 模拟脱机输入的外围控制机 |
| **输出进程** | 模拟脱机输出的外围控制机 |

**典型应用**：共享打印机 — 多个进程的打印任务先存入磁盘缓冲区，打印机逐个处理。

**Spooling 的特点**（★ 常考）：
- 提高了 I/O 速度（利用磁盘作缓冲）
- 将独占设备改造为共享设备
- 实现了虚拟设备功能

### 5.7 磁盘调度算法（★★ 偶考计算题）

**磁盘访问时间组成**：
```
总访问时间 = 寻道时间 + 旋转延迟 + 数据传输时间
```
- **寻道时间**：磁头移动到目标磁道的时间（通常最大头）
- **旋转延迟**：目标扇区旋转到磁头下方的时间（平均 = 半圈）
- **传输时间**：读写数据的时间（通常很小）

**调度算法**：

| 算法 | 原理 | 特点 |
|------|------|------|
| **FCFS** | 按请求到达顺序服务 | 公平但平均寻道距离大 |
| **SSTF（最短寻道时间优先）** | 选离当前磁头最近的 | 平均寻道时间短，但可能导致远端饥饿 |
| **SCAN（电梯算法）** | 磁头单向移动到端点，再反向扫描 | 兼顾效率和公平 |
| **CSCAN（循环扫描）** | 磁头单向扫描，到端点直接跳回另一端起始位置 | 响应时间更均匀 |

**磁盘调度计算示例**：
> 磁头当前位置：53，磁道请求序列：98, 183, 37, 122, 14, 124, 65, 67

| 算法 | 移动顺序 | 总寻道距离 |
|------|----------|------------|
| FCFS | 53→98→183→37→122→14→124→65→67 | 640 |
| SSTF | 53→65→67→37→14→98→122→124→183 | 236 |
| SCAN（向磁道号增大方向） | 53→65→67→98→122→124→183→37→14 | 322 |

**【2022年真题改编 — I/O 控制方式】**
> 在以下四种 I/O 控制方式中，CPU 利用率最高的是（）。

**答案：D**

> B. DMA 方式的数据传送不经过 CPU，中断方式需要
> C. 中断方式适合高速设备，DMA 方式适合低速设备
> D. DMA 方式每次传送一个字节，中断方式每次传送一个数据块

**答案：B**

> D. 减少磁盘的 I/O 操作次数

## 附录 C：必背公式汇总

```
死锁最小不发生资源数 = n × (R - 1) + 1
死锁最大发生资源数   = n × (R - 1)

分页物理地址 = 块号 × 页大小 + 页内偏移
页号 = 逻辑地址 / 页大小
页内偏移 = 逻辑地址 % 页大小

每索引块地址项数 = 数据块大小 / 地址项大小

位示图字位置 = ⌊物理块号 / 字长⌋
位示图位位置 = 物理块号 % 字长
位示图字数 = (磁盘大小 / 块大小) / 字长

快表有效访问时间 = h × (t_TLB + t_mem) + (1-h) × (t_TLB + 2×t_mem)
  （h = 命中率，t_TLB = 快表访问时间，t_mem = 内存访问时间）

磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间
平均旋转延迟 = 1 / (2 × 转速)

单缓冲处理时间 = max(T, C) + M
  （T = I/O时间, C = CPU处理时间, M = 缓冲区到用户区传送时间）
双缓冲处理时间 = max(T, C + M)
```
