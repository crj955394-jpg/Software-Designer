# 第一章：栈

栈是最典型的“**受限线性表**”：元素只能在同一端进行插入和删除，这一端叫**栈顶**，另一端叫**栈底**；其核心规律是**后进先出（LIFO）**。北大与清华的讲义都把栈列为最基础、最重要的数据结构之一，并明确给出 `push`、`pop`、`top`、`isEmpty`、`getSize` 等基本操作。([math.pku.edu.cn][2])

## 1.1 栈的抽象数据类型（ADT）

栈至少应支持这些操作：创建空栈、判空、入栈 `push(x)`、出栈 `pop()`、取栈顶 `top()`。清华讲义还强调，`push` 是把对象压到栈顶，`pop` 是删除并返回栈顶元素，`top` 只是查看，不删除。([math.pku.edu.cn][2])

## 1.2 顺序栈的实现与考试最常考的条件

北大讲义给出的顺序栈实现里，`t` 记录栈顶下标，初始化为 `-1`，因此**栈空条件**是 `t == -1`；当 `t >= MAXNUM - 1` 时发生**栈满（溢出）**。入栈时先 `t++` 再赋值，出栈时直接 `t--`，这些都是软考里非常标准的代码模型。([math.pku.edu.cn][3])

顺序栈的复杂度非常重要：只要不考虑扩容，`push`、`pop`、`top` 都是 **O(1)**。这也是为什么考试中看到“栈顶操作”“回溯”“递归”“表达式求值”时，第一反应应该是栈。([math.pku.edu.cn][3])

## 1.3 链式栈的实现与考试最常考的条件

北大讲义里，链式栈的结构是“**头指针 top 指向栈顶结点**”。创建空栈时 `top = NULL`，所以**栈空条件**是 `top == NULL`；入栈时新结点 `p->link = top; top = p`；出栈时先保存栈顶，再令 `top = top->link` 并释放原结点。([math.pku.edu.cn][3])

链式栈的“满”不是逻辑满，而是**内存不足**时才会出错；也就是说它没有顺序栈那种固定的“上界满条件”，这也是软考常考的对比点。北大讲义在链式栈创建中直接用 `malloc`，失败则说明空间不足。([math.pku.edu.cn][3])

## 1.4 栈最核心的应用

栈最重要的应用有三类：**递归调用、表达式求值、回溯搜索**。北大讲义明确写到“栈与递归”“后缀表达式求值”“中缀表达式转换为后缀表达式”“迷宫问题回溯”等都是典型应用。([math.pku.edu.cn][3])

后缀表达式求值时，规则非常固定：**遇到操作数入栈，遇到运算符就连续弹出两个操作数，计算后再把结果压回栈**；如果读完后栈里不是一个元素，就说明表达式结构有错。这个规则在北大讲义里写得非常标准，软考题几乎就是围绕这套流程出。([math.pku.edu.cn][3])

## 1.5 栈的高频考点

软考最爱考的栈考点是：
1）**LIFO**；
2）**顺序栈空/满条件**；
3）**链式栈空条件**；
4）**递归为什么用栈**；
5）**表达式求值为什么用栈**；
6）**回溯为什么用栈**。这些点都能在北大与清华讲义中找到对应。([math.pku.edu.cn][2])

# 第二章：队列

队列也是一种受限线性表，但它和栈完全相反：**只允许在队尾插入，在队头删除**，因此遵循**先进先出（FIFO）**。清华讲义把队列定义为“对象排成一个序列，只能访问和取出排在最前端的对象，只能在尾部插入新对象”。([dsa.cs.tsinghua.edu.cn][4])

## 2.1 队列的抽象数据类型（ADT）

队列的基本操作是 `enqueue(x)`（入队）、`dequeue()`（出队）、`front()`（查看队头）、`isEmpty()`、`getSize()`。清华讲义强调，`front` 只读不删，`enqueue` 只在尾部插入，`dequeue` 只从头部删除。([dsa.cs.tsinghua.edu.cn][4])

## 2.2 线性顺序队列为什么不行

如果直接用普通数组表示队列，并且每次出队都把后面的元素整体前移，那么当队长为 `n` 时，每次出队要做 **O(n)** 的搬移，效率很低。清华讲义明确指出这一点，所以软考里遇到“为什么要循环队列”时，答案就是为了避免整体移动。([dsa.cs.tsinghua.edu.cn][4])

## 2.3 循环队列：软考最常考的公式

清华讲义给出了循环数组实现队列的标准思想：设 `f` 始终指向**队首元素的位置**，`r` 始终指向**队尾后一位（下一次入队的位置）**；初始时 `f = r = 0` 表示队空；每次入队 `Q[r] = obj; r = (r+1) % capacity`；每次出队取 `Q[f]` 后再 `f = (f+1) % capacity`。这套定义是考试里所有循环队列公式的基础。([dsa.cs.tsinghua.edu.cn][4])

在“**空出一个位置**”的常见约定下，可以直接推出：
**队空条件**：`front == rear`；
**队满条件**：`(rear + 1) % maxsize == front`；
**队列长度**：`(rear - front + maxsize) % maxsize`。
这三个式子是对清华讲义中 `f`、`r`、取模环形化和“实际规模不超过 N-1”的直接推导。讲义还明确说明，为避免覆盖，会把实际规模限制为 `N-1`。([dsa.cs.tsinghua.edu.cn][4])

## 2.4 链式队列

链式队列的关键是同时维护**队头指针 f** 和**队尾指针 r**。北大讲义里，空队列时 `f = r = NULL`；判空条件是 `f == NULL`；入队时把新结点接到 `r` 后面，并更新 `r`；出队时把 `f` 移到下一个结点。这样 `enqueue` 和 `dequeue` 都能做到 **O(1)**。([math.pku.edu.cn][3])

## 2.5 队列最核心的应用

队列最典型的应用是**广度优先搜索（BFS）**和**按先来后到进行服务的调度**。清华讲义明确把“农夫过河问题”的广度优先搜索与队列联系起来，还说明层次遍历本质上也使用队列。([math.pku.edu.cn][3])

## 2.6 队列高频考点

软考里队列常考：
1）**FIFO**；
2）**队空/队满条件**；
3）**循环队列为什么要留一个空位**；
4）**front / rear 的含义**；
5）**链式队列为什么要保留队尾指针**；
6）**BFS 为什么用队列**。这些都能直接在清华讲义中找到。([dsa.cs.tsinghua.edu.cn][4])

# 第三章：数组

官方考试大纲把**数组**单列为数据结构与算法知识中的基础内容，所以它不是“顺手一提”的小知识，而是必须单独吃透的章节。([m.cnitpm.com][1])

## 3.1 数组是什么

数组是由**同类元素**按顺序组织起来的复合数据对象。PKU 的数组讲义强调，数组在语言中通过**下标**访问；一维数组有一个下标，多维数组有多个下标。它还列出了数组相关的重要概念：数组元素、下标、下标运算符、越界访问、数组参数、多维数组等。([math.pku.edu.cn][5])

## 3.2 软考最重要的数组性质

数组题最常考的是三件事：**按下标直接访问、元素位置固定、适合随机访问**。PKU 的数组相关讲义给了二维数组的访问示例，像 `matrix[i][j]` 这样的访问可通过基址加偏移量的方式定位，这正是数组支持快速访问的本质。([math.pku.edu.cn][6])

同时，数组的另一个考试重点是：**插入和删除通常不如链表方便**，因为中间插入/删除往往会牵动后续元素的搬移。这个结论和线性表章节的顺序存储分析是统一的。([math.pku.edu.cn][7])

## 3.3 二维数组、多维数组和常见题型

二维数组和多维数组经常出“偏移量”“地址”“存储次序”题。PKU 课程资料明确说明：多维数组可看成多个下标共同定位一个元素；在某些语言里，元素次序遵循特定的维度展开规则，题目会明确要求你按照给定次序计算。软考做这类题时，关键不是死记某一种语言，而是**按题目规定的存储顺序和下标从何开始**来算。([math.pku.edu.cn][8])

## 3.4 数组和字符串的关系

在 C 语言里，字符串常常就是用**字符数组**来实现的，结束标志是 `'\0'`。PKU 的数组讲义和字符串讲义都明确提到这一点，这也是为什么数组知识和字符串知识会在软考中经常交叉出题。([math.pku.edu.cn][9])

## 3.5 数组高频考点

软考里数组常考：
1）**下标从几开始**；
2）**二维数组如何定位元素**；
3）**数组参数在函数中退化为指针**；
4）**数组越界**；
5）**数组和链表的取舍**；
6）**数组是字符串的重要实现基础**。这些都在 PKU 的数组讲义和数据结构课程讲义里有明确对应。([math.pku.edu.cn][10])

# 第四章：字符串

官方大纲把“**字符串处理**”直接列入数据结构与算法知识，所以字符串绝不是边角料。([m.cnitpm.com][1])

## 4.1 字符串的定义

PKU 讲义对字符串的定义非常标准：**字符串简称串，是特殊的线性表，元素是字符**。它还明确给出：长度为零的串叫空串；子串是主串中连续字符组成的串；串可以按字典序比较。([math.pku.edu.cn][11])

## 4.2 字符串的基本操作

字符串的基本操作包括：创建空串、判空、求长度、拼接、求子串、查找模式串首次出现的位置、按字典序比较、替换、删除子串、清空等。PKU 字符串讲义把这些操作一条条列出来了，这些就是软考的“基础动作表”。([math.pku.edu.cn][11])

## 4.3 字符串的实现方式

字符串既可以顺序表示，也可以链式表示；在 C 中，最常见的是**字符数组 + 结束符 `'\0'`** 的方式。PKU 讲义明确说，许多语言和库都支持动态字符串；而 C 语言标准库 `string.h`、C++ `<string>` 等也都围绕字符串提供了大量常用操作。([math.pku.edu.cn][12])

## 4.4 模式匹配与 KMP

字符串题里最常考的是**模式匹配**，尤其是 **KMP**。PKU 讲义明确把“朴素匹配”和“无回溯的模式匹配（KMP）”并列讲解，并说明 KMP 的关键在于：预先分析模式串，记录前缀与后缀的匹配信息，匹配失败时不回头扫主串，而是依据 `next` 表移动模式串。([math.pku.edu.cn][12])

KMP 的 `next` 数组核心思想是：对模式串的每个位置，记录“**最长相同前后缀**”的长度或对应跳转位置；当发生失配时，模式串按 `next` 值跳转。PKU 讲义还给出了改进版 `makeNext` 算法，并明确说明其时间复杂度是线性的，不是表面上看起来的二重循环平方复杂度。([math.pku.edu.cn][12])

## 4.5 字符串高频考点

软考中字符串最常考：
1）**串是特殊线性表**；
2）**子串、主串、字典序**；
3）**C 字符串以 `'\0'` 结束**；
4）**KMP 的 next 数组**；
5）**模式匹配不回溯**。这些都在 PKU 讲义里有对应说明。([math.pku.edu.cn][11])

# 第五章：题目（例题、真题、解析）

下面我把你最需要的考试题型单独列出来，按“题目—答案—为什么”组织。题目来源分为两类：一类是**题库/每日一练**，一类是**真题/回忆版**；我会明确标注来源，避免混淆。([cnitpm.com][13])

## 5.1 栈题

**题 1：顺序栈的空条件是什么？满条件是什么？**
答案：空条件通常是 `t == -1`，满条件通常是 `t >= MAXNUM - 1`。这是北大讲义中顺序栈的标准实现。([math.pku.edu.cn][3])

**题 2：为什么递归调用要用栈？**
答案：因为递归满足“后调用先返回”，要保存返回地址、局部变量、实参等信息，天然符合栈的 LIFO 规律。北大讲义把“栈与递归”列为栈的核心应用。([math.pku.edu.cn][3])

**题 3：后缀表达式求值为什么必须用栈？**
答案：因为后缀表达式读到操作数就入栈，读到运算符就弹出两个操作数计算，再把结果入栈。北大讲义明确给出这套算法框架。([math.pku.edu.cn][3])

**题 4（题库/每日一练）：火车调度能否从 B 方向驶出指定序列，使用什么数据结构？**
答案：用栈。信管网每日一练把“车厢进站后不能再返回、出站后不能再回来”的过程直接对应为栈结构。([cnitpm.com][13])

## 5.2 队列题

**题 5：循环队列为什么要留一个空位？**
答案：为了区分队空和队满，避免 `front == rear` 时无法判断到底是“空”还是“满”。清华讲义说明，为避免覆盖，会把队列实际规模限制为 `N-1`。([dsa.cs.tsinghua.edu.cn][4])

**题 6：循环队列的长度公式是什么？**
答案：在常见约定下，长度可写为 `(rear - front + maxsize) % maxsize`。这是由 `front` 指向队首、`rear` 指向队尾后一位，并通过取模回绕的定义直接推出的。([dsa.cs.tsinghua.edu.cn][4])

**题 7：队列为什么适合广度优先搜索？**
答案：因为 BFS 需要“先进入的状态先展开”，这正是 FIFO；清华讲义明确把层次遍历和农夫过河的广度优先搜索与队列联系起来。([dsa.cs.tsinghua.edu.cn][14])

## 5.3 数组题

**题 8：数组和链表，谁更适合按下标快速访问？**
答案：数组。PKU 数组讲义和数据结构讲义都体现了数组可通过下标和偏移快速定位元素，而链表则需要顺链查找。([math.pku.edu.cn][6])

**题 9：二维数组 `matrix[i][j]` 为什么能通过偏移量直接定位？**
答案：因为二维数组在内存中的展开可以用基址加偏移来表达，PKU 课件里给出了 `&matrix[0][0] + i*n + j` 这样的访问思路。([math.pku.edu.cn][6])

**题 10（真题/回忆题常见风格）：数组、链表、栈、队列混在一起时，如何快速判断？**
答案：看到“下标定位、连续访问、随机访问”优先想数组；看到“只在一端操作”想栈；看到“队头删、队尾插”想队列；看到“指针串起来、插删方便”想链表。这个判断链和官方大纲列出的知识点完全一致。([m.cnitpm.com][1])

## 5.4 字符串题

**题 11：字符串在数据结构里是什么？**
答案：字符串是特殊的线性表，元素是字符。PKU 字符串讲义这样定义。([math.pku.edu.cn][11])

**题 12：KMP 为什么比朴素匹配快？**
答案：因为 KMP 利用 `next` 数组记录前后缀关系，失配时模式串跳转而主串不回头，从而避免重复比较。PKU 讲义明确把这点写成“无回溯的模式匹配”。([math.pku.edu.cn][12])

**题 13（题库/每日一练）：模式串 `abaac` 的 next 值是什么？**
答案：信管网题库给出的参考答案是 **B：01122**。这类题的本质就是按 `next` / 最长相同前后缀规则逐位推。([cnitpm.com][15])

**题 14（题库/每日一练）：模式串 `aaabaaa` 的 next 值是什么？**
答案：题库页面给出的标准值是 **0123123**。这也是软考常见 KMP 题型。([cnitpm.com][15])

### 1. FIFO
- 全称：**First In First Out，先进先出**，是队列结构的核心本质特性，也是队列区别于栈的核心标志。
- 官方定义：队列是一种只允许在表的一端（队尾rear）进行插入操作（入队），在另一端（队头front）进行删除操作（出队）的线性表。最先进入队列的元素，最先被取出；最后进入的元素，最后被取出，完全遵循FIFO规则。
- 软考考点：常和栈的LIFO（后进先出）做对比考查，区分二者的核心特性与适用场景。

---

### 2. 队空/队满条件
以下严格遵循官方教程的默认约定：循环队列的队头指针`front`指向队头元素的位置；队尾指针`rear`指向队尾元素的下一个空闲位置；队列最大容量为`MaxSize`。

#### （1）核心考点：预留1个空位的循环队列（考试默认场景，90%考题考查此类型）
- 队空条件：**`front == rear`**
  解释：队头指针和队尾指针重合，说明队列中无有效元素。
- 队满条件：**`(rear + 1) % MaxSize == front`**
  解释：队尾指针的下一个位置就是队头指针，说明队列仅剩预留的1个空位，无法再入队，判定为队满。

#### （2）次要考点：带计数器/标志位的循环队列
- 增设`size`计数器记录元素个数：
  队空：`size == 0`；队满：`size == MaxSize`
  此时`front == rear`既可以是队空也可以是队满，通过`size`区分，无需预留空位。
- 增设`tag`标志位：
  队空：`front == rear 且 tag == 0`；队满：`front == rear 且 tag == 1`

#### （3）链式队列（无固定队满限制）
- 队空条件：带头结点的链式队列：**`front->next == NULL`**；不带头结点：**`front == NULL 且 rear == NULL`**
- 队满条件：链式队列无固定队满上限，只要内存足够可无限入队，仅内存不足时会入队失败，考试中一般不考查。

---

### 3. 循环队列为什么要留一个空位
这是软考高频选择题考点，官方教程明确的核心原因是**解决「队空」和「队满」的判断歧义问题**，具体拆解：
1.  **问题根源**：循环队列是用顺序数组实现的环形结构，若不留空位，当队列填满整个数组时，队尾指针`rear`会绕回队头，此时会出现`front == rear`；而队空时，同样是`front == rear`。程序无法通过这个条件区分队列是空还是满，出现判断歧义。
2.  **解决方案**：强制预留1个不存储有效元素的空闲空位，约定「当队尾指针的下一个位置就是队头指针时，判定为队满」。此时队满条件和队空条件完全区分开，无需额外的计数器/标志位，实现逻辑最简单、开销最小，也是考试和工业界最通用的实现方式。
3.  **补充考点**：预留1个空位后，循环队列的最大有效元素个数为`MaxSize - 1`，是软考计算题的常考点。

---

### 4. front / rear 的含义
以下是**软考官方教程和历年真题统一的默认约定**（绝对不能混淆，否则计算题必错）：
- **`front`：队头指针**
  核心含义：指向队列中**第一个有效元素（队头元素）的存储位置**。
  操作特性：出队操作时，`front`向后移动（循环队列中为`front = (front+1)%MaxSize`），指向新的队头元素。
- **`rear`：队尾指针**
  核心含义：指向队列中**最后一个有效元素的下一个空闲存储位置**（划重点：不是指向队尾元素本身，是下一个空位！这是考试的核心约定）。
  操作特性：入队操作时，先将元素存入`rear`指向的位置，再将`rear`向后移动（循环队列中为`rear = (rear+1)%MaxSize`）。

### 5. 链式队列为什么要保留队尾指针
这个考点完全承接单链表的核心特性，官方教程的标准解释如下：
1.  链式队列是基于单链表实现的队列，入队操作只能在队尾（链表尾部）执行，出队操作在队头（链表头部）执行。
2.  若不保留队尾指针：每次入队都必须从队头指针开始，遍历整个链表才能找到尾节点，时间复杂度为**O(n)**，队列元素越多，效率越低。
3.  保留队尾指针后：可直接通过队尾指针定位到尾节点，无需遍历链表，入队操作的时间复杂度直接降为**O(1)**，和出队操作的时间复杂度一致，实现了队列入队、出队双端的常数级操作，是链式队列的标准实现方式。
4.  补充考点：带头结点的链式队列保留队尾指针后，空队列、单元素队列的入队/出队逻辑可完全统一，无需额外的边界判断，代码更简洁、出错率更低。

---

### 6. BFS 为什么用队列
这是软考图的遍历章节的核心考点，官方教程明确规定BFS（广度优先搜索/宽度优先搜索）的实现必须使用队列，核心原因是**队列的FIFO特性与BFS的遍历逻辑完全匹配**：
1.  **BFS的核心逻辑**：按「层次」遍历图/树，先访问当前层的所有节点，再依次访问下一层的节点；且**先被访问的节点，它的邻接节点也要先被访问**。
2.  **队列的完美适配**：
    - 遍历起始节点时，将起始节点入队；
    - 每次从队头取出一个节点，访问该节点，再将它所有未被访问的邻接节点按顺序从队尾入队；
    - 重复操作直到队列为空。
    这个过程中，先入队的节点（先被访问的上层节点）会先被取出，它的邻接节点也会先入队、先被访问，严格保证了按层次遍历的顺序，完全符合BFS的规则。
3.  **反向验证**：如果用栈（LIFO）实现，会变成先访问后入队的节点，最终成为深度优先遍历（DFS），无法实现按层次的广度优先遍历。
