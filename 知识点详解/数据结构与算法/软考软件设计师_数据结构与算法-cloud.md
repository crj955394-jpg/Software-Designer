# 第一章：算法基础与复杂度分析

## 1.1 算法的基本概念 ★★★

### 定义

算法（Algorithm）是对特定问题**求解步骤的有限描述**，是指令的有限序列。

> **程序 vs 算法**：程序是算法的代码实现，但程序可以不满足有穷性（如操作系统永远运行），算法必须有穷。这是考试常考的区别点。

### 算法的五个基本特性（必背）

| 特性 | 含义 | 记忆关键词 |
|------|------|-----------|
| **有穷性** | 算法必须在有限步骤后终止 | 不能死循环 |
| **确定性** | 每条指令含义唯一，无歧义 | 步骤明确 |
| **可行性** | 每个操作都能通过基本运算完成 | 可以执行 |
| **输入** | 有**零个或多个**输入 | 可以没有输入 |
| **输出** | 有**一个或多个**输出 | 必须有输出 |

> ⚠️ **常考陷阱**：输入可以为零个，但输出**至少一个**。

---

## 1.2 时间复杂度 ★★★★★

### 大 O 表示法

用 **O(f(n))** 表示算法运行时间随输入规模 n 增长的**上界**趋势，忽略低阶项和常数系数。

**规则**：
- 只保留最高阶项，例如 `3n² + 2n + 1` → `O(n²)`
- 忽略系数，例如 `5n` → `O(n)`

### 常见复杂度从低到高排列（必背）

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

| 复杂度 | 名称 | 典型场景 |
|--------|------|---------|
| O(1) | 常数阶 | 数组按下标访问、入栈、出栈 |
| O(log n) | 对数阶 | 二分查找、平衡二叉树操作 |
| O(n) | 线性阶 | 线性查找、遍历链表 |
| O(n log n) | 线性对数阶 | 快速排序（平均）、归并排序、堆排序 |
| O(n²) | 平方阶 | 冒泡排序、插入排序、选择排序 |
| O(2ⁿ) | 指数阶 | 朴素递归斐波那契、枚举子集 |

### 计算方法详解

**规则一：单层循环，看循环次数**

```c
for(int i = 0; i < n; i++)   // 执行 n 次
    x++;
// 时间复杂度：O(n)
```

**规则二：嵌套循环，相乘**

```c
for(int i = 0; i < n; i++)      // 外层 n 次
    for(int j = 0; j < n; j++)  // 内层每次 n 次
        x++;
// 总执行次数：n × n = n²
// 时间复杂度：O(n²)
```

**规则三：循环变量乘法增长，取对数**

```c
for(int i = 1; i < n; i = i * 2)  // i: 1, 2, 4, 8, ..., 2^k
    x++;
// 设循环执行 k 次，则 2^k < n，即 k < log₂n
// 时间复杂度：O(log n)
```

**规则四：多段独立代码，取最大值**

```c
for(int i = 0; i < n; i++) x++;        // O(n)
for(int i = 0; i < n; i++)             // O(n²)
    for(int j = 0; j < n; j++) x++;
// 总复杂度：max(O(n), O(n²)) = O(n²)
```

**规则五：三角形循环（常考！）**

```c
for(int i = 0; i < n; i++)
    for(int j = i; j < n; j++)   // 注意：j 从 i 开始，不是从 0
        x++;
```

计算执行次数：
- i=0 时，j 执行 n 次
- i=1 时，j 执行 n-1 次
- ...
- i=n-1 时，j 执行 1 次
- 总次数 = n + (n-1) + ... + 1 = **n(n+1)/2**

取最高阶项，忽略系数：**O(n²)**

## 1.3 空间复杂度 ★★★

**定义**：算法执行过程中所需**额外存储空间**的度量。

| 情况 | 空间复杂度 | 例子 |
|------|-----------|------|
| 只用几个变量 | O(1) | 冒泡排序、堆排序 |
| 额外开一个大小为 n 的数组 | O(n) | 归并排序的辅助数组 |
| 递归深度为 log n | O(log n) | 快速排序（平均递归深度） |
| 递归深度为 n | O(n) | 快速排序最坏情况 |

> ⚠️ **考点**：递归算法的空间复杂度 = 递归调用栈的最大深度。

## 第一章练习题

### 练习 1（基础）

下列算法的时间复杂度是（　　）

```c
int sum = 0;
for(int i = 1; i <= n; i++)
    for(int j = 1; j <= i; j++)
        sum++;
```

A. O(n)　　B. O(n log n)　　C. O(n²)　　D. O(n³)

**答案：C**

**解析**：
- i=1 时，j 执行 1 次
- i=2 时，j 执行 2 次
- ...
- i=n 时，j 执行 n 次
- 总次数 = 1 + 2 + ... + n = n(n+1)/2
- 忽略低阶项和系数，时间复杂度为 **O(n²)**

---

### 练习 2（基础）

以下关于算法特性的描述，**错误**的是（　　）

A. 算法必须有输出
B. 算法必须在有限步内终止
C. 算法必须有输入
D. 算法的每一步必须是确定的

**答案：C**

**解析**：算法的输入可以为**零个**（如求圆周率 π 的算法不需要输入），但必须有至少一个输出。选项 C 说"必须有输入"是错误的。

---

### 练习 3（中等）

以下算法的时间复杂度是（　　）

```c
int fib(int n) {
    if(n <= 1) return n;
    return fib(n-1) + fib(n-2);
}
```

A. O(n)　　B. O(n log n)　　C. O(n²)　　D. O(2ⁿ)

**答案：D**

**解析**：
- 每次调用 fib(n) 会产生 fib(n-1) 和 fib(n-2) 两个子调用
- 展开成递归树，树的节点数约为 2ⁿ
- 大量子问题被**重复计算**（如 fib(3) 会被计算多次）
- 时间复杂度为 **O(2ⁿ)**

> 改进：使用动态规划（记忆化）可优化为 O(n)。

---

### 真题解析 1（2019 年上半年）

一个算法所需的存储空间为 f(n)，若 f(n) = n²+2n+3，则该算法的空间复杂度为（　　）

A. O(n)　　B. O(n²)　　C. O(2n)　　D. O(3)

**答案：B**

**解析**：f(n) = n² + 2n + 3，保留最高阶项 n²，忽略系数，空间复杂度为 **O(n²)**。

# 第二章：线性表

## 2.1 线性表的基本概念 ★★★

**定义**：n（n≥0）个**相同类型**数据元素的**有限序列**。

**特点**：
- 有且仅有一个**首元素**（没有前驱）
- 有且仅有一个**尾元素**（没有后继）
- 除首尾外，每个元素有且仅有一个**直接前驱**和一个**直接后继**
- 元素个数 n 为线性表的**长度**，n=0 时称为**空表**

线性表有两种存储方式：**顺序存储**（顺序表）和**链式存储**（链表）。

---

## 2.2 顺序表 ★★★

### 定义

用一组**连续的存储单元**依次存放线性表中的元素，逻辑上相邻的元素，物理位置也相邻。

### 地址计算公式（必考）

设：
- 首元素 a₁ 的存储地址为 `LOC(a₁)`
- 每个元素占 `d` 个字节

则第 `i` 个元素的地址为：

```
LOC(aᵢ) = LOC(a₁) + (i - 1) × d
```

**例题**：顺序表首地址为 1000，每个整型元素占 4 字节，求第 8 个元素的地址。

```
LOC(a₈) = 1000 + (8 - 1) × 4 = 1000 + 28 = 1028
```

**答案：1028**

> ⚠️ 注意：下标从 **1** 开始时用 `(i-1)×d`；若题目说从 **0** 开始，则直接用 `i×d`。

### 插入与删除操作（必考）

**插入操作**：在第 i 个位置插入新元素，需要将第 i～n 个元素**依次向后移动一位**。

- 移动次数 = n - i + 1
- 最好情况（插入末尾 i=n+1）：移动 **0** 次
- 最坏情况（插入开头 i=1）：移动 **n** 次
- 平均移动次数 = **n/2** 次 → 时间复杂度 **O(n)**

**删除操作**：删除第 i 个元素，需要将第 i+1～n 个元素**依次向前移动一位**。

- 移动次数 = n - i
- 最好情况（删除末尾 i=n）：移动 **0** 次
- 最坏情况（删除开头 i=1）：移动 **n-1** 次
- 平均移动次数 = **(n-1)/2** 次 → 时间复杂度 **O(n)**

### 顺序表操作复杂度汇总

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 按下标访问 | O(1) | 直接计算地址 |
| 按值查找 | O(n) | 需要逐个比较 |
| 插入元素 | O(n) | 需要移动元素 |
| 删除元素 | O(n) | 需要移动元素 |

---

## 2.3 链表 ★★★

链表中每个**节点**包含两部分：
- **数据域**：存储数据
- **指针域（next）**：存储下一个节点的地址

### 单链表

```
[头节点] → [a₁|next] → [a₂|next] → [a₃|next] → NULL
```

**带头节点的好处**：
- 对链表的第一个数据节点和其他节点操作统一，不需要特殊处理
- 空表时头节点的 next 指向 NULL

**单链表操作复杂度**：

| 操作 | 时间复杂度 | 说明 |
|------|-----------|------|
| 按位置访问第 i 个 | O(n) | 只能从头遍历 |
| 头部插入/删除 | O(1) | 只改指针 |
| 尾部插入（无尾指针） | O(n) | 需先找到尾节点 |
| 已知节点 p 后插入 | O(1) | 只改指针 |
| 已知节点 p 的删除（单链表） | O(n) | 需找 p 的前驱 |

> ⚠️ **考点**：在已知节点 p **后**插入：O(1)；要**删除节点 p 本身**（单链表）：O(n)，因为需要找到 p 的前驱节点。

### 双链表

每个节点有两个指针：`prev`（前驱）和 `next`（后继）。

```
NULL ← [prev|a₁|next] ↔ [prev|a₂|next] ↔ [prev|a₃|next] → NULL
```

**优点**：删除已知节点 p 时，可直接修改 p.prev 和 p.next，时间复杂度 **O(1)**（单链表需 O(n)）。

### 循环链表

- **循环单链表**：尾节点的 next 指向头节点（而非 NULL）
- **循环双链表**：头节点的 prev 指向尾节点，尾节点的 next 指向头节点
- **优点**：从任意节点出发可遍历整个链表

### 顺序表 vs 链表（必考对比）

| 特性 | 顺序表 | 链表 |
|------|--------|------|
| 存储方式 | 连续内存 | 非连续内存 |
| 随机访问 | **O(1)** | O(n) |
| 插入/删除 | O(n)（移动元素） | **O(1)**（已知位置，改指针）|
| 存储密度 | **高**（只存数据） | 低（有指针开销）|
| 查找操作 | O(n) | O(n) |
| 适用场景 | 频繁查询、少修改 | 频繁插入删除 |

## 第二章练习题

### 练习 1（真题改编）

在一个长度为 n 的顺序表中，在**第 i 个位置**插入一个新元素，需要向后移动（　　）个元素。

A. n-i　　B. n-i+1　　C. i-1　　D. i

**答案：B**

**解析**：
- 顺序表有 n 个元素，位置编号为 1 到 n
- 在第 i 个位置插入，原来第 i、i+1、...、n 个元素都要向后移动一格
- 移动元素数量 = n - i + 1

验证：若 n=5，在第 3 个位置插入，需移动第 3、4、5 个元素，共 5-3+1=**3** 个。✓

---

### 练习 2（基础）

以下关于链表的描述，正确的是（　　）

A. 链表可以随机访问任意位置的元素
B. 在已知节点 p 后插入新节点，时间复杂度为 O(1)
C. 链表比顺序表占用更少的存储空间
D. 链表删除头节点比顺序表删除头节点更慢

**答案：B**

**解析**：
- A 错：链表不支持随机访问，必须从头遍历，O(n)
- B 正确：只需修改两个指针，O(1)
- C 错：链表每个节点需要额外存储指针，存储密度低于顺序表
- D 错：链表删除头节点只需修改头指针，O(1)；顺序表删除头元素需移动所有元素，O(n)

---

### 练习 3（真题风格）

设单链表共有 n 个节点，以下操作中时间复杂度为 O(1) 的是（　　）

A. 查找第 i 个节点（1 ≤ i ≤ n）
B. 删除第 i 个节点（1 ≤ i ≤ n）
C. 在已知节点 p 之后插入新节点
D. 在已知节点 p 之前插入新节点（单链表）

**答案：C**

**解析**：
- A：需从头遍历，O(n)
- B：需先找到第 i-1 个节点，O(n)
- C：只需 `new.next = p.next; p.next = new`，两步操作，**O(1)** ✓
- D：需先找到 p 的前驱节点（单链表无法直接找前驱），O(n)；若是双链表则 O(1)

### 真题解析 2（2020 年上半年）

以下关于顺序存储结构和链式存储结构的叙述，正确的是（　　）

A. 顺序存储结构的存储密度小于链式存储结构
B. 顺序存储结构比链式存储结构更适合进行插入和删除操作
C. 链式存储结构比顺序存储结构更适合进行随机存取操作
D. 顺序存储结构比链式存储结构更适合进行随机存取操作

**答案：D**

**解析**：
- A 错：顺序存储只存数据，密度**大**；链式存储还需存指针，密度小
- B 错：链式结构更适合插入删除（O(1)），顺序结构插入删除需移动元素（O(n)）
- C 错：链式结构不支持随机存取，需从头遍历
- D 正确：顺序存储按下标直接计算地址，支持 O(1) 随机存取

# 第三章：栈与队列

## 3.1 栈（Stack）★★★★

### 定义与特点

**栈**是一种**后进先出（LIFO, Last In First Out）**的线性结构。

```
        ↑ 出栈（pop）
  ┌─────┐
  │  e  │  ← 栈顶（top）
  ├─────┤
  │  d  │
  ├─────┤
  │  c  │
  ├─────┤
  │  b  │
  ├─────┤
  │  a  │  ← 栈底（bottom）
  └─────┘
        ↓ 入栈（push）
```

只有**栈顶**可以进行操作：
- **入栈（push）**：将元素压入栈顶
- **出栈（pop）**：将栈顶元素弹出

### 栈的基本操作复杂度

| 操作 | 时间复杂度 |
|------|-----------|
| 入栈 push | O(1) |
| 出栈 pop | O(1) |
| 取栈顶 peek | O(1) |

### 栈的应用（考试常考）

| 应用场景 | 说明 |
|---------|------|
| 表达式求值 | 中缀转后缀、后缀表达式计算 |
| 括号匹配 | 检查括号是否正确配对 |
| 递归实现 | 函数调用使用系统栈 |
| 深度优先搜索（DFS） | 用栈模拟递归 |
| 进制转换 | 余数入栈，出栈得结果 |

### 中缀表达式转后缀表达式（必考）

**规则**：
1. 从左到右扫描中缀表达式
2. 遇到**操作数**：直接输出
3. 遇到**左括号 (`(`)**：压栈
4. 遇到**右括号 (`)`)**：弹出栈顶元素并输出，直到遇到左括号（左括号出栈但不输出）
5. 遇到**运算符**：
   - 若栈为空或栈顶为左括号，直接压栈
   - 若当前运算符优先级**大于**栈顶运算符，压栈
   - 否则，弹出栈顶运算符输出，重复比较，直到可以压栈

**运算符优先级**：`*` `/` > `+` `-` > `(`

**例题**：将 `A + B * C - D` 转换为后缀表达式

| 步骤 | 读入 | 栈状态 | 输出 |
|------|------|--------|------|
| 1 | A | 空 | A |
| 2 | + | + | A |
| 3 | B | + | A B |
| 4 | * | + * | A B |
| 5 | C | + * | A B C |
| 6 | - | 弹出 *、+，压入 - | A B C * + |
| 7 | D | - | A B C * + D |
| 8 | 结束 | 弹出 - | A B C * + D - |

**结果**：`A B C * + D -`

**验证**：`A + B*C - D` = `A + (B*C) - D`
后缀计算：读 A、B、C，遇 * 计算 B*C，结果压栈，遇 + 计算 A+(B*C)，读 D，遇 - 计算结果-D。正确 ✓

### 顺序栈的溢出问题

- **上溢（overflow）**：栈已满时还执行 push（存储空间不够）
- **下溢（underflow）**：栈已空时还执行 pop（对空栈出栈）

---

## 3.2 队列（Queue）★★★

### 定义与特点

**队列**是一种**先进先出（FIFO, First In First Out）**的线性结构。

```
出队（dequeue）← [a][b][c][d][e] ← 入队（enqueue）
                  ↑                   ↑
                 队头(front)         队尾(rear)
```

只允许在**队尾入队**，在**队头出队**。

### 循环队列（必考重点）

**问题**：顺序队列在频繁入队出队后，虽然队列中元素不多，但 rear 已到数组末尾，出现"假溢出"。

**解决方案**：循环队列——将数组首尾相接，形成环形结构。

设数组大小为 `MAXSIZE`：
- 入队：`rear = (rear + 1) % MAXSIZE`
- 出队：`front = (front + 1) % MAXSIZE`

**判断循环队列空/满（必背）**：

| 状态 | 判断条件 | 说明 |
|------|---------|------|
| 队空 | `front == rear` | 队头等于队尾 |
| 队满 | `(rear + 1) % MAXSIZE == front` | 牺牲一个存储位置来区分空和满 |

**队列中元素个数**：`(rear - front + MAXSIZE) % MAXSIZE`

> ⚠️ **为什么队满条件不是 `rear == front`？** 因为这与队空条件相同，无法区分。所以规定队满时还剩一个空位置不用，即牺牲一个存储单元。

**例题**：循环队列大小 MAXSIZE=6，front=2，rear=5，队列中有几个元素？

`(rear - front + MAXSIZE) % MAXSIZE = (5 - 2 + 6) % 6 = 9 % 6 = 3`

**答案：3 个元素**

### 队列的应用

| 应用场景 | 说明 |
|---------|------|
| 广度优先搜索（BFS） | 按层级遍历 |
| 进程调度 | 操作系统中的就绪队列 |
| 打印机缓冲 | 先来先服务 |
| 消息队列 | 系统间异步通信 |

# 第四章：串与模式匹配

## 4.1 串的基本概念 ★★

**串（String）**：由零个或多个字符组成的有限序列。

- **空串**：长度为 0 的串，记为 `""`
- **空格串**：包含一个或多个空格的串（不是空串）
- **子串**：串中任意个连续字符组成的子序列（包含空串和本身）
- **主串**：包含子串的完整串
- **字符位置**：字符在串中的序号（从 1 开始）
- **子串位置**：子串在主串中**第一个字符**的位置

**例题**：串 `S = "abcabc"`，子串 `T = "cab"`
- T 在 S 中的位置为 **3**（T 的第一个字符 'c' 在 S 中的下标为 3）

---

## 4.2 KMP 模式匹配算法 ★★★

### 朴素模式匹配（暴力算法）

逐个位置尝试匹配，时间复杂度 **O(m×n)**（m 为主串长度，n 为模式串长度）。

### KMP 算法

KMP 算法利用**已匹配信息**避免重复比较，时间复杂度 **O(m+n)**。

核心是 **next 数组**（失配函数），表示当在模式串第 j 位失配时，模式串应移动到哪个位置重新开始匹配。

### next 数组的计算规则

`next[j]` = 模式串 T 中，**前缀和后缀**中最长相等串的长度（不包括整个串本身）。

**定义**：
- `next[1] = 0`（第 1 位失配，模式串整体右移）
- `next[j]` = 模式串 `T[1..j-1]` 中最长相等**真前缀**和**真后缀**的长度

**例题**：求模式串 `T = "abaab"` 的 next 数组

| j | T[j] | T[1..j-1] | 最长相等前后缀 | next[j] |
|---|------|-----------|--------------|---------|
| 1 | a | （空）| 0 | 0 |
| 2 | b | "a" | 无 | 1 |
| 3 | a | "ab" | 无 | 1 |
| 4 | a | "aba" | "a" 长度 1 | 2 |
| 5 | b | "abaa" | "a" 长度 1 | 2 |

> next 数组：[0, 1, 1, 2, 2]

> ⚠️ **考试提示**：KMP 一般只考 next 数组的填写，不常考完整算法代码。重点掌握 next 数组的含义和计算规则。

# 第五章：数组与矩阵压缩

## 5.1 数组的存储 ★★

### 行优先与列优先

二维数组 A[m][n]（m 行 n 列，下标从 0 开始）：

**行优先（C 语言、大多数语言）**：

```
A[i][j] 的地址 = 基地址 + (i × n + j) × d
```

**列优先（FORTRAN 语言）**：

```
A[i][j] 的地址 = 基地址 + (j × m + i) × d
```

其中 d 为每个元素所占字节数。

**例题**：数组 A[0..4][0..5]（5行6列），首地址 1000，每元素 2 字节，行优先，求 A[2][3] 的地址。

```
地址 = 1000 + (2 × 6 + 3) × 2 = 1000 + 15 × 2 = 1000 + 30 = 1030
```

**答案：1030**

---

## 5.2 特殊矩阵的压缩存储 ★★★

### 对称矩阵

n×n 对称矩阵（`a[i][j] = a[j][i]`）只需存储下三角（含对角线），元素个数为 `n(n+1)/2`。

存储下三角元素 `a[i][j]`（i ≥ j，下标从 1 开始）在一维数组中的位置：

```
k = i(i-1)/2 + j      （其中 i ≥ j，下标从 1 开始）
若 i < j，则 a[i][j] = a[j][i]，用 a[j][i] 的公式计算
```

### 稀疏矩阵

矩阵中**非零元素很少**（通常非零元素个数远小于矩阵总元素个数）。

**压缩存储方式**：三元组表 `(row, col, value)`，只存储非零元素的行号、列号和值。

# 第六章：树与二叉树

> **本章是整个数据结构模块的核心，历年必考，占 3～5 分。重点掌握二叉树性质、遍历、哈夫曼树。**

## 6.1 树的基本概念 ★★★

### 基本术语（必背）

| 术语 | 含义 |
|------|------|
| **节点的度** | 该节点拥有的**子树数量** |
| **树的度** | 树中所有节点度的**最大值** |
| **叶节点（终端节点）** | 度为 0 的节点（无子节点） |
| **分支节点（内部节点）** | 度大于 0 的节点 |
| **节点的层次** | 根节点为第 1 层，向下递增 |
| **树的高度/深度** | 树中所有节点层次的最大值 |
| **路径** | 从一个节点到另一个节点所经过的节点序列 |
| **祖先** | 从根到某节点路径上的所有节点 |
| **子孙** | 某节点的子树中的所有节点 |
| **森林** | m（m≥0）棵互不相交的树的集合 |

**重要关系**：树中的**节点数 = 总度数 + 1**（每条边贡献一个度，根节点没有入边）

---

## 6.2 二叉树的定义与性质 ★★★★★

### 定义

二叉树是每个节点**最多有两棵子树**的有序树（左子树和右子树有区别）。

### 特殊二叉树

| 类型 | 定义 | 图示特征 |
|------|------|---------|
| **满二叉树** | 每层节点数都达到最大值，高度为 h 时有 2ʰ-1 个节点 | 完美三角形 |
| **完全二叉树** | 除最后一层外各层节点数均满，最后一层节点从**左到右**连续排列 | 满二叉树去掉右下角 |
| **二叉排序树（BST）** | 左子树所有节点 < 根节点 < 右子树所有节点 | 有序结构 |
| **平衡二叉树（AVL）** | 任意节点的左右子树高度差不超过 1 | 高度均衡 |

### 二叉树的重要性质（★★★★★ 必背）

**性质 1**：第 i 层最多有 **2^(i-1)** 个节点（i ≥ 1）

**性质 2**：高度为 h 的二叉树最多有 **2ʰ - 1** 个节点

**性质 3**（最重要！）：**n₀ = n₂ + 1**

即：叶节点数 = 度为 2 的节点数 + 1

**性质 3 推导（理解后可自行推导，不用死记）**：
```
设：n₀ = 叶节点数，n₁ = 度为1的节点数，n₂ = 度为2的节点数
节点总数：n = n₀ + n₁ + n₂  ···①
边总数 = n - 1（树的性质：n个节点有n-1条边）
边总数也等于所有节点的出度之和：0×n₀ + 1×n₁ + 2×n₂ = n₁ + 2n₂  ···②
由①②：n₀ + n₁ + n₂ - 1 = n₁ + 2n₂
化简：n₀ = n₂ + 1  ✓
```

**性质 4**：有 n 个节点的完全二叉树，高度为 **⌊log₂n⌋ + 1**

**性质 5**：完全二叉树的节点编号（从 1 开始，从上到下、从左到右编号）：
- 节点 i 的**父节点**编号：`⌊i/2⌋`（i > 1 时）
- 节点 i 的**左子节点**编号：`2i`（若 2i ≤ n）
- 节点 i 的**右子节点**编号：`2i+1`（若 2i+1 ≤ n）
- **叶节点**编号范围：`⌊n/2⌋+1` 到 `n`

**例题 1**：一棵二叉树有 5 个度为 2 的节点，则它的叶节点数为多少？

由 n₀ = n₂ + 1 = 5 + 1 = **6 个叶节点**

**例题 2**：一棵完全二叉树有 100 个节点，问有多少个叶节点？

- 高度 = ⌊log₂100⌋ + 1 = 6 + 1 = 7
- 利用 n₀ = n₂ + 1：需先知道 n₁ 和 n₂

**简便方法**：完全二叉树中，n₁ 只可能为 0 或 1
- 若 n 为偶数：n₁ = 1，n₀ = n/2
- 若 n 为奇数：n₁ = 0，n₀ = (n+1)/2

n=100（偶数），叶节点数 = 100/2 = **50**

**验证**：n₀=50，n₁=1，n₂=n₀-1=49，总节点 50+1+49=100 ✓

---

## 6.3 二叉树的遍历 ★★★★★

遍历就是按某种规则**不重复地访问每个节点一次**。

### 四种遍历方式

设节点 N 有左子树 L 和右子树 R：

| 遍历方式 | 访问顺序 | 记忆口诀 |
|---------|---------|---------|
| **先序遍历（前序）** | N → L → R | **根**左右 |
| **中序遍历** | L → N → R | 左**根**右 |
| **后序遍历** | L → R → N | 左右**根** |
| **层序遍历** | 从上到下，从左到右逐层 | 按层访问 |

### 遍历示例（重点）

以下二叉树：
```
        A
       / \
      B   C
     / \   \
    D   E   F
```

- **先序**：A → B → D → E → C → F
- **中序**：D → B → E → A → C → F
- **后序**：D → E → B → F → C → A
- **层序**：A → B → C → D → E → F

> ⚠️ **口诀验证**：中序遍历二叉排序树（BST），得到的序列一定是**有序的（递增序列）**。

### 由两种遍历序列还原二叉树（必考）

**规律**：
- **先序序列的第一个元素** = 根节点
- **后序序列的最后一个元素** = 根节点
- 找到根节点后，在**中序序列**中定位根节点，根节点**左边**是左子树，**右边**是右子树

> ⚠️ **注意**：必须有中序序列才能唯一确定二叉树。仅凭先序+后序无法唯一确定。

**例题（必做）**：已知先序序列 `ABCDEFGH`，中序序列 `CBEDFAGH`，求后序序列。

**解题步骤**：

**第一步**：先序第一个字母 A = 根节点

**第二步**：在中序 `CBEDFAGH` 中找 A，A 在第 6 位
- A 左边：`CBEDF`（5 个节点）= 左子树的中序
- A 右边：`GH`（2 个节点）= 右子树的中序

**第三步**：先序中 A 之后的 5 个字母 `BCDEF` = 左子树的先序，`GH` = 右子树的先序

**第四步**：递归处理左子树
- 左子树先序：`BCDEF`，中序：`CBEDF`
- 根 = B，B 在中序位置 3：左子树中序 `CB`（2个），右子树中序 `DF`（2个）
  - 左子树先序 `CD`...

实际上这种题在考场上用**画树**的方法更快：

```
              A
            /   \
           B     G
          / \     \
         C   D     H
              \
               ...
```

逐步还原，最终后序遍历结果为：**CEFDBHGA**

## 6.4 线索二叉树 ★★

**背景**：n 个节点的二叉树有 n+1 个空指针域（浪费），利用这些空指针存储前驱/后继信息。

**规则**：
- 若某节点**左子节点为空**，将左指针改为指向该节点的**中序前驱**
- 若某节点**右子节点为空**，将右指针改为指向该节点的**中序后继**

考试主要考：**中序线索二叉树**中某节点的后继怎么找：
- 若右指针是线索（rtag=1）：直接指向后继
- 若右指针是正常子节点（rtag=0）：后继是右子树中最左边的节点

---

## 6.5 哈夫曼树（最优二叉树）★★★★★

### 基本概念

- **路径长度**：从树根到某节点的路径上的边数
- **节点的带权路径长度**：节点的权值 × 该节点到根的路径长度
- **树的带权路径长度（WPL）**：所有**叶节点**带权路径长度之和
- **哈夫曼树**：给定 n 个权值的叶节点，构造 WPL 最小的二叉树

### 哈夫曼树的构造算法（必背）

1. 将 n 个权值视为 n 棵单节点树
2. 每次从森林中选出**权值最小的两棵树**，合并为一棵新树（新树根的权值 = 两子树根权值之和）
3. 将新树加入森林，重复步骤 2，直到只剩一棵树

### 哈夫曼树的特点

- 哈夫曼树中**没有度为 1 的节点**（n₁ = 0）
- n 个叶节点构造的哈夫曼树共有 **2n-1** 个节点
- **权值越大的节点，路径越短**（越靠近根）

### 例题（必做，验证 WPL 计算）

**给定权值 {2, 3, 4, 7}，构造哈夫曼树并求 WPL。**

**步骤**：
1. 初始：{2, 3, 4, 7}
2. 选最小两个 2 和 3，合并为 5：{4, 5, 7}
3. 选最小两个 4 和 5，合并为 9：{7, 9}
4. 合并 7 和 9 为 16：{16}，构造完成

**树结构**：
```
         16
        /  \
       7    9
           / \
          4   5
             / \
            2   3
```

**WPL 计算**（只计算叶节点）：
- 节点 7：路径长度 1，带权路径 = 7 × 1 = 7
- 节点 4：路径长度 2，带权路径 = 4 × 2 = 8
- 节点 2：路径长度 3，带权路径 = 2 × 3 = 6
- 节点 3：路径长度 3，带权路径 = 3 × 3 = 9

**WPL = 7 + 8 + 6 + 9 = 30**

**验证（简便算法）**：WPL = 每次合并的权值之和的总和 = 5 + 9 + 16 = **30** ✓

---

## 6.6 二叉排序树（BST）★★★★

### 定义

对于每个节点：
- **左子树**所有节点的值 < 该节点的值
- **右子树**所有节点的值 > 该节点的值

### 特点

- **中序遍历 BST 得到有序（递增）序列**（重要性质）
- 查找/插入/删除平均时间复杂度：**O(log n)**
- 最坏情况（树退化为链表，即输入有序序列）：**O(n)**

### BST 查找过程

```
查找值 x：
1. 若树空，查找失败
2. x == 根节点值，查找成功
3. x < 根节点值，在左子树中查找
4. x > 根节点值，在右子树中查找
```

### 例题

**将序列 {45, 24, 53, 12, 37} 依次插入构造 BST，并给出中序遍历结果。**

插入过程：
1. 插入 45：根
2. 插入 24：24 < 45，放 45 左子树
3. 插入 53：53 > 45，放 45 右子树
4. 插入 12：12 < 45，到左子树；12 < 24，放 24 左子树
5. 插入 37：37 < 45，到左子树；37 > 24，放 24 右子树

```
        45
       /  \
     24    53
    /  \
  12   37
```

**中序遍历**（左根右）：12 → 24 → 37 → 45 → 53

**结果**：`12, 24, 37, 45, 53`（有序升序序列）✓

---

## 6.7 平衡二叉树（AVL 树）★★

### 定义

**平衡因子（BF）**= 左子树高度 - 右子树高度

AVL 树要求每个节点的 **|BF| ≤ 1**（即 BF ∈ {-1, 0, 1}）。

### 四种失衡类型和旋转操作

| 失衡类型 | 描述 | 解决方法 |
|---------|------|---------|
| **LL 型** | 在左子树的左子树插入 | 单次右旋 |
| **RR 型** | 在右子树的右子树插入 | 单次左旋 |
| **LR 型** | 在左子树的右子树插入 | 先左旋后右旋 |
| **RL 型** | 在右子树的左子树插入 | 先右旋后左旋 |

> ⚠️ **考试提示**：AVL 树主要考"什么是平衡二叉树"和"平衡因子"的概念，四种旋转类型需了解，一般不考详细旋转过程。

---

## 6.8 B 树与 B+ 树 ★★

### B 树（m 阶 B 树）

B 树是**多路平衡查找树**，主要用于磁盘文件系统和数据库索引。

**m 阶 B 树的性质**：
1. 每个节点最多有 **m 棵子树**（m-1 个关键字）
2. 根节点至少有 **2 棵子树**（若非叶节点）
3. 非根非叶节点至少有 **⌈m/2⌉ 棵子树**
4. 所有叶节点在**同一层**（不存储信息）

### B+ 树（B 树的变体）

| 特点 | B 树 | B+ 树 |
|------|------|-------|
| 数据存储位置 | 所有节点 | **只在叶节点** |
| 内部节点 | 存数据 | 只存索引（冗余键）|
| 叶节点连接 | 无 | **叶节点形成有序链表** |
| 适用范围查询 | 不擅长 | **擅长** |

> ⚠️ **考点**：数据库中的索引结构常用 **B+ 树**，因为叶节点链表支持高效的范围查询。

# 第七章：图

> **图是考试的另一大重点，每年必考 2～4 分。重点掌握：最短路径算法（Dijkstra）、最小生成树、拓扑排序、关键路径。**

## 7.1 图的基本概念 ★★★

### 图的定义

图 G 由顶点集 V 和边集 E 组成，记为 G=(V, E)。

| 概念 | 说明 |
|------|------|
| **有向图** | 边有方向，用 `<u, v>` 表示从 u 到 v 的弧 |
| **无向图** | 边无方向，用 `(u, v)` 表示 |
| **顶点的度** | 与该顶点相连的边数；有向图中分**入度**和**出度** |
| **完全图** | 任意两顶点间都有边；无向完全图有 n(n-1)/2 条边，有向完全图有 n(n-1) 条边 |
| **连通图** | 无向图中任意两顶点间都有路径 |
| **强连通图** | 有向图中任意两顶点间都有**双向**路径 |
| **连通分量** | 无向图的极大连通子图 |
| **权** | 边上的数值（如距离、费用）|
| **网** | 带权图 |

### 重要数量关系

| 结构 | 边数范围 |
|------|---------|
| 无向完全图（n 个顶点） | `n(n-1)/2` 条边 |
| 有向完全图（n 个顶点） | `n(n-1)` 条弧 |
| 无向图 | `0` 到 `n(n-1)/2` |
| 有向图 | `0` 到 `n(n-1)` |

---

## 7.2 图的存储结构 ★★★

### 邻接矩阵

用 n×n 的矩阵 A 存储，`A[i][j]=1` 表示 i 到 j 有边，`A[i][j]=0` 表示无边（带权图存权值，无边存 ∞）。

**优点**：判断两点是否有边 O(1)，适合**稠密图**
**缺点**：空间 O(n²)，对稀疏图浪费

**示例**（无向图）：
```
顶点：{1, 2, 3, 4}
边：(1,2), (1,3), (2,4), (3,4)

邻接矩阵：
     1  2  3  4
  1[ 0  1  1  0 ]
  2[ 1  0  0  1 ]
  3[ 1  0  0  1 ]
  4[ 0  1  1  0 ]
```

> ⚠️ 无向图的邻接矩阵是**对称矩阵**，矩阵中 1 的个数 = 边数×2。

### 邻接表

每个顶点维护一个**链表**，存储与该顶点相邻的所有顶点。

**优点**：空间 O(n+e)（e 为边数），适合**稀疏图**
**缺点**：判断两点是否有边需要遍历链表，O(degree)

---

## 7.3 图的遍历 ★★★

### 深度优先搜索（DFS）

类似树的先序遍历，用**栈**（或递归）实现。

**过程**：
1. 从起点出发，访问该顶点
2. 访问一个未访问的邻居，递归访问
3. 若无未访问邻居，回退
4. 直到所有顶点都被访问

**时间复杂度**：邻接矩阵 O(n²)，邻接表 O(n+e)

### 广度优先搜索（BFS）

类似树的层序遍历，用**队列**实现。

**过程**：
1. 从起点出发，将其入队
2. 出队一个顶点，访问其所有未访问邻居，并将邻居入队
3. 重复步骤 2，直到队列为空

**时间复杂度**：邻接矩阵 O(n²)，邻接表 O(n+e)

**DFS vs BFS 对比**：

| 特性 | DFS | BFS |
|------|-----|-----|
| 数据结构 | 栈（递归） | 队列 |
| 找最短路径 | 不保证 | **保证**（无权图） |
| 空间复杂度 | O(n)（递归栈） | O(n)（队列） |

### 遍历示例

图：顶点 {1,2,3,4,5}，边 {(1,2),(1,3),(2,4),(2,5),(3,5)}，从顶点 1 出发。

**DFS 序列**（假设邻接表按顶点编号排列）：
1 → 2 → 4 → 5 → 3

**BFS 序列**：
1 → 2 → 3 → 4 → 5

---

## 7.4 最小生成树 ★★★★

**生成树**：连通图的生成树是包含图中**全部顶点**、且边数最少（n-1 条）的连通子图。

**最小生成树（MST）**：带权连通图中，权值总和最小的生成树。

### Prim 算法（普里姆算法）

**思想**：从任意顶点出发，每次选择**距离当前树最近**的顶点加入。

**步骤**：
1. 初始化：选一个顶点加入集合 U
2. 在所有一端在 U 内、一端在 U 外的边中，选权值最小的边
3. 将该边另一端顶点加入 U
4. 重复步骤 2-3，直到所有顶点都在 U 中

**时间复杂度**：O(n²)（适合**稠密图**）

### Kruskal 算法（克鲁斯卡尔算法）

**思想**：按边的权值**从小到大**依次加入，若不形成环则保留，否则丢弃。

**步骤**：
1. 将所有边按权值从小到大排序
2. 依次选择权值最小的边
3. 若该边两端点不在同一连通分量（不构成环），加入生成树
4. 重复，直到有 n-1 条边

**时间复杂度**：O(e log e)（适合**稀疏图**，e 为边数）

### 例题（必做）

下图各边权值：A-B=4, A-C=2, B-C=1, B-D=5, C-D=8, C-E=10, D-E=2，求最小生成树。

**Kruskal 步骤**（按权值排序）：

| 步骤 | 选边 | 权值 | 是否成环 | 操作 |
|------|------|------|---------|------|
| 1 | B-C | 1 | 否 | 加入 |
| 2 | A-C | 2 | 否 | 加入 |
| 3 | D-E | 2 | 否 | 加入 |
| 4 | A-B | 4 | **是**（A-C-B已连通）| 丢弃 |
| 5 | B-D | 5 | 否（B,D不在同一分量）| 加入 |

此时已有 4 条边，5 个顶点，MST 完成！

**MST 边集**：{B-C(1), A-C(2), D-E(2), B-D(5)}，总权值 = 1+2+2+5 = **10**

---

## 7.5 最短路径 ★★★★★

### Dijkstra 算法（单源最短路径）

从**单个源点**出发，求到所有其他顶点的最短路径。

**限制**：不能处理**负权边**（边的权值必须 ≥ 0）

**算法思想**：贪心策略，每次选择距源点最近的未访问顶点，更新其邻居的最短距离。

**步骤**：
1. 初始化：dist[源点]=0，其他 dist[v]=∞，标记集合 S={源点}
2. 对源点的每个邻居 v，更新 dist[v] = 边权
3. 从未访问顶点中选 dist 最小的顶点 u，加入 S
4. 更新 u 的邻居：若 dist[u]+w(u,v) < dist[v]，则 dist[v] = dist[u]+w(u,v)
5. 重复步骤 3-4，直到所有顶点都在 S 中

**时间复杂度**：O(n²)（n 为顶点数）

### Dijkstra 详细例题（必做）

图（有向图）：
```
A --4--> B
A --2--> C
C --1--> B
B --3--> D
C --5--> D
```

从顶点 A 出发，用 Dijkstra 求到所有顶点的最短路径。

**初始状态**：dist = {A:0, B:∞, C:∞, D:∞}，已访问集合 S = {}

| 步骤 | 选顶点 | dist[A] | dist[B] | dist[C] | dist[D] | 说明 |
|------|--------|---------|---------|---------|---------|------|
| 初始 | - | 0 | ∞ | ∞ | ∞ | |
| 1 | **A**（dist=0）| 0 | 4 | 2 | ∞ | A→B=4, A→C=2 |
| 2 | **C**（dist=2，最小）| 0 | **3** | 2 | **7** | C→B: 2+1=3<4，更新B; C→D: 2+5=7 |
| 3 | **B**（dist=3，最小）| 0 | 3 | 2 | **6** | B→D: 3+3=6<7，更新D |
| 4 | **D**（dist=6）| 0 | 3 | 2 | 6 | D无出边 |

**最终最短路径**：
- A → A：0
- A → C：2（直接 A→C）
- A → B：3（路径 A→C→B）
- A → D：6（路径 A→C→B→D）

### Floyd 算法（多源最短路径）

求图中**所有顶点对**之间的最短路径。

**特点**：
- 可以处理**负权边**（但不能有负权环）
- 时间复杂度：**O(n³)**
- 动态规划思想：逐步考虑以每个顶点作为中间节点

**核心公式**：
```
dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j])
```

对每个中间顶点 k（k 从 1 到 n），更新所有顶点对 (i, j) 的距离。

## 7.6 拓扑排序 ★★★★

**适用条件**：**有向无环图（DAG）**

**拓扑序列**：若图中存在弧 `<vi, vj>`，则 vi 一定在 vj 之前出现。

**应用**：课程先修关系、项目任务安排

### Kahn 算法（基于入度）

1. 找所有**入度为 0**的顶点，加入队列
2. 从队列取出一个顶点 u，输出 u
3. 删除 u 的所有出边，相应邻居入度减 1
4. 若邻居入度变为 0，加入队列
5. 重复步骤 2-4
6. 若输出顶点数 < 总顶点数，说明图中有**环**，拓扑排序失败

> ⚠️ **考点**：拓扑排序结果**不唯一**（可能存在多个入度为 0 的顶点，选择顺序不同）；若排序失败，说明存在环。

### 例题

有向图的弧：`<1,2>, <1,3>, <2,4>, <3,4>, <4,5>`

初始入度：顶点1入度0，顶点2入度1，顶点3入度1，顶点4入度2，顶点5入度1

| 步骤 | 队列 | 输出 | 操作 |
|------|------|------|------|
| 初始 | {1} | | |
| 1 | {} | 1 | 2和3的入度各减1，变为0，入队 |
| 2 | {3} | 1,2 | 4的入度减1，变为1 |
| 3 | {} | 1,2,3 | 4的入度减1，变为0，入队 |
| 4 | {} | 1,2,3,4 | 5的入度减1，变为0，入队 |
| 5 | {} | 1,2,3,4,5 | 完成 |

**拓扑序列**：1 → 2 → 3 → 4 → 5（或 1 → 3 → 2 → 4 → 5，均合法）

---

## 7.7 关键路径 ★★★★

**AOE 网（Activity On Edge）**：用边表示活动，顶点表示事件，边的权值表示活动持续时间。

**关键路径**：从源点到汇点的**最长路径**，决定项目的最短完成时间。

**关键术语**：

| 术语 | 符号 | 含义 |
|------|------|------|
| 事件最早发生时间 | ve(i) | 从源点到顶点 i 的最长路径长度 |
| 事件最迟发生时间 | vl(i) | 不影响总进度的前提下，顶点 i 最迟发生时间 |
| 活动最早开始时间 | e(i) | e(i) = ve(起点) |
| 活动最迟开始时间 | l(i) | l(i) = vl(终点) - 活动持续时间 |
| 活动时间余量（松弛时间）| l(i) - e(i) | 为 0 的活动是**关键活动** |

**关键活动**：`l(i) - e(i) = 0` 的活动，缩短关键活动时间可以缩短总工期。

### 例题（必做）

AOE 网：
```
顶点：{1,2,3,4,5}（1为源点，5为汇点）
活动：a1(1→2,3), a2(1→3,2), a3(2→4,2), a4(3→4,3), a5(2→5,1), a6(4→5,2)
```

**第一步：计算 ve（从左到右，取最大）**
- ve(1) = 0（源点）
- ve(2) = ve(1) + 3 = 3
- ve(3) = ve(1) + 2 = 2
- ve(4) = max(ve(2)+2, ve(3)+3) = max(5, 5) = 5
- ve(5) = max(ve(2)+1, ve(4)+2) = max(4, 7) = 7

**总工期 = ve(汇点) = ve(5) = 7**

**第二步：计算 vl（从右到左，取最小）**
- vl(5) = 7（汇点，等于 ve(5)）
- vl(4) = vl(5) - 2 = 5
- vl(2) = min(vl(4)-2, vl(5)-1) = min(3, 6) = 3
- vl(3) = vl(4) - 3 = 2
- vl(1) = min(vl(2)-3, vl(3)-2) = min(0, 0) = 0

**第三步：计算各活动的 e 和 l，找关键活动**

| 活动 | 弧 | 持续时间 | e=ve(起点) | l=vl(终点)-时间 | l-e | 是否关键 |
|------|-----|---------|-----------|----------------|-----|---------|
| a1 | 1→2 | 3 | 0 | 3-3=0 | 0 | **是** |
| a2 | 1→3 | 2 | 0 | 2-2=0 | 0 | **是** |
| a3 | 2→4 | 2 | 3 | 5-2=3 | 0 | **是** |
| a4 | 3→4 | 3 | 2 | 5-3=2 | 0 | **是** |
| a5 | 2→5 | 1 | 3 | 7-1=6 | 3 | 否 |
| a6 | 4→5 | 2 | 5 | 7-2=5 | 0 | **是** |

**关键路径**：1→2→4→5（经过 a1, a3, a6）**和** 1→3→4→5（经过 a2, a4, a6）

注意：**本例存在两条关键路径**，总工期为 7。

# 第八章：查找算法

> **查找是高频考点，每年出题 2～3 道。重点掌握折半查找的比较次数、哈希查找的冲突处理。**

## 8.1 顺序查找（线性查找）★★★

**思想**：从表头逐个比较，直到找到目标或扫描完全表。

- **时间复杂度**：O(n)
- **适用**：无序或有序的线性表（顺序表或链表）
- **平均比较次数**：(n+1)/2（假设等概率查找成功）

---

## 8.2 折半查找（二分查找）★★★★★

**前提条件**：**有序的顺序表**（必须是顺序表，不能是链表）

**思想**：每次取中间元素与目标比较，缩小查找范围。

```
low = 1, high = n
while(low <= high):
    mid = (low + high) / 2
    if A[mid] == key: 找到，返回 mid
    if A[mid] < key: low = mid + 1  （在右半部分找）
    if A[mid] > key: high = mid - 1 （在左半部分找）
返回失败
```

- **时间复杂度**：O(log n)
- **最多比较次数**：⌊log₂n⌋ + 1

### 判定树（必考）

折半查找过程可以用**判定树**描述，找到元素所需比较次数 = 该节点在判定树中的**层数**。

**例题（必做）**：有序表 {1, 3, 5, 7, 9, 11, 13}（共 7 个元素），查找 key=7。

```
步骤：
初始：low=1, high=7, mid=(1+7)/2=4
A[4]=7 == 7，查找成功！比较 1 次。
```

**查找 key=6（查找失败的情况）**：
```
步骤1：low=1, high=7, mid=4, A[4]=7 > 6, high=3
步骤2：low=1, high=3, mid=2, A[2]=3 < 6, low=3
步骤3：low=3, high=3, mid=3, A[3]=5 < 6, low=4
此时 low(4) > high(3)，查找失败。共比较 3 次。
```

### 最多比较次数（必背）

| 元素个数 n | 最多比较次数 |
|-----------|------------|
| 1 | 1 |
| 2～3 | 2 |
| 4～7 | 3 |
| 8～15 | 4 |
| n | ⌊log₂n⌋ + 1 |

---

## 8.3 哈希查找（散列查找）★★★★

### 基本概念

**哈希函数（散列函数）**：将关键字 key 映射到存储地址的函数，`H(key) = 存储地址`。

**冲突（碰撞）**：两个不同的 key 映射到相同地址，即 `H(k₁) = H(k₂)`（k₁ ≠ k₂）。

**装填因子（负载因子）**：`α = 表中元素个数 / 哈希表长度`

α 越小，冲突越少，查找越快；α 越大，空间利用率越高。

### 常用哈希函数

**除留余数法（最常用）**：`H(key) = key mod p`（p 通常取不大于表长的最大质数）

**直接定址法**：`H(key) = a × key + b`（适合关键字均匀分布）

### 冲突解决方法（必考）

#### 开放寻址法（闭散列）

当发生冲突时，在哈希表中找另一个空槽。

| 方法 | 公式 | 说明 |
|------|------|------|
| **线性探测** | `H(key) = (H₀ + i) mod m` | 依次找下一个槽 |
| **平方探测** | `H(key) = (H₀ ± i²) mod m` | 跳跃式探测，减少聚集 |
| **双重哈希** | `H(key) = (H₀ + i×H₂(key)) mod m` | 步长由第二哈希函数决定 |

其中 H₀ = H(key)，i = 1, 2, 3, ...

#### 拉链法（开散列）

每个哈希槽存储一个链表，冲突的元素链接到同一槽的链表上。

**优点**：删除操作简单，装填因子可以 > 1
**缺点**：有额外指针开销

### 哈希查找时间复杂度

- **平均情况**：O(1)（装填因子适当时）
- **最坏情况**：O(n)（全部冲突，退化为链表）

### 例题（必做）

哈希表大小 m=11，哈希函数 `H(key) = key mod 11`，采用**线性探测**解决冲突。

依次插入关键字：{22, 33, 4, 28, 55, 11}

| 关键字 | H(key) | 探测序列 | 最终位置 |
|--------|--------|---------|---------|
| 22 | 22 mod 11 = 0 | 0 空 | **0** |
| 33 | 33 mod 11 = 0 | 0 占用→1 空 | **1** |
| 4 | 4 mod 11 = 4 | 4 空 | **4** |
| 28 | 28 mod 11 = 6 | 6 空 | **6** |
| 55 | 55 mod 11 = 0 | 0 占用→1 占用→2 空 | **2** |
| 11 | 11 mod 11 = 0 | 0 占用→1 占用→2 占用→3 空 | **3** |

哈希表：`[22, 33, 55, 11, 4, -, 28, -, -, -, -]`

查找 55 需要比较：H(55)=0，位置 0 是 22（不等），位置 1 是 33（不等），位置 2 是 55（等），共比较 **3 次**。

# 第九章：排序算法

> **排序是整个数据结构考试中出题最多的知识点，每年必考 3～5 分。重点掌握：各算法的时间/空间复杂度、稳定性、适用场景。**

## 9.1 排序算法总览（必背表格）★★★★★

| 排序算法 | 最好时间 | 平均时间 | 最坏时间 | 空间复杂度 | 稳定性 |
|---------|---------|---------|---------|----------|--------|
| **冒泡排序** | O(n) | O(n²) | O(n²) | O(1) | **稳定** |
| **选择排序** | O(n²) | O(n²) | O(n²) | O(1) | **不稳定** |
| **插入排序** | O(n) | O(n²) | O(n²) | O(1) | **稳定** |
| **希尔排序** | O(n log n) | O(n^1.3) | O(n²) | O(1) | **不稳定** |
| **快速排序** | O(n log n) | O(n log n) | O(n²) | O(log n) | **不稳定** |
| **归并排序** | O(n log n) | O(n log n) | O(n log n) | O(n) | **稳定** |
| **堆排序** | O(n log n) | O(n log n) | O(n log n) | O(1) | **不稳定** |
| **基数排序** | O(d(n+r)) | O(d(n+r)) | O(d(n+r)) | O(n+r) | **稳定** |

> d：关键字位数；r：基数（进制数）

**记忆口诀**：
- **稳定**的排序：冒泡、插入、归并、基数（**冒插归基**）
- **不稳定**的排序：选择、希尔、快速、堆（**选希快堆**）
- 任何情况下都是 O(n log n)：只有**归并排序**和**堆排序**

---

## 9.2 冒泡排序 ★★★

**思想**：每次比较相邻元素，将较大的元素向右（向后）"冒泡"。每一趟将最大元素移到末尾。

**过程**（升序）：
```
原始：[5, 3, 1, 4, 2]

第1趟：比较相邻，将最大值5移到末尾
[3, 1, 4, 2, 5]

第2趟：将次大值4移到末尾
[1, 3, 2, 4, 5]

第3趟：
[1, 2, 3, 4, 5]

已有序，提前结束
```

**改进**：若某趟没有发生交换，说明已有序，可提前结束（最好情况 O(n)）。

**最多需要 n-1 趟**（每趟至少将一个元素放到最终位置）。

---

## 9.3 选择排序 ★★

**思想**：每趟在剩余未排序元素中找最小值，放到已排序序列末尾。

```
原始：[5, 3, 1, 4, 2]

第1趟：找最小值1，与位置1的5交换 → [1, 3, 5, 4, 2]
第2趟：找最小值2，与位置2的3交换 → [1, 2, 5, 4, 3]
第3趟：找最小值3，与位置3的5交换 → [1, 2, 3, 4, 5]
第4趟：[1, 2, 3, 4, 5]（已有序）
```

**为什么不稳定？**：例如 [3, 3*, 1]，第1趟找最小1与3交换 → [1, 3*, 3]，两个3的相对顺序改变了。

---

## 9.4 插入排序 ★★★

**思想**：将未排序元素**插入**到已排序序列的适当位置（类似整理扑克牌）。

```
原始：[5, 3, 1, 4, 2]

初始有序部分：[5]
插入3：[3, 5]
插入1：[1, 3, 5]
插入4：[1, 3, 4, 5]
插入2：[1, 2, 3, 4, 5]
```

**特点**：
- 对**基本有序**的数据效果好（最好情况 O(n)）
- 适合数据量小或基本有序的场景

---

## 9.5 希尔排序 ★★

**思想**：插入排序的改进版，先按较大间隔分组排序，逐渐减小间隔，最后进行间隔为 1 的插入排序。

- 可以让元素跨越较大距离移动，比普通插入排序更快
- 时间复杂度取决于增量序列，通常约 O(n^1.3)
- **不稳定**：不同子序列的排序可能打乱相等元素的相对顺序

---

## 9.6 快速排序 ★★★★★

**思想**：分治法。选一个**基准元素（pivot）**，将序列划分为左（小于 pivot）和右（大于 pivot）两部分，递归排序。

**Partition 过程**（以第一个元素为基准）：

```
原始：[4, 3, 5, 1, 2]   pivot = 4

双指针法：left=1, right=5（下标）
right 向左找小于 4 的元素：right=5（2<4），停止
left 向右找大于 4 的元素：left=2（3<4，继续），left=3（5>4），停止
交换 A[left] 和 A[right]：[4, 3, 2, 1, 5]
right 向左：right=4（1<4），停止
left 向右：left=4（1<4，继续），left=5 但 left > right，停止
交换 pivot 与 A[right]：[1, 3, 2, 4, 5]

pivot(4) 已就位，左边 [1,3,2] 均小于 4，右边 [5] 均大于 4
```

**一次 Partition 后**：pivot 在最终位置，左边 < pivot，右边 > pivot。

**快速排序最坏情况**：当数组已经**有序**（升序或降序）时，每次 pivot 选到最大或最小元素，划分极不均匀，时间退化为 **O(n²)**。

**优化方法**：
- **随机选 pivot**：避免已排序数据的最坏情况
- **三数取中**：取首、中、尾三个元素的中间值作为 pivot

---

## 9.7 归并排序 ★★★★

**思想**：分治法。将序列分为两半，分别排序，再**合并（Merge）**为有序序列。

```
原始：[5, 3, 1, 4, 2]

分割：[5,3] 和 [1,4,2]
递归排序：[3,5] 和 [1,2,4]
合并：比较两序列首元素，取小的放入结果
[1,2,3,4,5]
```

**特点**：
- 任何情况下时间复杂度都是 **O(n log n)**（最好、最坏、平均）
- 需要 **O(n) 额外空间**（辅助数组）
- **稳定排序**

**2路归并**：每次合并两个有序子序列，是最常见的形式。

---

## 9.8 堆排序 ★★★★

### 堆的定义

**大根堆（最大堆）**：父节点 ≥ 子节点（堆顶是最大元素）
**小根堆（最小堆）**：父节点 ≤ 子节点（堆顶是最小元素）

堆用**完全二叉树**表示，存储在数组中（1-indexed）：
- 节点 i 的父节点：⌊i/2⌋
- 节点 i 的左子节点：2i
- 节点 i 的右子节点：2i+1

### 堆排序过程（以升序为例，用大根堆）

1. **建堆**：将 n 个元素建为大根堆（从第 ⌊n/2⌋ 个节点开始，向上执行"下滤"操作）
2. **排序**：重复以下步骤 n-1 次：
   - 将堆顶（最大元素）与最后一个元素交换
   - 堆大小减 1
   - 对新堆顶执行"下滤（sift down）"，重新维护堆性质

**建堆时间**：O(n)（不是 O(n log n)，这是一个重要结论）

**每次"下滤"**：O(log n)

**总时间**：O(n)（建堆）+ O(n log n)（n-1 次下滤）= **O(n log n)**

### 例题（必做）

序列 {4, 1, 3, 2, 16, 9, 10, 14, 8, 7}，建立大根堆。

初始数组（1-indexed）：4, 1, 3, 2, 16, 9, 10, 14, 8, 7

n=10，从 ⌊10/2⌋=5 开始向上执行下滤：
- 位置 5（值16）：子节点是位置10（值7），16>7，不需调整
- 位置 4（值2）：子节点是位置8（值14）和9（值8），14>2，交换 → 位置4变为14
- 位置 3（值3）：子节点是位置6（值9）和7（值10），10>3，交换 → 位置3变为10
- 位置 2（值1）：子节点是位置4（值14）和5（值16），16>1，交换 → 位置2变为16；继续对位置5（值1）下滤，子节点是位置10（值7），7>1，交换
- 位置 1（值4）：子节点是位置2（值16）和3（值10），16>4，交换 → 位置1变为16；继续对位置2（值4）下滤，子节点是位置4（值14）和5（值1→7），14>4，交换 → 位置2变为14；继续...

**最终大根堆**：16, 14, 10, 8, 7, 9, 3, 2, 4, 1

（建堆结果，堆顶 16 是最大元素）

## 9.9 基数排序 ★★

**思想**：不比较元素大小，按位数依次排序（最低位优先 LSD，或最高位优先 MSD）。

**过程（LSD，最低位优先）**：
1. 按个位数分桶，收集
2. 按十位数分桶，收集
3. 按百位数分桶，收集
4. ...直到最高位

**时间复杂度**：O(d×(n+r))，其中 d 是位数，n 是元素个数，r 是基数（进制，如十进制 r=10）

**特点**：
- 稳定排序
- 不基于比较（突破了比较排序 O(n log n) 的下界）
- 适用于**关键字值范围已知且位数不多**的场景

## 第九章练习题

### 练习 1（必做，极高频）

以下排序算法中，时间复杂度在**任何情况下（最好、平均、最坏）**都是 O(n log n) 的是（　　）

A. 快速排序　　B. 归并排序　　C. 堆排序　　D. B 和 C

**答案：D**

**解析**：
- 快速排序最坏情况为 O(n²)（输入有序时），排除 A
- **归并排序**：最好/平均/最坏都是 O(n log n) ✓
- **堆排序**：最好/平均/最坏都是 O(n log n) ✓
- 因此 D 正确

### 练习 2（必做，极高频）

以下关于排序算法稳定性的描述，**错误**的是（　　）

A. 冒泡排序是稳定的
B. 快速排序是不稳定的
C. 归并排序是稳定的
D. 堆排序是稳定的

**答案：D**

**解析**：堆排序在堆调整（sift down）过程中可能改变相等元素的相对顺序，是**不稳定**的排序算法。

稳定排序（记忆：**冒插归基**）：冒泡、插入、归并、基数

### 练习 3（中等）

对序列 {36, 25, 48, 12, 60, 18} 进行从小到大排序，使用**快速排序**以第一个元素 36 为基准，一趟划分后的结果是（　　）

A. 12, 18, 25, 36, 60, 48
B. 25, 12, 18, 36, 48, 60
C. 25, 12, 18, 36, 60, 48
D. 12, 18, 25, 48, 36, 60

**答案：B**

**解析**：

以 36 为 pivot，使用双端扫描：

```
原始：[36, 25, 48, 12, 60, 18]
pivot = 36, left=0, right=5

right 向左找 < 36 的：A[5]=18 < 36，停止（right=5）
left 向右找 > 36 的：A[1]=25 < 36（继续），A[2]=48 > 36，停止（left=2）
交换 A[left] 和 A[right]：[36, 25, 18, 12, 60, 48]

right 向左找 < 36 的：A[4]=60 > 36（继续），A[3]=12 < 36，停止（right=3）
left 向右找 > 36 的：A[3]=12 < 36（继续），left=4 > right=3，停止

交换 pivot(A[0]) 与 A[right=3]：[12, 25, 18, 36, 60, 48]
```

等等，这样结果是 [12, 25, 18, 36, 60, 48]，pivot=36 在位置 3，左边 {12,25,18} < 36，右边 {60,48} > 36。

但选项中没有 `12, 25, 18, 36, 60, 48`，而 B 是 `25, 12, 18, 36, 48, 60`。

> 不同快排实现（从右先扫 vs 从左先扫）结果不同，关键是：**pivot 36 应在最终位置，左边 < 36，右边 > 36**。验证 B：`25, 12, 18, **36**, 48, 60` → 左边{25,12,18}均 < 36，右边{48,60}均 > 36，符合划分结果。答案 **B** 正确。

### 真题解析 10（2021 年真题风格）

以下关于各排序算法的叙述，正确的是（　　）

A. 快速排序在所有情况下时间复杂度都是 O(n log n)
B. 希尔排序是稳定排序
C. 对于逆序排列的数据，插入排序性能最差
D. 归并排序需要额外的 O(n) 辅助空间

**答案：D**

**解析**：

# 第十章：算法设计策略

> **算法设计策略是上午选择题和下午案例分析的重要考点，约占 2～4 分。**

## 10.1 分治法 ★★★

### 基本思想

将问题**分**解为若干个**规模较小**且**结构相同**的子问题，**递归地**解决子问题，再将子问题的解**合并**为原问题的解。

**适用条件**：
1. 问题可以分解为规模更小的相同子问题
2. 子问题的解可以合并为原问题的解
3. 子问题之间**相互独立**（无公共子问题，否则用动态规划）

**典型应用**：
| 算法 | 时间复杂度 |
|------|----------|
| 归并排序 | O(n log n) |
| 快速排序 | O(n log n) |
| 二分查找 | O(log n) |
| 矩阵乘法（Strassen）| O(n^2.81) |
| 大整数乘法（Karatsuba）| O(n^1.58) |

---

## 10.2 动态规划 ★★★★

### 基本思想

将原问题分解为若干**重叠子问题**，通过**记忆化**存储子问题的解（避免重复计算），自底向上求解。

**与分治法的区别**：
- 分治法：子问题**相互独立**，无重叠
- 动态规划：子问题**有重叠**，存在重复计算，用表格存储避免重算

**适用条件（必背）**：
1. **最优子结构**：问题的最优解包含子问题的最优解
2. **重叠子问题**：递归计算时会反复遇到相同子问题

**典型应用**：
| 问题 | 说明 |
|------|------|
| 斐波那契数列 | f(n) = f(n-1) + f(n-2)，用数组存储避免重复计算 |
| 最长公共子序列（LCS）| 字符串匹配的经典 DP |
| 背包问题 | 0-1背包、完全背包 |
| 矩阵链乘 | 找最优括号化方案 |
| 最短路径（Floyd）| O(n³) 全源最短路径 |

### 最长公共子序列（LCS）

**定义**：两个序列 X 和 Y 的 LCS 是**最长的**同时是 X 和 Y 子序列的序列（子序列不要求连续）。

**递推公式**：

设 X = x₁x₂...xₘ，Y = y₁y₂...yₙ，c[i][j] = LCS(X[:i], Y[:j]) 的长度：

```
c[i][j] = 0                          若 i=0 或 j=0
c[i][j] = c[i-1][j-1] + 1           若 xᵢ = yⱼ
c[i][j] = max(c[i-1][j], c[i][j-1]) 若 xᵢ ≠ yⱼ
```

**例题**：X = "ABCBDAB"，Y = "BDCAB"，LCS 长度？

（此为经典教材例题，LCS 长度为 4，如 "BCAB" 或 "BDAB"）

## 10.3 贪心算法 ★★★

### 基本思想

每步都做出**当前看来最优**的选择（局部最优），希望这样得到全局最优解。

**注意**：贪心算法**不一定**能得到全局最优解，适用于满足**贪心选择性质**和**最优子结构**的问题。

**适用的典型问题**：
| 问题 | 贪心策略 |
|------|---------|
| 活动选择问题 | 每次选结束时间最早的活动 |
| 哈夫曼编码 | 每次合并权值最小的两棵树 |
| 最小生成树（Kruskal）| 每次选权值最小且不成环的边 |
| 最小生成树（Prim）| 每次选距离当前树最近的顶点 |
| 背包问题（分数版）| 每次选单位重量价值最高的物品 |

> ⚠️ 注意：0-1 背包**不能用贪心**（必须用动态规划），但分数背包可以用贪心。

---

## 10.4 回溯法 ★★

### 基本思想

系统地搜索问题的解空间，当确定某条路径不能到达目标时，**回溯**（撤销选择）并尝试其他路径。

**本质**：深度优先搜索 + 剪枝

**适用**：组合优化、约束满足问题
| 问题 | 说明 |
|------|------|
| N 皇后问题 | 在 N×N 棋盘上放 N 个皇后，互不攻击 |
| 图的着色问题 | 用最少颜色给图着色 |
| 0-1 背包 | 找最优解（回溯 + 剪枝） |
| 旅行商问题（TSP）| 最短哈密顿回路（NP 难问题）|

**与穷举的区别**：回溯有**剪枝**，避免对不可能成功的路径进行搜索，效率远高于纯穷举。

以下问题中，适合用**动态规划**而不适合用**贪心算法**求解的是（　　）

- C. 分数背包：贪心，按单位重量价值排序取即可 ✓
- D. 活动选择：贪心，按结束时间排序选 ✓

分治法和动态规划的主要区别是（　　）

A. 分治法用递归，动态规划不用递归
B. 分治法的子问题相互独立，动态规划的子问题存在重叠

Dijkstra 单源最短路径算法使用的算法设计策略是（　　）
