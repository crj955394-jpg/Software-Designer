算法基础：
    五大重要特性
        有限性：有限步骤内能够完成，单个步骤在有限时间内能完成
        确定性：每一步不能有歧义，相同输入需要有相同输出
        可行性：每一步都是可行的，通过优先步骤能够完成
        输入：算法有0个或多个输入
        输出：算法有一个或多个输出
    算法复杂度分析
        时间复杂度：用 O 大写的O标识 ，常见时间复杂度排序 O(1) < O(log₂n) < O(n) < O(n·log₂n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
            | 复杂度 | 名称 | 典型场景 |
            |--------|------|----------|
            | O(1) | 常数阶 | 数组按下标访问、哈希表查找（理想） |
            | O(log₂n) | 对数阶 | 二分查找、平衡BST查找 |
            | O(n) | 线性阶 | 顺序查找、遍历数组 |
            | O(n·log₂n) | 线性对数阶 | 快速排序、归并排序、堆排序（平均） |
            | O(n²) | 平方阶 | 冒泡排序、直接插入排序、简单选择排序 |
            | O(n³) | 立方阶 | 普通矩阵乘法 |
            | O(2ⁿ) | 指数阶 | 穷举法、递归求斐波那契（朴素） |
            | O(n!) | 阶乘阶 | 旅行商问题（穷举） |
            计算方法：
                找算法中执行次数最多的语句，计算这个语句执行的数量级，就只看这个，其他比他数量级低的语句不管
            常数项规则：只要代码中执行次数和n无关，无论执行多少次 结果都是O（1）
            加法规则：顺序执行的代码，总复杂度等于最复杂的那个， 一段代码前半部分 O (n)，后半部分 O (n²)，总复杂度为 O (n²)。
            乘法规则：嵌套循环的复杂度=内外循环的乘积。 外层循环 O (n)，内层循环 O (logn)，总复杂度为 O (nlogn)。
            对数阶 O (logn) 计算
                int i = 1;
                    while(i <= n){
                        i = i * 2; // 核心语句，执行次数最多
                    }
                在这里核心语句不是执行n次，i是成倍增长的，初始 i=1，每次乘 2，执行 k 次后，i=2ᵏ。当 2ᵏ > n 时，循环停止，解得 k = log₂n（以 2 为底 n 的对数）
                对数阶的底数无论为多少，统一写成 O (logn)
                只要循环变量是成倍增长 / 缩减，时间复杂度就是 O (logn)。例：i=i*3、i=i/2，都是 O (logn)。
        空间复杂度
            空间复杂度用来估算算法执行过程中，额外占用的存储空间随问题规模 n 增长的变化趋势,空间复杂度只算额外申请的存储空间，输入数据本身占用的空间不算！
            O(1)：常数阶，额外申请的空间固定，和 n 无关。这种算法也叫原地算法（例：冒泡排序，只需要几个临时变量，空间复杂度 O (1)）。
            O(n)：线性阶，额外申请的空间和 n 成正比（例：申请一个长度为 n 的辅助数组）。
            O(logn)：对数阶，常见于递归算法的调用栈空间（例：折半查找的递归实现）。
            O(n²)：平方阶，额外申请 n 行 n 列的二维数组。
            递归算法的空间复杂度 = 递归的深度 × 每次递归额外申请的空间。如果每次递归申请的空间是 O (1)，那么空间复杂度就等于递归的深度。
            斐波那契递归实现
                int fib(int n){
                        if(n <= 1) return n; // 递归终止条件
                        return fib(n-1) + fib(n-2); // 递归调用
                    }
            递归的深度为 n，每次递归只申请常数个变量，空间复杂度为O(n)。

线性表
    n个相同类型数据元素的有限序列
    特点：只有一个首元素，只有一个尾元素，除首尾外，每个元素只有一个直接前驱和一个直接后驱，元素个数n 为线性表的长度，n=0时称为空表
    线性表有两种存储方式：顺序存储（顺序表）和链式存储（链表）
    顺序表：用一组连续的存储单元顺序存放线性表中的元素，逻辑上相邻的元素，物理位置上也相邻
        求顺序表中某个元素的地址：首元素地址 + （i-1）*每个元素的大小
            顺序表首地址为 1000，每个整型元素占 4 字节，求第 8 个元素的地址。 LOC(a₈) = 1000 + (8 - 1) × 4 = 1000 + 28 = 1028
        插入操作：在地i个位置插入元素，从i~n后移一位，移动了 n-i+1 位个元素，时间复杂度为O（n）
        删除操作：将第i个元素删除，需要将 i+1~n的元素前移一位，移动了n-i位个元素，时间复杂度为O（n）
        顺序表操作时间复杂度：按下标访问：O（1），插入：O（n），删除：O（n），按值查找：O（n）
        插入操作平均移动了 n/2个元素
        删除操作平均移动了 （n-1）/2个元素
    链表：链表每个节点分为两个部分：数据域（存储数据）和指针域（存储下一个节点的地址）
        单链表：带头结点的好处：首个数据和其他数据节点的操作统一，删除整个链表的时候，头节点指向空即可
            常见操作时间复杂度：
                按位置访问第i个：O（n）
                头部插入/删除：O（1）
                尾部插入（无尾指针）：O(n)
                已知节点后插入：O（1）
                已知节点后删除：O（n）因为需要找前驱节点
        双链表：有两个指针，一个前驱指针，一个后驱指针，这样删除节点的时候时间复杂度就是O（1）
        循环链表：优点是从任意节点出发都能遍历整个链表
            循环单链表：尾节点的指针指向头节点，而不是NULL
            循环双链表：头节点的前驱指向尾节点，尾节点的后继指向头节点
        | 特性 | 顺序表 | 链表 |
        |------|--------|------|
        | 存储方式 | 连续内存 | 非连续内存 |
        | 随机访问 | **O(1)** | O(n) |
        | 插入/删除 | O(n)（移动元素） | **O(1)**（已知位置，改指针）|
        | 存储密度 | **高**（只存数据） | 低（有指针开销）|
        | 查找操作 | O(n) | O(n) |
        | 适用场景 | 频繁查询、少修改 | 频繁插入删除 |

栈和队列
    栈：栈是一种后进先出的数据结构，只有栈顶能够操作
        入栈：push   时间复杂度：O（1）
        出栈：pop    时间复杂度：O（1）
        取栈顶：peek 时间复杂度：O（1）
        应用场景：表达式求值：中缀表达式转后缀，后缀表达式计算
                 括号匹配，检查括号是否匹配
                 递归实现，函数调用使用系统栈
                 深度优先搜索，用栈模拟递归
                 进制转换，余数入栈，出栈得结果
        计算中缀转后缀
        顺序栈的溢出问题：
            上溢：栈满还push
            下溢：栈空还pop
    队列：队列是一种先进先出的数据结构，只能在在队头出队，在队尾入队
        循环队列
            入队：rear = (rear + 1) % MaxSize。
            出队：front = (front + 1) % MaxSize。
            队空 | `front == rear` | 队头等于队尾
            队满 | `(rear + 1) % MAXSIZE == front` | 牺牲一个存储位置来区分空和满
            队列中元素个数**：`(rear - front + MAXSIZE) % MAXSIZE`
        链队列
            入队：在链表尾部插入新结点，修改队尾指针。
            出队：删除链表头部的结点，修改队头指针。
            队空条件：队头和队尾都指向头结点。
            软考常考的队列应用场景：
                操作系统的作业调度、进程调度。
                缓冲区管理（例：打印队列、消息队列）。
                广度优先搜索（BFS）算法的实现。
                排队系统的模拟。
