算法基础：
    五大重要特性
        有限性：有限步骤内能够完成，单个步骤在有限时间内能完成
        确定性：每一步不能有歧义，相同输入需要有相同输出
        可行性：每一步都是可行的，通过优先步骤能够完成
        输入：算法有0个或多个输入
        输出：算法有一个或多个输出
    算法复杂度分析
        时间复杂度：用 O 大写的O标识 ，常见时间复杂度排序 O(1) < O(log₂n) < O(n) < O(n·log₂n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
            | 复杂度 | 名称 | 典型场景 |
            |--------|------|----------|
            | O(1) | 常数阶 | 数组按下标访问、哈希表查找（理想） |
            | O(log₂n) | 对数阶 | 二分查找、平衡BST查找 |
            | O(n) | 线性阶 | 顺序查找、遍历数组 |
            | O(n·log₂n) | 线性对数阶 | 快速排序、归并排序、堆排序（平均） |
            | O(n²) | 平方阶 | 冒泡排序、直接插入排序、简单选择排序 |
            | O(n³) | 立方阶 | 普通矩阵乘法 |
            | O(2ⁿ) | 指数阶 | 穷举法、递归求斐波那契（朴素） |
            | O(n!) | 阶乘阶 | 旅行商问题（穷举） |
            计算方法：
                找算法中执行次数最多的语句，计算这个语句执行的数量级，就只看这个，其他比他数量级低的语句不管
            常数项规则：只要代码中执行次数和n无关，无论执行多少次 结果都是O（1）
            加法规则：顺序执行的代码，总复杂度等于最复杂的那个， 一段代码前半部分 O (n)，后半部分 O (n²)，总复杂度为 O (n²)。
            乘法规则：嵌套循环的复杂度=内外循环的乘积。 外层循环 O (n)，内层循环 O (logn)，总复杂度为 O (nlogn)。
            对数阶 O (logn) 计算
                int i = 1;
                    while(i <= n){
                        i = i * 2; // 核心语句，执行次数最多
                    }
                在这里核心语句不是执行n次，i是成倍增长的，初始 i=1，每次乘 2，执行 k 次后，i=2ᵏ。当 2ᵏ > n 时，循环停止，解得 k = log₂n（以 2 为底 n 的对数）
                对数阶的底数无论为多少，统一写成 O (logn)
                只要循环变量是成倍增长 / 缩减，时间复杂度就是 O (logn)。例：i=i*3、i=i/2，都是 O (logn)。
        空间复杂度
            空间复杂度用来估算算法执行过程中，额外占用的存储空间随问题规模 n 增长的变化趋势,空间复杂度只算额外申请的存储空间，输入数据本身占用的空间不算！
            O(1)：常数阶，额外申请的空间固定，和 n 无关。这种算法也叫原地算法（例：冒泡排序，只需要几个临时变量，空间复杂度 O (1)）。
            O(n)：线性阶，额外申请的空间和 n 成正比（例：申请一个长度为 n 的辅助数组）。
            O(logn)：对数阶，常见于递归算法的调用栈空间（例：折半查找的递归实现）。
            O(n²)：平方阶，额外申请 n 行 n 列的二维数组。
            递归算法的空间复杂度 = 递归的深度 × 每次递归额外申请的空间。如果每次递归申请的空间是 O (1)，那么空间复杂度就等于递归的深度。
            斐波那契递归实现
                int fib(int n){
                        if(n <= 1) return n; // 递归终止条件
                        return fib(n-1) + fib(n-2); // 递归调用
                    }
            递归的深度为 n，每次递归只申请常数个变量，空间复杂度为O(n)。

线性表
    n个相同类型数据元素的有限序列
    特点：只有一个首元素，只有一个尾元素，除首尾外，每个元素只有一个直接前驱和一个直接后驱，元素个数n 为线性表的长度，n=0时称为空表
    线性表有两种存储方式：顺序存储（顺序表）和链式存储（链表）
    顺序表：用一组连续的存储单元顺序存放线性表中的元素，逻辑上相邻的元素，物理位置上也相邻
        求顺序表中某个元素的地址：首元素地址 + （i-1）*每个元素的大小
            顺序表首地址为 1000，每个整型元素占 4 字节，求第 8 个元素的地址。 LOC(a₈) = 1000 + (8 - 1) × 4 = 1000 + 28 = 1028
        插入操作：在地i个位置插入元素，从i~n后移一位，移动了 n-i+1 位个元素，时间复杂度为O（n）
        删除操作：将第i个元素删除，需要将 i+1~n的元素前移一位，移动了n-i位个元素，时间复杂度为O（n）
        顺序表操作时间复杂度：按下标访问：O（1），插入：O（n），删除：O（n），按值查找：O（n）
        插入操作平均移动了 n/2个元素
        删除操作平均移动了 （n-1）/2个元素
    链表：链表每个节点分为两个部分：数据域（存储数据）和指针域（存储下一个节点的地址）
        单链表：带头结点的好处：首个数据和其他数据节点的操作统一，删除整个链表的时候，头节点指向空即可
            常见操作时间复杂度：
                按位置访问第i个：O（n）
                头部插入/删除：O（1）
                尾部插入（无尾指针）：O(n)
                已知节点后插入：O（1）
                已知节点后删除：O（n）因为需要找前驱节点
        双链表：有两个指针，一个前驱指针，一个后驱指针，这样删除节点的时候时间复杂度就是O（1）
        循环链表：优点是从任意节点出发都能遍历整个链表
            循环单链表：尾节点的指针指向头节点，而不是NULL
            循环双链表：头节点的前驱指向尾节点，尾节点的后继指向头节点
        | 特性 | 顺序表 | 链表 |
        |------|--------|------|
        | 存储方式 | 连续内存 | 非连续内存 |
        | 随机访问 | **O(1)** | O(n) |
        | 插入/删除 | O(n)（移动元素） | **O(1)**（已知位置，改指针）|
        | 存储密度 | **高**（只存数据） | 低（有指针开销）|
        | 查找操作 | O(n) | O(n) |
        | 适用场景 | 频繁查询、少修改 | 频繁插入删除 |

栈和队列
    栈：栈是一种后进先出的数据结构，只有栈顶能够操作
        入栈：push   时间复杂度：O（1）
        出栈：pop    时间复杂度：O（1）
        取栈顶：peek 时间复杂度：O（1）
        应用场景：表达式求值：中缀表达式转后缀，后缀表达式计算
                 括号匹配，检查括号是否匹配
                 递归实现，函数调用使用系统栈
                 深度优先搜索，用栈模拟递归
                 进制转换，余数入栈，出栈得结果
        计算中缀转后缀
        顺序栈的溢出问题：
            上溢：栈满还push
            下溢：栈空还pop
    队列：队列是一种先进先出的数据结构，只能在在队头出队，在队尾入队
        循环队列
            入队：rear = (rear + 1) % MaxSize。
            出队：front = (front + 1) % MaxSize。
            队空 | `front == rear` | 队头等于队尾
            队满 | `(rear + 1) % MAXSIZE == front` | 牺牲一个存储位置来区分空和满
            队列中元素个数**：`(rear - front + MAXSIZE) % MAXSIZE`
        链队列
            入队：在链表尾部插入新结点，修改队尾指针。
            出队：删除链表头部的结点，修改队头指针。
            队空条件：队头和队尾都指向头结点。
            软考常考的队列应用场景：
                操作系统的作业调度、进程调度。
                缓冲区管理（例：打印队列、消息队列）。
                广度优先搜索（BFS）算法的实现。
                排队系统的模拟。

数组和矩阵
    数组是由 n (n≥1) 个相同类型的数据元素组成的有序序列，每个元素在数组中有唯一的下标，数组存储在连续的内存单元中，支持随机访问。
    二维数组分为行优先存储和列优先存储两种
    行优先存储：先存储第一行的所有元素，再存储第二行，以此类推，每行内按列顺序存储
        二维数组 A[m][n]（m 行 n 列，下标从 0 开始）：

            **行优先（C 语言、大多数语言）**：
            A[i][j] 的地址 = 基地址 + (i × n + j) × d
            **列优先（FORTRAN 语言）**：
            A[i][j] 的地址 = 基地址 + (j × m + i) × d
            其中 d 为每个元素所占字节数。
            **例题**：数组 A[0..4][0..5]（5行6列），首地址 1000，每元素 2 字节，行优先，求 A[2][3] 的地址。
            地址 = 1000 + (2 × 6 + 3) × 2 = 1000 + 15 × 2 = 1000 + 30 = 1030
            **答案：1030**
        以上公式要注意的是 数组中的m和n与计算时不一样，计算时的mn指的是数组的长度，所以是5和6
                         还有一点是，如果开始的下标是从1开始，比如A[1..4][1..5]，长度就是4和5，这时候求得地址的行列也需要减一 也就是 （1*5）+2

    特殊矩阵的压缩存储
        对称矩阵，三角矩阵，稀疏矩阵
        对称矩阵 ：长度n*(n+1)/2，一维数组存储  B [0...n (n+1)/2 - 1]
        三角矩阵 ：长度n*(n+1)/2，一维数组存储  B [0...n (n+1)/2 - 1]
        稀疏矩阵：三元组的结构：(i, j, value)，其中 i 是行号，j 是列号，value 是元素值，节省存储空间，但失去了随机访问的能力，只能顺序访问。

广义表
    就是list 能存储单个元素和另一个广义表
    表头，广义表的第一个元素就是表头，不管是单个元素还是另一个广义表，空表没有表头
    表尾，广义表除了表头之外，剩余元素组成的另一个广义表就是表尾，就算只有一个元素，也要用括号包起来，空表没有表尾
    深度，广义表的深度是广义表中所有括号的嵌套括号的最大嵌套，空表的深度是一，D = ((), (a,(b,c),d)) 深度是3

树和二叉树
    二叉树：每个节点最多有两个子树的有序树，左子树和右子树有区别
    特殊二叉树：满二叉树，完全二叉树，二叉排序树，平衡二叉树
    性质
        一：第i层，最多的节点不超过 2⁽ⁱ⁻¹⁾ 个节点
        二：高度为h的树，最多有2⁽ʰ⁻¹⁾个节点
        三：n₀ = n₂ + 1 ，叶子节点数等于 度为2的节点数+1
        四：有n个节点的完全二叉树，的高度为 ⌊log₂n⌋ + 1
        五：对完全二叉树的节点进行编号，从上到下，从左到右，从1开始。节点i的父节点i/2。节点i的左子节点2i，右节点2i+1。叶子节点的编号范围：n/2 + 1 ~~ n 
            完全二叉树节点数为偶数时，叶节点 = $n/2$；为奇数时，叶节点 = $(n+1)/2$
    二叉树的遍历
        先序遍历-根左右，中序遍历-左根右，后序遍历-左右根，层序遍历-从上到下，从左到右逐层遍历
        以下二叉树：
        ```
                A
               / \
              B   C
             / \   \
            D   E   F
        ```
        
        - **先序**：A → B → D → E → C → F
        - **中序**：D → B → E → A → C → F
        - **后序**：D → E → B → F → C → A
        - **层序**：A → B → C → D → E → F
        先序遍历的第一个是整个树的根节点
        后序遍历的最后一个节点是整个树的根节点
        会考核 ，给出两个遍历的序列，让求另一个，如
            **题干**：某二叉树的前序遍历序列为ABCDEFG，中序遍历序列为BDCAFEG，则该二叉树的后序遍历序列是（  ）
                    A. DCBFGEA
                    B. DCBGFEA
                    C. BCDFGEA
                    D. BDCFGEA

                    **答案**：A 
            解题时先根据前序或者后序 确定根节点，然后根据中序确定左右子树，循环这个操作，就能得到最终的树形图
    线索二叉树：n 个节点的二叉树有 n+1 个空指针域（浪费），利用这些空指针存储前驱/后继信息。，就是二叉树有一边可能为空，需要利用上
        - 若某节点**左子节点为空**，将左指针改为指向该节点的**中序前驱**
        - 若某节点**右子节点为空**，将右指针改为指向该节点的**中序后继**
    哈夫曼树（最优二叉树）
        - 哈夫曼树中**没有度为 1 的节点**（n₁ = 0）
        - n 个叶节点构造的哈夫曼树共有 **2n-1** 个节点
        - **权值越大的节点，路径越短**（越靠近根）
        wpl（带权路径长度）求的时候 可以用画树时，自己加出来的那个节点的和来验算
    二叉排序树
        对于每个节点，左子树所有节点的值小于当前节点的值
        对于每个节点，右子树所有节点的值大于当前节点的值
        如果用中序遍历，得到的是一个递增的有序序列
        插入，查找，删除的平均时间复杂度为O(log n)
        最坏情况时间复杂度为O（n）
        BST查找过程：查找值x，若树空，查找失败，若x=节点值，查找成功，若x>节点值，从左子树中查找，若x<节点值，从右子树查找
    平衡二叉树
        平衡因子：左子树高度-右子树高度
        要求平衡因子的绝对值小于等于1 也就是 平衡因子 = -1，0，1
        四种失衡：LL-左子树的左子树，RR-右子树的右子树，LR-左子树的右子树，RL-右子树的左子树
        解决：      单次右旋            单次左旋           先左旋后右旋        先右旋再左旋
    B树
       m阶的B树
        最多有m个子树和m-1个关键字
        根节点至少有两个非叶子节点的子树
        非根节点至少有m/2个子树
        所有叶子节点在同一层
        只要用于文件磁盘系统和数据库索引 
    B+树
        数据只存在所有叶子节点
        内部节点只存索引或者冗余键
        叶子节点形成有序链表
        擅长范围查询，B树不擅长范围查询
    一棵二叉树共有 60 个叶节点和 30 个度为 1 的节点，则该树共有（　　）个节点。

    A. 148　　B. 149　　C. 150　　D. 151

    **答案：B**

    **解析**：
    - 已知：n₀ = 60，n₁ = 30
    - 由性质 n₀ = n₂ + 1，得 n₂ = n₀ - 1 = 59
    - 总节点数 n = n₀ + n₁ + n₂ = 60 + 30 + 59 = **149**

    将序列 {10, 5, 15, 3, 7, 12, 20} 依次插入空的二叉排序树，以下说法正确的是（　　）

    A. 10 是树的根，5 是叶节点
    B. 该 BST 中序遍历结果为 3 5 7 10 12 15 20
    C. 15 的左子节点是 20
    D. 3 的父节点是 10

    **答案：B**

    **解析**：构造 BST：
    ```
            10
            /  \
        5    15
        / \  /  \
        3   7 12  20
    ```
    - A 错：5 有两个子节点 3 和 7，不是叶节点
    - B 正确：BST 中序遍历得到升序序列 3 5 7 10 12 15 20 ✓
    - C 错：15 的左子节点是 12（12 < 15），右子节点是 20
    - D 错：3 的父节点是 5，不是 10


图
    无向图 ：边没有方向，两个点之间只有一条
    有向图：边有方向，两个点之间最多两条
    完全无向图：任意两个点之间都有一条无向边。总边数公式：n*（n-1）/2 
    完全有向图：任意两个点之间都有两条方向相反的有向边。总边数公式：n*(n-1)
    顶点的度：
        和一个顶点有关联的边的数量
        无向图：和这个点有关联的边的数量
        有向图：分出度和入度
            出度：以这个点为起点的边的数量
            入读：已这个点为终点的边的数量
        核心公式：图的总度数=总边数*2
        有向图公式：出度之和=入度之和 =总边数
    【2022 年下半年软考真题】一个有向图的顶点数为 5，所有顶点的出度之和为 7，则该图的总边数为（ ）。
        A. 3 B. 5 C. 7 D. 14
        答案：C
    图的两种核心存储结构
        邻接矩阵：
            对于有n个顶点的图，用n*n的二维数组存储，这个数组称为邻接矩阵A
                无向图：顶点i,j有边，那数组中A[i][j]=1,A[j][i]=1,无边则为0，A[i][i]=0 也就是对角线为0，默认无自己指向自己的边
                有向图：顶点i，j有边，那数组中A[i][j]=1,无边则为0，A[i][i]=0 也就是对角线为0，默认无自己指向自己的边
                带权图：顶点i，j有边，且权值为w，那数组A[i][j]=w,无边则值为无穷大，默认65535，对角线为0
            特点：
                无向图的邻接矩阵为对称矩阵，边的个数 = 矩阵中1的数量/2
                有向图的邻接矩阵不一定为对称矩阵，边的个数 = 矩阵中1的数量
                无向图的i点度 = 矩阵中第i行或第i列的1的和
                有向图的i点度：出度 = 第i行1的和，入度 = 第i列1的和
            优缺点：
                优点：判断两个点之间有没有边，只需看A[i][j]的值即可，时间复杂度O（1）
                缺点：空间复杂度固定为n²,对于顶点多，边稀少的，浪费的空间多
        邻接表（数组+链表存储）：
            对于图中的每个顶点有个单链表存储 所有和这个点相连的点。
            所有顶点的邻接链表的头指针存在一个一维数组中，这个整体就是邻接表
                无向图：单链表存储顶点 i 所有相关的点
                    顶点i的度=单链表中节点的个数
                    总边数=所有单链表的节点数/2
                有向图：单链表存储顶点i，以顶点i为起点的有向边的终点
                    顶点i的出度=单链表中节点的个数
                    顶点i的入度=所有单链表中节点i出现的个数
                    总变数=所有链表的节点数
                优点：只存储有的边，节省空间
                缺点：判断两个点有没有边，需要遍历i的链表最坏的时间复杂度O（n），比邻接矩阵慢 
        【2024 年下半年软考真题】有一个有向图具有有序的拓扑排序序列，它的邻接矩阵必定为（ ）。
        A. 对称矩阵 B. 稀疏矩阵 C. 三角矩阵 D. 一般矩阵
        答案：C
        解析：拓扑排序的有向无环图，顶点按拓扑序列排序后，所有边都从序列前的顶点指向后的顶点，邻接矩阵只有上三角 / 下三角有非 0 元素，因此是三角矩阵。
        【2023 年上半年软考真题】对于 n 个顶点、e 条边的无向图，采用邻接表存储时，邻接表中节点的总数是（ ）。

        答案：C
        解析：无向图的每条边，会在两个顶点的链表里各存一次，因此总节点数是 2e。

        广度优先搜索适合解决最短路径问题（无权图中）
    图的两种遍历查找方式
        深度优先搜索
            访问起始顶点 v，标记visited[v]=1。
            找到 v 的第一个未被访问的邻接顶点 w，访问 w，标记visited[w]=1，以 w 为新起点，重复步骤 2（递归）。
            若 v 的所有邻接顶点都已访问，回退到上一个顶点，重复步骤 2。
            若为非连通图，遍历完一个连通分量后，找下一个未访问的顶点，重复上述步骤。
        广度优先搜索
            访问起始顶点 v，标记visited[v]=1，将 v 加入队列。
            队列不为空时，取出队首顶点 u。
            依次访问 u 的所有未被访问的邻接顶点 w，标记visited[w]=1，将 w 加入队列。
            重复步骤 2-3，直到队列为空。非连通图需遍历所有连通分量。
        无向图，顶点 0、1、2、3，边为 (0,1)、(0,2)、(1,3)、(2,3)，起始顶点为 0。
            深度优先：访问 0→访问 1→访问 3→访问 2
            广度优先：访问0-访问1-访问2-访问3
        非连通图的遍历次数：调用 DFS/BFS 的次数 = 连通分量的个数（高频考点）。
        时间复杂度：
        邻接表存储：O (n+e)（n 为顶点数，e 为边数）
        邻接矩阵存储：O (n²)
        核心考点与真题
            非连通图的遍历次数：调用 DFS/BFS 的次数 = 连通分量的个数（高频考点）。
            时间复杂度：
                邻接表存储：O (n+e)（n 为顶点数，e 为边数）
                邻接矩阵存储：O (n²)
        【2021 年下半年软考真题】对有 n 个顶点、e 条边的无向图进行深度优先遍历，算法的时间复杂度为（ ）。
        A. O (n) B. O (e) C. O (n+e) D. O (n×e)
        答案：C
        解析：邻接表存储的 DFS，每个顶点访问一次，每条边访问一次，时间复杂度为 O (n+e)，这是软考默认考察的场景。
        【2020 年下半年软考真题】一个非连通无向图有 3 个连通分量，遍历该图需要调用 DFS 的次数是（ ）。
        A. 1 B. 2 C. 3 D. 0
        答案：C
        解析：每个连通分量需要调用一次 DFS，3 个连通分量需要调用 3 次。

    最短路径
        Dijkstra算法（单源最短路径）
            从一个顶点出发，到其他所有顶点的最短路径。

            - 不能处理**负权边**
            - 时间复杂度：**O(n²)**

查找（检索）
    平均查找长度：在查找过程中需要和关键字比较的平均次数，衡量查找算法效率的核心标准
    ASL= ∑ⁿᵢ₌₁Pᵢ*Cᵢ
        ASL：平均查找长度
        n:查找表中的元素个数
        Pᵢ:查找到第i个元素的概率
        Cᵢ:查找到第i个元素的时候进行了多少次关键字比较
    公式分为查找成功和查找失败
    顺序查找：
        一个一个的比较，对是否有序，怎么存储都支持
        查找成功ASL （n+1）/2
        查找失败ASL n
        时间复杂度O（n）
    折半查找（二分查找）
        要求数据是顺序存储结构，且按照关键字有序排列
        时间复杂度：logⁿ 效率高于顺序查找
        有个需要注意的点 ，取中间值的时候 ，取到的中间值，还需要判断数据在左半段还是右半段，在左需要中间值减一，在右需要中间值加一
        树的深度：log₂ⁿ+1  n是元素个数
        ### 例1：在A[1..6]中查找7（大于所有元素）

        | 步骤 | low | high | mid | 比较 | 结果 | 新范围 |
        |------|-----|------|-----|------|------|--------|
        | 1 | 1 | 6 | 3 | A[3] | 7 > A[3] | low = 3+1 = 4 → [4,6] |
        | 2 | 4 | 6 | 5 | A[5] | 7 > A[5] | low = 5+1 = 6 → [6,6] |
        | 3 | 6 | 6 | 6 | A[6] | 7 > A[6] | low = 7, high = 6 → 结束 |

        **比较序列：A[3]、A[5]、A[6]**（共3次）

        ### 例2：在A[1..6]中查找1（在表中）

        | 步骤 | low | high | mid | 比较 | 结果 | 新范围 |
        |------|-----|------|-----|------|------|--------|
        | 1 | 1 | 6 | 3 | A[3] | 1 < A[3] | high = 3-1 = 2 → [1,2] |
        | 2 | 1 | 2 | 1 | A[1] | 1 = A[1] | 找到！ |

        **比较序列：A[3]、A[1]**（共2次）
    分块查找
        顺序查找和折半查找结合，将数据分块，块间有顺序，块内无序，块间使用折半或者顺序，块内使用顺序查找
        规则是前一个块的最大值小于后一个块的最小值
        时间复杂度在顺序查找O（n）和O（logⁿ）之间为O（√n）
    二叉排序树查找
        左子树的所有节点小于根节点，右子树的所有节点大于根节点
        查找：从根节点开始比较，如果一样就查找成功，如果查找值大于根节点就从右子树中递归查找，查找值小于根节点就从左子树中递归查找
        时间复杂度：logⁿ    
        二叉排序树的中序遍历结果是 有序序列
    平衡二叉树查找
        树种左子树和右子树的高度差的绝对值不会超过1，为了解决二叉排序树退化为单链表的情况
        查找方式和排序树一样
        时间复杂度稳定在logⁿ 不受插入顺序影响
    散列查找（哈希查找）-重点！
        算法是用要排序的数对给定散列表长度取余，但是会有余数相同的情况
        线性探测法（开放定址法）：就是如果被占用了 ，就向后探测一个，没占用就放置，占用就继续向后
        链地址法（拉链法）：把取余相同的关键字放在一个链表中，散列表的每个位置存储对应链表的头指针
        时间复杂度：理想情况下 O（1）
        ASL：
            查找成功的 ASL：成功所有关键字的查找次数之和关键字总个数
            查找失败的 ASL：失败所有散列地址的失败查找次数之和散列地址总个数

排序
    直接插入排序 -插入排序
        将待排序元素插入到已排序序列中的适当位置
            时间复杂度：最好O（n），平均/最坏O（n²）
            空间复杂第：O（1）
            稳定性好，适用于数据量小，基本有序的
    希尔排序 -插入排序
    简单选择排序 -选择排序
        每次从序列中取数据最小的 放在已排序序列的末尾
        时间复杂度：始终O（n²）
        不稳定
    堆排序 -选择排序 没学会！！！
        完全二叉树中 节点的值都大于或小于叶子节点的值
        大顶堆：节点的值大于叶子节点，用于升序排列
        小顶堆：节点的值小于叶子节点，用于降序排列
        完全二叉树是从上到下，从左到右编号，且除了叶子节点之外没有空余的，叶子节点都是靠左排列
        完全二叉树中，第i个节点，他的父节点是i/2 (向下取整)，他的左子节点为2i，右子节点为2i+1
        最后一个非叶子节点的编号是 n/2 n是总节点数
        步骤：
           1. 将无序序列构建成大顶堆
           2. 将堆顶元素（最大值）与末尾元素交换
           3. 调整剩余元素重新成为堆
           4. 重复2~3
        时间复杂度：O（nlogⁿ）
        空间复杂度：O（1）
        不稳定
    冒泡排序 -交换排序
        两两交换，大的向后冒泡
        时间复杂度：好的情况O（n），最坏情况是O（n²）
        空间复杂度：O（1）
        稳定性：稳定
        优化点：如果一趟排序中没有交换，说明数据已经是有序的了，提前结束
        时间复杂度：最好的情况是O（n），最坏和平均时间复杂度都是O（n²）
        空间复杂度：O（1）
        稳定性：稳定
        数组都是从下标为1开始，比较a[1]和a[2]比较，每次扫描到最后都会得到本次的最大值
    快速排序 -交换排序 ！！！重点
        时间复杂度：平均是O（nlogⁿ），最坏的情况是 ，序列本身有序，且以第一个元素为基准，O（n²）,和冒泡排序一致
        空间复杂度：最好和平均是O（logⁿ）,最坏是O（n）
        稳定性：不稳定
        适用于大规模，随机序列的，是工业界最广泛的排序算法
    归并排序    
        时间复杂度：都为O(nlogⁿ) 和数据初始顺序无关
        空间复杂度O（n）
        算法策略：分治法，快排也是分治法
        适合最坏情况有严格时间要求的，
        稳定性：稳定排序
        多路归并公式：拆成最小单元后，每次合并两个有序子数组，就叫两路归并，如果是一次合并m个有序数组，就叫m路归并
            元素个数：n，m路归并，需要k次，就有公式 mᵏ>=n(向上取整)
            比如对27个元素的序列，三路归并，就是 3ᵏ>=27 ,也就是log₃²⁷ 也就是3 需要三趟
        归并排序是稳定且最好最坏的时间复杂度都是O（nlogⁿ）
        归并排序的时间复杂度占用很高，是O（n）
    基数排序
        非比较型排序：不直接比较元素本身大小
        稳定性：稳定排序
        时间复杂度：好坏都是O（d（n+r）） d = 最大元素的位数，n = 元素总数，r = 基数（十进制 r=10）；d 为常数、r 远小于 n 时，可近似为 O (n)
        空间复杂度：O（n+r）
        适用于：元素为整数或固定长度字符串，手机号身份证号等

算法设计方法
    判断算法好坏的核心指标：时间复杂度，空间复杂度
        O（1）< O（logⁿ）< O(n) < O(nlogⁿ) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
    分治法：
        将复杂的大问题拆成若干个小问题，解决完所有小问题，大问题就解决了
        核心特点：
            大问题能拆成若干个结构相同的小问题
            小问题之间互相独立，互不影响（和动态规划的核心区别）
            子问题的解能和成大问题的解
            几乎都用递归实现
        经典应用：快速排序，归并排序，二分查找，汉诺塔问题
    动态规划：
        最长公共子序列的时间复杂度是O（mn），两层循环分别遍历两个串的长度
