一、零基础入门：什么是动态规划？

1.1 核心概念

动态规划（Dynamic Programming，简称 DP），本质是把一个复杂的大问题，拆成一堆小的、相似的子问题，然后把每个子问题的答案存起来，避免重复计算，最后用子问题的答案推导出大问题的答案。

举个最简单的例子：你要算斐波那契数列的第 10 项，F (10)=F (9)+F (8)，如果用纯递归，你会发现 F (8) 会被算两次，F (7) 会被算三次，大量重复工作。而动态规划就是，算完 F (8) 就把结果存起来，下次要用直接拿，不用再算一遍，这就省了大量时间。

1.2 动态规划的 3 个核心特点

一个问题能不能用动态规划解决，就看它有没有这三个特点：

1. 大问题拆成相似子问题：原问题和子问题的求解逻辑是一样的，比如算 F (10) 和算 F (9) 的逻辑完全一样。

2. 子问题只需要算一次：所有的子问题，我们算一遍就够了，不会重复算。

3. 存储子问题的解：用一个数组 / 表格把算过的子问题的答案存起来，下次直接用，这就是典型的「空间换时间」。

1.3 和其他算法的区别（软考上午题常考）

很多同学会把动态规划和分治、贪心搞混，这里给你讲清楚：

1.4 软考常考的动态规划经典模型

软考里的动态规划题，翻来覆去就是这几个模型，把这几个搞懂，所有题都能套：

1. 0-1 背包问题：最最常考，下午题考过 N 次，就是物品选或不选，求最大价值。

2. 编辑距离问题：两个字符串的最少操作次数，下午题也考过。

3. 矩阵链乘法：多个矩阵相乘，怎么加括号让乘法次数最少。

4. 最长公共子序列（LCS）：两个字符串的最长公共子串的长度。

5. 钢条切割问题：把长钢条切成短的，卖最多的钱。

6. 凸多边形最优三角剖分：几何类的 DP，考过一次。

7. RNA 二级结构配对：生物信息的 DP，考过一次。

真题 1：2025 年上半年真题

题目：以下关于动态规划算法的描述，正确的是（）
A. 动态规划算法仅能解决最优化问题
B. 核心思想是将大问题分解为小问题，重复利用小问题的解
C. 动态规划算法的时间复杂度一定低于贪心算法
D. 无需存储小问题的解，直接递归求解即可

答案：B

解析：

- A 错：动态规划不只能解决最优化问题，也可以解决计数等其他问题，只是软考里常考最优化。

- B 对：这就是动态规划的核心，拆子问题，存子问题解，重复利用。

- C 错：不一定，比如贪心的时间复杂度经常是 O (n)，比动态规划的 O (n²) 低多了。

- D 错：动态规划就是要存小问题的解，不然就是纯递归了，会重复计算。

真题 3：经典概念题

题目：关于 0-1 背包问题，以下说法正确的是？
A. 可以用贪心算法得到最优解
B. 可以用动态规划算法得到最优解
C. 时间复杂度是 O (n)
D. 每个物品可以拿多次

答案：B

解析：

- A 错：0-1 背包不能用贪心，贪心只能解决部分背包（物品可以拆分）的问题，0-1 的物品不能拆分，贪心会出错。

- B 对：0-1 背包是动态规划的经典应用，能得到全局最优。

- C 错：0-1 背包的时间复杂度是 O (n*W)，n 是物品数，W 是背包容量。

- D 错：0-1 背包的每个物品只能拿一次，能拿多次的是完全背包。

真题 1：2019 下半年 0-1 背包问题（最经典考点）

问题背景

0-1 背包：有 N 个物品，每个物品有重量 w [i] 和价值 v [i]，背包最大容量是 T，每个物品最多拿一次，问怎么选能让背包里的总价值最大？这是软考下午题考的最多的 DP 问题。

题目说明

我们有 5 个物品，背包容量 T，要算最大能装的价值。这里用的是「自顶向下 + 记忆化」的动态规划写法，也就是递归的时候把算过的子问题存起来，避免重复算。

C 语言代码

#include<stdio.h>
#include<math.h>
#define N 6
#define maxT 1000
int c[N][maxT]={0};

int Memoized_Knapsack(int v[N], int w[N], int T){
    int i; int j;
    for(i = 0;i< N;i++){
        for(j = 0;j <= T;j++){
            c[i][j] = -1; // 初始化为-1，表示这个子问题还没算过
        }
    }
    return Calculate_Max_Value(v,w,N-1, T);
}

int Calculate_Max_Value(int v[N], int w[N],int i,int j){
    int temp = 0;
    if(c[i][j] != -1){ 
        return (1) ; // 如果已经算过了，直接返回
    }
    if(i == 0 || j == 0){ // 边界：没有物品了，或者背包满了，价值是0
        c[i][j] = 0;
    }else{
        // 第一种情况：不拿第i个物品，那结果就是前i-1个物品，容量j的最大价值
        c[i][j] = Calculate_Max_Value(v,w,i-1,j);
        if( (2) ){ // 第二种情况：能不能拿第i个物品？要看重量够不够
            temp = (3); // 拿的话，剩下的容量是j-w[i]，前i-1个物品的最大价值，加当前物品的价值
            if(c[i][j] < temp){ 
                (4); // 如果拿的方案价值更大，就更新结果
            }
        }
    }
    return c[i][j];
}

问题 1：填充代码中的空 (1)-(4)

答案：
(1) c[i][j]
(2) w[i] <= j
(3) Calculate_Max_Value(v,w,i-1,j-w[i]) + v[i]
(4) c[i][j] = temp

解析：

- 空 (1)：c [i][j] 存的就是子问题「前 i 个物品，容量 j 的最大价值」的答案，如果已经算过了（不是 - 1），直接返回这个结果就行，不用再递归了，这就是记忆化的核心。

- 空 (2)：要拿第 i 个物品，首先得保证这个物品的重量 w [i]，小于等于背包当前的剩余容量 j，不然装不下，所以判断 w [i] <= j。

- 空 (3)：如果拿了第 i 个物品，那背包剩下的容量就是 j - w [i]，这时候我们只需要算前 i-1 个物品，在剩余容量下的最大价值，然后加上当前物品的价值 v [i]，就是拿这个物品的总价值。

- 空 (4)：如果拿这个物品的总价值 temp，比不拿的价值 c [i][j] 更大，那我们就更新 c [i][j] 为 temp，选更好的方案。

问题 2：算法策略和实现方式

问题：该算法采用了\\(5)设计策略。在求解过程中，采用了(6)\\（自底向上或者自顶向下）的方式。

答案：
(5) 动态规划
(6) 自顶向下

解析：

- 这个问题有最优子结构（大问题的最优解包含子问题的最优解），还有重叠子问题，所以用的是动态规划。

- 自顶向下：就是从大问题开始，递归拆成小问题，比如我们一开始要算的是 N-1 个物品、容量 T 的大问题，然后拆成更小的子问题，这就是自顶向下。

- 对应的，自底向上是从最小的子问题开始，用循环一步步推到大问题，比如我们平时写的二维数组循环填表的 01 背包，就是自底向上。

真题 2：2021 下半年 编辑距离问题

问题背景

编辑距离：把字符串 A 变成字符串 B，最少需要多少次操作？操作有三种：

1. 删除：删掉 A 里的一个字符

2. 插入：在 A 里插入一个字符

3. 替换：把 A 里的一个字符换成另一个

比如把 "CTGA" 变成 "ACGCTA"，最少要几次操作？这就是编辑距离，用来算字符串相似度的。

题目说明

给定两个字符串 A 和 B，求它们的编辑距离。

C 语言代码

#include <stdio.h>
#define N 100
char A[N] = "CTGA";
char B[N] = "ACGCTA";
int d[N][N]; // d[i][j]表示：A的前i个字符，变成B的前j个字符，需要的最少操作次数

int min(int a, int b) { return a < b ? a : b; }

int editdistance(char *str1, int len1, char *str2, int len2) {
    int i, j;
    int diff;
    int temp;
    // 初始化：如果B是空字符串，把A的前i个字符删掉就行，要i次操作
    for (i = 0; i <= len1; i++) {
        d[i][0] = i;
    }
    // 初始化：如果A是空字符串，要插入j个字符，才能变成B的前j个，要j次操作
    for (j = 0; j <= len2; j++) {
        (1) ;
    }
    // 遍历A的每个字符
    for (i = 1; i <= len1; i++) {
        // 遍历B的每个字符
        for (j = 1; j <= len2; j++) {
            if ( (2) ) { 
                // 如果当前两个字符相等，那不需要操作，直接继承之前的结果
                d[i][j] = d[i - 1][j - 1];
            } else {
                // 不相等的话，有三种操作，选最小的
                // 1. 删除A的当前字符：d[i-1][j] +1
                // 2. 插入一个字符到A：d[i][j-1] +1
                temp = min(d[i - 1][j] + 1, d[i][j - 1] + 1);
                // 3. 替换A的当前字符：d[i-1][j-1] +1
                d[i][j] = min(temp, (3) );
            }
        }
    }
    return (4) ;
}

问题 1：填充代码中的空 (1)-(4)

答案：
(1) d[0][j] = j
(2) str1[i - 1] == str2[j - 1]
(3) d[i - 1][j - 1] + 1
(4) d[len1][len2]

解析：

- 空 (1)：初始化，A 是空的时候，要变成 B 的前 j 个字符，需要 j 次插入，所以 d [0][j] = j。

- 空 (2)：判断当前的两个字符是不是相等，注意：dp 里的 i 是 A 的前 i 个字符，对应字符串的下标是 i-1（因为字符串从 0 开始），j 同理，所以是 str1 [i-1] == str2 [j-1]。

- 空 (3)：替换操作的代价，就是把 A 的当前字符换成 B 的，那之前的子问题是 A 的前 i-1 个变成 B 的前 j-1 个，加 1 次替换操作，所以是 d [i-1][j-1]+1。

- 空 (4)：最终的结果，就是 A 的全部 len1 个字符，变成 B 的全部 len2 个字符的操作次数，也就是 d [len1][len2]。

问题 2：算法策略和复杂度

问题：该算法采用了\\(5)设计策略，时间复杂度为(6)\\（用 O 符号表示，两个字符串的长度分别用 m 和 n 表示）。

答案：
(5) 动态规划法
(6) O (mn)

解析：
编辑距离有最优子结构，子问题重叠，所以是动态规划。两层循环，分别遍历 m 和 n，所以时间复杂度是 O (mn)。

真题 3：2022 上半年 矩阵链乘法问题

问题背景

矩阵链乘法：我们有 n 个矩阵，A1,A2,...An，要把它们乘起来，矩阵乘法满足结合律，所以加括号的顺序不一样，乘法的次数会差很多！比如 A1 是 155，A2 是 510，A3 是 1020，那 (A1A2) A3 的乘法次数是 15510 +151020=3750，而 A1 (A2A3) 是 51020+155*20=2500，差了 1250 次！所以我们要找最优的加括号顺序，让总乘法次数最少，这就是矩阵链乘法，典型的 DP 问题。

题目说明

给定 n 个矩阵的维数序列，求最优的乘法顺序，最少的乘法次数。

C 语言代码

#define N 100
int cost[N][N]; // cost[i][j]表示：从第i个矩阵到第j个矩阵，相乘的最小乘法次数
int trace[N][N]; // 记录分割点，用来输出最优顺序

int cmm(int n, int seq[]) {
    int tempCost;
    int tempTrace;
    int i, j, k, p;
    int temp;
    // 初始化：单个矩阵，不需要乘，次数是0
    for (i = 0; i < n; i++) {
        cost[i][i] = 0;
    }
    // p是链的长度，也就是我们算的子问题的规模，从2个矩阵开始，一直到n个
    for (p = 1; p < n; p++) {
        // i是子问题的起始矩阵
        for (i = 0; i < n - p; i++) {
            // j是子问题的结束矩阵
            j = (1) ;
            tempCost = -1;
            // 遍历分割点k，把i~j分成i~k和k+1~j
            for (k = (2) ; k < j; k++) {
                // 总次数：i~k的次数 + k+1~j的次数 + 两个结果矩阵相乘的次数
                temp = (3) ;
                if (tempCost == -1 || tempCost > temp) {
                    tempCost = temp;
                    tempTrace = k;
                }
            }
            cost[i][j] = tempCost;
            trace[i][j] = (4) ;
        }
    }
    return cost[0][n - 1];
}

问题 1：填充代码中的空 (1)-(4)

答案：
(1) i + p
(2) i
(3) cost[i][k] + cost[k + 1][j] + seq[i] * seq[k + 1] * seq[j + 1]
(4) tempTrace

解析：

- 空 (1)：p 是子问题的长度，比如 p=1，就是两个矩阵，i 是起始，那结束 j 就是 i+p，比如 i=0，p=1，j=1，就是 A0 到 A1 两个矩阵。

- 空 (2)：分割点 k，从 i 开始，到 j-1 结束，所以 k 的初始值是 i。

- 空 (3)：总乘法次数，i 到 k 的最小次数，加 k+1 到 j 的最小次数，然后两个结果矩阵相乘的次数：i~k 的结果矩阵是 seq [i] * seq [k+1]，k+1~j 的是 seq [k+1] * seq [j+1]，所以相乘的次数就是 seq [i] * seq [k+1] * seq [j+1]。

- 空 (4)：把最优的分割点 k 存到 trace 里，就是 tempTrace。

问题 2：算法策略和复杂度

问题：该问题采用了\\(5)算法设计策略，时间复杂度为(6)\\（用 O 符号表示）。

答案：
(5) 动态规划
(6) O (n³)

解析：
三重循环，p、i、k，所以时间复杂度是 O (n³)，这是矩阵链乘法的标准复杂度。

问题 3：实例计算

问题：考虑实例 n=4，各个矩阵的维数：A1 为 155，A2 为 510，A3 为 1020，A4 为 2025，即维数序列为 15,5,10,20,25。则根据上述 C 代码得到的一个最优计算顺序为\\(7)，所需要的乘法运算次数为(8)\\。

答案：
(7) A1 * ((A2 * A3) * A4)
(8) 5375

手动验算过程：

真题 4：2021 上半年 凸多边形最优三角剖分

问题背景

凸多边形最优三角剖分：给一个凸多边形，顶点是 V0,V1,...Vn-1，我们要把它切成 n-2 个三角形，切的时候只能连不相邻的顶点，问怎么切，能让所有三角形的权重和最小？权重是三角形三个顶点之间的权重和，这也是一个 DP 问题，和矩阵链乘法的逻辑几乎一模一样。

C 语言代码

#include<stdio.h>
#define N 6 // 凸多边形规模
int m[N + 1][N + 1]; // m[i][j]表示多边形Vi-1到Vj最优三角剖分的权值
int S[N + 1][N + 1]; // 记录分割点
int W[N + 1][N + 1]; // 权重矩阵

/* 三角形的权重 */
int get_triangle_weight(int a, int b, int c) {
    return W[a][b] + W[b][c] + W[c][a];
}

void triangle_partition() {
    int i, r, k, j;
    int temp;
    /* 初始化：只有一个顶点，权值是0 */
    for (i = 1; i <= N; i++) {
        m[i][i] = 0;
    }
    /* r是子问题的规模，也就是子多边形的顶点数，从2开始 */
    for (r = 2; (1) ; r++) {
        /* i是子多边形的起始顶点 */
        for (i = 1; i <= N - r + 1; i++) {
            // j是子多边形的结束顶点
            (2) ;
            // 初始的分割点是k=i
            m[i][j] = m[i][i] + m[i + 1][j] + get_triangle_weight(i - 1, i, j);
            S[i][j] = i;
            // 遍历其他分割点k
            for (k = i + 1; k < j; k++) {
                temp = m[i][k] + m[k + 1][j] + get_triangle_weight(i - 1, k, j);
                if ( (3) ) { // 找最小的权值
                    m[i][j] = temp;
                    S[i][j] = k;
                }
            }
        }
    }
}

/* 输出剖分的三角形 */
void print_triangle(int i, int j) {
    if (i == j) return;
    print_triangle(i, S[i][j]);
    print_triangle( (4) );
    printf("V%d--V%d--V%d\n", i - 1, S[i][j], j);
}

问题 1：填充代码中的空 (1)-(4)

答案：
(1) r <= N
(2) j = i + r - 1
(3) temp < m[i][j]
(4) S[i][j] + 1, j

解析：

- 空 (1)：r 是子多边形的顶点数，最大到 N，也就是整个多边形，所以循环到 r<=N。

- 空 (2)：i 是起始，r 是顶点数，所以结束 j 就是 i + r -1，比如 i=1，r=2，j=2，就是两个顶点的子多边形。

- 空 (3)：如果当前的 temp 比之前的 m [i][j] 更小，那就更新，所以判断 temp < m [i][j]。

- 空 (4)：递归输出，分割点是 S [i][j]，所以右边的子问题就是 S [i][j]+1 到 j，所以递归调用 print_triangle (S [i][j]+1, j)。

问题：该算法采用的设计策略为\\(5)。算法的时间复杂度为(6)，空间复杂度为(7)\\。

答案：
(5) 动态规划
(6) O (n³)
(7) O (n²)

解析：
和矩阵链乘法一样，三重循环，时间复杂度 O (n³)，二维数组存结果，空间 O (n²)。

问题背景

钢条切割：给一个长度为 n 的钢条，不同长度的钢条价格不一样，比如长度 1 卖 1，长度 2 卖 5，问怎么切这个钢条，能卖最多的钱？这也是经典的 DP 问题。

C 语言代码

问题 1：填充代码中的空 (1)-(4)

答案：

(2) i <= j
(3) temp >= p[i] + r[j - i] ? temp : p[i] + r[j - i]
