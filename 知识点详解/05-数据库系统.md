# 第六章 数据库系统（重点章节）

## 6.1 数据库基础

### 6.1.1 数据库系统三级模式结构

**知识点**:
- **外模式（用户模式、子模式）**：
  - 用户看到的数据视图
  - 一个数据库可以有多个外模式

- **模式（概念模式、逻辑模式）**：
  - 数据库的整体逻辑结构
  - 一个数据库只有一个模式

- **内模式（存储模式、物理模式）**：
  - 数据的物理存储结构
  - 一个数据库只有一个内模式

- **两级映像**：
  - 外模式/模式映像：保证数据的逻辑独立性
  - 模式/内模式映像：保证数据的物理独立性

### 6.1.2 数据模型

**知识点**:
1. **层次模型**：
   - 树形结构
   - 每个节点最多一个父节点
   - 例如：IMS

2. **网状模型**：
   - 图形结构
   - 节点可以有多个父节点
   - 例如：DBTG

3. **关系模型**：
   - 二维表结构
   - 最常用
   - 例如：Oracle、MySQL、SQL Server

### 6.1.3 关系代数

**知识点**:
- **选择（σ）**：选择满足条件的元组（行）
- **投影（π）**：选择指定的属性（列）
- **并（∪）**：两个关系的并集
- **交（∩）**：两个关系的交集
- **差（-）**：两个关系的差集
- **笛卡尔积（×）**：两个关系的笛卡尔积
- **连接（⋈）**：
  - 等值连接：连接条件为相等
  - 自然连接：去掉重复列的等值连接

## 6.2 关系数据库设计理论

### 6.2.1 函数依赖

**知识点**:
- **函数依赖（FD）**：X → Y
  - 若X的值能唯一确定Y的值，则Y函数依赖于X

- **完全函数依赖**：Y完全依赖于X，不依赖于X的任何真子集
- **部分函数依赖**：Y依赖于X的某个真子集
- **传递函数依赖**：X → Y，Y → Z，则Z传递依赖于X

C. 传递函数依赖于
D. 不依赖于

**示例题8**:
设关系模式R(A,B,C,D)，函数依赖集F={(A,B)→C, (A,B)→D, A→B}，则C对(A,B)是( )。
A. 完全函数依赖
B. 部分函数依赖
C. 传递函数依赖
D. 多值依赖

### 6.2.2 范式

**知识点**:
1. **第一范式（1NF）**：
   - 所有属性都是原子的（不可分）
   - 关系数据库的最基本要求

2. **第二范式（2NF）**：
   - 满足1NF
   - 非主属性完全函数依赖于码

3. **第三范式（3NF）**：
   - 满足2NF
   - 非主属性不传递依赖于码

4. **BCNF（巴斯-科德范式）**：
   - 满足3NF
   - 主属性也不传递依赖于码
   - 或：所有决定因素都包含码

**知识点**:
- **无损连接分解**：分解后的关系通过自然连接能恢复原关系
- **保持函数依赖分解**：分解后的关系保留了原有的函数依赖

### 6.4.1 ER模型到关系模式的转换

   - 可以合并到任一实体的关系中
   - 或单独建立关系

   - 在n端实体的关系中加入1端实体的主码作为外码

4. **m:n联系**：

   - 包含两个实体的主码作为外码
   - 两个主码的组合作为主码

## 6.5 事务管理

### 6.5.1 事务的ACID特性

**知识点**:
- **原子性（Atomicity）**：事务是不可分割的单位
- **一致性（Consistency）**：事务执行前后数据库保持一致状态
- **隔离性（Isolation）**：并发事务互不干扰
- **持久性（Durability）**：事务提交后永久保存

### 6.5.2 并发控制

**知识点**:
- **并发问题**：
  1. 丢失修改：两个事务同时修改同一数据
  2. 不可重复读：事务两次读取的数据不一致
  3. 脏读：读取未提交的数据
  4. 幻读：两次查询结果集不同

- **封锁协议**：
  - **一级封锁协议**：事务修改数据前加写锁，防止丢失修改
  - **二级封锁协议**：在一级基础上，读数据前加读锁，读完释放，防止脏读
  - **三级封锁协议**：在一级基础上，读数据前加读锁，事务结束释放，防止不可重复读

- **两段锁协议**：
  - 加锁阶段：只加锁，不释放锁
  - 解锁阶段：只释放锁，不加锁
  - 可以保证串行化
