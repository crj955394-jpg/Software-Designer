我检索到的 2025 年《软件设计师》考试大纲里，明确要求掌握“计算机内的数据表示”“算术和逻辑运算方法”“编码基础”等内容，所以**校验码**（奇偶校验码、海明码、CRC）就是软考里非常典型的一块高频考点。([educity.cn][1])

先抓住底层思路：**校验码本质上是在原始信息里增加冗余位**，目的是做检错，进一步还能纠错；**码距**则是任意两个合法码字之间最少需要改动的二进制位数，码距越大，检错和纠错能力越强。软考里常用的判断规则是：要检测 `e` 个误码，最小码距要满足 `d >= e + 1`；要纠正 `t` 个误码，最小码距要满足 `d >= 2t + 1`。([阿里云开发者社区][2])

## 1）奇偶校验码：最简单，但只能“查错”，不能“改错”

奇偶校验码就是在数据前或数据后加 1 位校验位，让整个码字里的 `1` 的个数变成**奇数**（奇校验）或者**偶数**（偶校验）；它的码距是 2，所以只能发现错误，不能定位错误位置，因此**不能纠错**。软考题里也常用“如果只有一位错误”的前提来讨论它。([阿里云开发者社区][3])

### 例子：对 `1011` 做奇校验、偶校验

`1011` 里有 3 个 `1`。
奇校验要求总 `1` 的个数仍然是奇数，所以校验位取 `0`；偶校验要求总 `1` 的个数变成偶数，所以校验位取 `1`。如果校验位放在末尾，结果分别就是 `10110` 和 `10111`；如果题目要求放在开头，位置换一下即可。([阿里云开发者社区][3])

### 软考里奇偶校验最常考什么

1. 看原始数据中有几个 `1`。
2. 判断是奇校验还是偶校验。
3. 写出应该补的那一位。
4. 判断“能否检错/能否纠错”。
   奇偶校验的标准答案通常就是：**能检错，不能纠错**。([阿里云开发者社区][3])

## 2）海明码：软考里最爱考的“能纠错”校验码

海明码是在数据位里插入若干校验位，常见位置是 `1、2、4、8...` 这些 2 的幂次位置；它的核心价值是：**能定位出错位，从而纠正单个错误**。软考教材和讲义里常用的校验位数量公式是：`2^k >= n + k + 1`（`n` 是信息位个数，`k` 是校验位个数）。([阿里云开发者社区][2])

海明码的纠错思路非常关键：每个校验位负责检查一组特定位置；当接收端重新计算这些校验组时，把“是否出错”的结果按约定顺序拼起来，就能直接得到**错误位置的二进制编号**，这就是海明码能纠错的原因。教材还明确写到：海明码能做单比特纠错；如果出现两位错误，通常只能检出，不能纠正。([Scribd][4])

### 例子：4 位信息 `1011`，求海明码

4 位信息要满足 `2^k >= 4 + k + 1`，最小 `k = 3`，所以要插入 3 位校验位，码长变成 7 位。软考常见放法是把校验位放在 `1、2、4` 位，数据位放在 `3、5、6、7` 位；按常见偶校验规则计算后，结果是 `1010101`。这个例题在软考资料里是经典样板题。([阿里云开发者社区][2])

### 例子：收到 `1011101`，怎么纠错

如果收到的码字是 `1011101`，重新代入海明校验组去算，会发现只有一组校验结果不为 0，这说明出错的是某个单独位置；在这类题里，把“哪几个校验组失败”按规定顺序拼成二进制数，就能直接定位错误位，然后把那一位翻转回来。教材的示例里，正是用这种方式把错误定位到校验位并恢复原码。([阿里云开发者社区][2])

### 海明码题目最常见的两种问法

1. **求校验位个数**：先套 `2^k >= n + k + 1`。
2. **求完整码字/纠错位**：先按 `1、2、4...` 插位，再按各组做异或或奇偶计算。
   这两类题几乎就是软考里海明码的核心出题方式。([阿里云开发者社区][2])

## 3）CRC：软考里最像“计算题”的校验码

CRC（循环冗余校验）是把信息码看成一个二进制多项式，再用**生成多项式**去做**模 2 除法**；模 2 除法里减法就是异或，所以 CRC 题的本质其实就是“按位异或除法”。编码时要先在信息码后面补 `R` 个 0，再除以生成多项式，最后得到的余数就是 CRC 校验码；接收端再除一次，如果余数为 0，通常认为数据正确。([Scribd][4])

教材还强调了一个很重要的点：CRC 的生成多项式通常要求**最高位和最低位都是 1**；在软考场景里，CRC更偏向**检错**，而不是纠错。它的检测能力很强，资料里常见的总结是：可以检测所有双错、奇数位错，以及长度不超过 `R` 的突发错误。([Scribd][4])

### 例子：原始报文 `11001010101`，生成多项式 `x^4 + x^3 + x + 1`

这个多项式对应二进制除数 `11011`。因为最高次数是 4，所以先在原始报文后补 4 个 0，变成 `110010101010000`，再做模 2 除法；教材示例的余数是 `0011`，所以最终 CRC 编码结果是 `110010101010011`。([阿里云开发者社区][5])

## 4）软考里最常见的“秒题模板”

**看到“奇偶校验”**：先数 `1` 的个数，再决定校验位是 0 还是 1；记住它只能检错，不能纠错。([阿里云开发者社区][3])

**看到“海明码”**：先算校验位个数 `k`，再把校验位放在 `1、2、4、8...` 位；最后按校验组求异或，错误定位直接看综合症。([阿里云开发者社区][2])

**看到“CRC”**：先把生成多项式转成二进制，再在信息位后补 `R` 个 0，做模 2 除法，取余数。([阿里云开发者社区][5])

**看到“码距”**：先想它和检错纠错的关系：检测 `e` 个误码要 `d >= e + 1`，纠正 `t` 个误码要 `d >= 2t + 1`。([CSDN博客][6])

## 5）你复习时最该死记的结论

奇偶校验码：**简单，码距 2，只能检错**。
海明码：**能纠错，尤其是单比特纠错，关键公式是 `2^k >= n + k + 1`**。
CRC：**强检错，靠生成多项式和模 2 除法，软考里通常按“检错”记**。([阿里云开发者社区][3])
