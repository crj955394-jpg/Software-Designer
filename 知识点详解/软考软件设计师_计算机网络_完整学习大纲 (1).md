## 第一章 计算机网络概述（★★ 基础）

### 1.1 计算机网络的定义与功能

**教材定义**：
> 计算机网络是利用通信设备和线路将地理位置分散的、功能独立的多个计算机系统互连起来，以功能完善的网络软件实现网络中资源共享和信息传递的系统。

**核心功能**（教材）：

| 功能 | 说明 |
|------|------|
| **数据通信** | 实现计算机之间的数据传输 |
| **资源共享** | 共享硬件、软件和数据资源 |
| **提高可靠性** | 通过冗余提高系统的可靠性 |
| **负载均衡** | 将任务分配到多个计算机上 |
| **分布式处理** | 将复杂任务分布到多台计算机处理 |

### 1.2 计算机网络的分类

**按分布范围**（★★ 常考）：

| 类型 | 范围 | 典型例子 |
|------|------|----------|
| **局域网（LAN）** | 几米到几公里 | 以太网、校园网 |
| **城域网（MAN）** | 一个城市范围 | 城市宽带网 |
| **广域网（WAN）** | 跨城市、跨国 | Internet |

**按拓扑结构**（★★ 常考）：

| 拓扑 | 特点 | 优缺点 |
|------|------|--------|
| **总线型** | 所有节点挂在一条总线上 | 优：简单、易扩展；缺：总线故障全网瘫痪 |
| **星型** | 所有节点连接到中心节点 | 优：管理方便；缺：中心节点故障则全网瘫痪 |
| **环型** | 节点首尾相连成环 | 优：延迟确定；缺：某节点故障影响全环 |
| **树型** | 层次化结构 | 优：扩展灵活；缺：根节点是瓶颈 |
| **网状型** | 节点间有多条路径 | 优：可靠性高；缺：管理复杂、成本高 |

**按交换技术**（★★★ 必考）：

| 交换方式 | 原理 | 特点 |
|----------|------|------|
| **电路交换** | 通信前建立专用物理通路（如电话网） | 实时性好、独占资源、无存储转发延迟 |
| **报文交换** | 整个报文存储转发 | 不独占线路、延迟大、对存储要求高 |
| **分组交换** | 将报文分成若干分组独立传输 | 综合优点、Internet 采用 |

> 🎯 **分组交换两种方式**（★ 常考）：
> - **数据报方式**：每个分组独立路由、可乱序到达（IP 协议采用）
> - **虚电路方式**：先建立虚电路、分组按序到达（ATM 采用）

### 1.3 网络性能指标

| 指标 | 定义 | 单位 |
|------|------|------|
| **带宽** | 信道的最大数据传输速率 | bps（比特每秒） |
| **时延** | 数据从发送方到接收方的总时间 | ms |
| **吞吐量** | 单位时间内通过网络的实际数据量 | bps |
| **信道利用率** | 信道被利用的时间比例 | % |

**时延的组成**：
```
总时延 = 发送时延 + 传播时延 + 处理时延 + 排队时延
```
- **发送时延** = 数据块长度 / 信道带宽
- **传播时延** = 信道长度 / 信号传播速度

### 1.4 网络规划与设计（★ 了解）

**分层设计**（教材）：

| 层次 | 功能 |
|------|------|
| **核心层** | 高速数据交换，提供骨干传输 |
| **汇聚层** | 流量汇聚、策略实施、VLAN 间路由 |
| **接入层** | 用户接入、提供端口和安全控制 |

## 第二章 网络体系结构（★★★ 核心重点）

### 2.1 OSI/RM 七层参考模型

**教材定义**：
> OSI/RM（Open Systems Interconnection Reference Model，开放系统互连参考模型）是国际标准化组织（ISO）制定的网络通信的七层框架模型。

**七层功能详解**（★★★ 必背表格）：

| 层次 | 名称 | 功能 | 典型协议/设备 | 数据单位 |
|------|------|------|---------------|----------|
| **7** | **应用层** | 为用户提供网络服务接口 | HTTP、FTP、SMTP、DNS、Telnet | 数据 |
| **6** | **表示层** | 数据格式转换、加密解密、压缩解压 | JPEG、ASCII、SSL | 数据 |
| **5** | **会话层** | 建立、管理、终止会话 | RPC、SQL、NFS | 数据 |
| **4** | **传输层** | 端到端的可靠/不可靠传输、流量控制 | TCP、UDP | 报文段 |
| **3** | **网络层** | 路由选择、逻辑寻址、拥塞控制 | IP、ICMP、ARP、OSPF、BGP | 分组/包 |
| **2** | **数据链路层** | 帧传输、差错控制、流量控制 | PPP、以太网、帧中继 | 帧 |
| **1** | **物理层** | 比特流传输、电气特性、物理连接 | RJ-45、光纤、集线器 | 比特 |

> 🎯 **记忆口诀**（从上到下）：应表会传网数物
> 或反过来（从下到上）：物数网传会表应

### 2.2 TCP/IP 参考模型

**教材定义**：
> TCP/IP 参考模型是 Internet 的基础，由四层组成。

**TCP/IP 四层模型**：

| 层次 | 对应 OSI | 主要协议 |
|------|----------|----------|
| **应用层** | 应用层+表示层+会话层 | HTTP、FTP、SMTP、DNS、Telnet、SNMP |
| **传输层** | 传输层 | TCP、UDP |
| **网际层（网络层）** | 网络层 | IP、ICMP、ARP、RARP |
| **网络接口层（主机-网络层）** | 数据链路层+物理层 | Ethernet、PPP、ATM |

**OSI 与 TCP/IP 对比**（★ 常考）：

| 对比项 | OSI | TCP/IP |
|--------|-----|--------|
| 层数 | 7 层 | 4 层 |
| 设计思想 | 先有模型再有协议 | 先有协议再有模型 |
| 应用范围 | 理论参考 | 实际标准 |
| 会话层/表示层 | 有 | 无（合并到应用层） |

### 2.3 各层典型协议与设备对照（★★★ 必背）

| 网络设备 | 工作层次 | 功能 |
|----------|----------|------|
| **中继器（Repeater）** | 物理层 | 信号放大和再生 |
| **集线器（Hub）** | 物理层 | 多端口中继器，广播方式工作 |
| **网桥（Bridge）** | 数据链路层 | 连接两个网段，基于 MAC 地址转发 |
| **交换机（Switch）** | 数据链路层 | 多端口网桥，基于 MAC 地址转发 |
| **路由器（Router）** | 网络层 | 连接不同网络，基于 IP 地址路由 |
| **网关（Gateway）** | 应用层 | 协议转换 |

> 🎯 **广播域与冲突域**（★ 常考）：
> - **集线器**：一个冲突域、一个广播域
> - **交换机**：每个端口一个冲突域、一个广播域（默认）
> - **路由器**：每个端口一个冲突域、一个广播域（隔离广播）

## 第三章 物理层（★ 了解为主）

### 3.1 数据通信基础

**信道容量计算（★★★ 计算题）**：

**（1）奈奎斯特定理（无噪声信道）**：
```
C = 2 × B × log₂V
```
- C = 最大数据传输速率（bps）
- B = 信道带宽（Hz）
- V = 信号电平级数（离散电平数）

**示例**：
> 某信道带宽 4kHz，使用 8 个电平级，则最大数据传输速率：
> C = 2 × 4000 × log₂8 = 2 × 4000 × 3 = 24000 bps = 24 kbps

**（2）香农定理（有噪声信道）**：
```
C = B × log₂(1 + S/N)
```
- C = 最大数据传输速率（bps）
- B = 信道带宽（Hz）
- S/N = 信噪比

> ⚠️ 如果信噪比以 dB 给出：S/N(dB) = 10 × log₁₀(S/N)，需先还原成比值。

**示例**：
> 信道带宽 3kHz，信噪比 30dB，求最大数据传输速率：
> 30 = 10 × log₁₀(S/N) → S/N = 10³ = 1000
> C = 3000 × log₂(1 + 1000) ≈ 3000 × log₂1001 ≈ 3000 × 10 = 30000 bps

### 3.2 传输介质

| 介质 | 特点 | 适用场景 |
|------|------|----------|
| **双绞线** | 便宜、易安装，距离短（100m） | 局域网 |
| **同轴电缆** | 抗干扰较好 | 有线电视 |
| **光纤** | 高速率、长距离、抗干扰 | 干线传输 |
| **无线** | 灵活、无布线 | 移动通信 |

---

### 📝 本章典型真题

**【2021年真题改编】**
> 某信道带宽为 3kHz，信噪比为 30dB，则该信道的最大数据传输速率约为（）kbps。
> A. 10　　B. 20　　C. 30　　D. 60

**答案：C**
**解析**：S/N(dB) = 30 → S/N = 1000。C = 3000 × log₂(1001) ≈ 3000 × 10 = 30kbps。

## 第四章 数据链路层（★★ 重要）

### 4.1 数据链路层的功能

**教材定义**：
> 数据链路层在物理层提供的服务基础上，将物理层提供的可能出错的物理连接改造为逻辑上无差错的数据链路。

**主要功能**：
- **帧同步**：将比特流划分成帧
- **差错控制**：检测和纠正传输错误
- **流量控制**：控制发送速率

### 4.2 差错控制

#### 4.2.1 CRC 循环冗余校验（★★★ 高频计算题）

**原理**：
> 发送方用生成多项式 G(x) 去除数据，将余数作为校验码附加在数据后面。接收方用同样的 G(x) 去除，余数为 0 则无差错。

**计算步骤**：
1. 设数据为 D，生成多项式为 G（r+1 位）
2. 在 D 后面补 r 个 0（r = G 的最高次幂）
3. 用模 2 除法（异或）计算 D 后补零 / G 的余数
4. 余数即为 CRC 校验码，附加在 D 后面

**示例**：
> 数据 1101011011，生成多项式 G(x) = x⁴ + x + 1（即 10011）
> - G 为 5 位，r = 4
> - 数据后补 4 个 0：1101011011**0000**
> - 模 2 除法求余数
> - 余数即为 CRC 校验码

> ⚠️ **模 2 除法**：不进位、不借位，按位异或。这与普通除法不同。

#### 4.2.2 海明码（★★★ 高频计算题）

**原理**：
> 海明码是一种可以纠错的编码方式。通过在数据位中插入若干校验位，实现单比特错误的检测和纠正。

**校验位数量公式**：
```
2^r ≥ m + r + 1
```
- r = 校验位数
- m = 数据位数

**示例**：
> 数据 7 位，求需要几个校验位：
> 2^r ≥ 7 + r + 1
> r=3: 8 ≥ 11? 不满足
> r=4: 16 ≥ 12? 满足
> 需要 4 个校验位

**海明码编码规则**：
- 校验位放在 2 的幂次位置（第 1, 2, 4, 8, ... 位）
- 每个校验位负责若干数据位（按位号二进制分解确定）

### 4.3 局域网

**IEEE 802 标准**（★ 常考）：

| 标准 | 说明 |
|------|------|
| **IEEE 802.3** | CSMA/CD 以太网 |
| **IEEE 802.5** | 令牌环网 |
| **IEEE 802.11** | 无线局域网（Wi-Fi） |

**CSMA/CD（载波监听多路访问/冲突检测）**：
- 工作原理：先听后发、边发边听、冲突停止、随机重发
- 用于以太网（有线）
- 最短帧长 = 2 × 传播时延 × 数据传输速率

## 第五章 网络层（★★★ 核心重点）

### 5.1 IP 协议与 IPv4 地址

**IPv4 地址**：
- 32 位二进制，分为 4 个字节，用点分十进制表示
- 如：192.168.1.1

**IP 地址分类**（★★★ 必背表格）：

| 类别 | 首字节范围 | 网络号位数 | 主机号位数 | 默认子网掩码 | 网络数 | 主机数/网络 |
|------|-----------|-----------|-----------|------------|--------|------------|
| **A 类** | 1–126 | 8 位 | 24 位 | 255.0.0.0 (/8) | 126 | 2²⁴-2 |
| **B 类** | 128–191 | 16 位 | 16 位 | 255.255.0.0 (/16) | 2¹⁴ | 2¹⁶-2 |
| **C 类** | 192–223 | 24 位 | 8 位 | 255.255.255.0 (/24) | 2²¹ | 254 |
| **D 类** | 224–239 | — | — | — | 组播地址 | — |
| **E 类** | 240–255 | — | — | — | 保留 | — |

> ⚠️ **注意**：A 类中 127.x.x.x 是回环地址，不算网络号。

**特殊 IP 地址**（★★ 常考）：

| 地址 | 含义 |
|------|------|
| **全 0 主机号** | 表示网络本身 |
| **全 1 主机号** | 广播地址 |
| **127.x.x.x** | 回环地址（本机测试） |
| **10.0.0.0/8** | A 类私有地址 |
| **172.16.0.0/12** | B 类私有地址 |
| **192.168.0.0/16** | C 类私有地址 |
| **0.0.0.0** | 本网络本主机 |
| **255.255.255.255** | 受限广播 |

### 5.2 子网划分与子网掩码（★★★ 超高频计算题）

**子网掩码**：
> 用于区分 IP 地址中的网络号和主机号。网络号部分全为 1，主机号部分全为 0。

**子网划分步骤**（★ 必须掌握）：
1. 确定原网络的网络号和主机号
2. 确定需要划分的子网数 n
3. 从主机号借 k 位作为子网号，满足 2^k ≥ n
4. 新子网掩码 = 原子网掩码 + k 位
5. 每个子网的主机数 = 2^(剩余主机位) - 2

**计算示例**：
> B 类网络 172.16.0.0/16，需要划分 10 个子网
> - 需借位 k：2^k ≥ 10，k=4（2⁴=16 ≥ 10）
> - 新子网掩码：/20 = 255.255.240.0
> - 每个子网主机数：2¹² - 2 = 4094
> - 子网 1：172.16.0.0/20（地址范围 172.16.0.1–172.16.15.254）
> - 子网 2：172.16.16.0/20
> - ...

**给定子网掩码求子网数**：
> C 类网络 192.168.1.0/28
> - 子网位数：28 - 24 = 4 位 → 子网数 = 2⁴ = 16
> - 主机位数：32 - 28 = 4 位 → 主机数 = 2⁴ - 2 = 14
> - 子网1：192.168.1.0，可用 192.168.1.1–192.168.1.14，广播 192.168.1.15
> - 子网2：192.168.1.16，可用 192.168.1.17–192.168.1.30，广播 192.168.1.31

### 5.3 CIDR 无类域间路由（★ 常考）

**教材定义**：
> CIDR（Classless Inter-Domain Routing）消除了传统的 A、B、C 类地址的界限，使用"IP地址/前缀长度"的表示方法。

**示例**：
> 210.115.192.0/20 表示前 20 位为网络号
> 可容纳的主机数 = 2^(32-20) - 2 = 2¹² - 2 = 4094

**路由汇聚（超网）**：
> 将多个连续的子网合并为一个更大的网络。

### 5.4 ARP 与 RARP 协议

| 协议 | 功能 | 工作方向 |
|------|------|----------|
| **ARP** | 将 IP 地址解析为 MAC 地址 | IP → MAC |
| **RARP** | 将 MAC 地址解析为 IP 地址 | MAC → IP |

### 5.5 ICMP 协议

**教材定义**：
> ICMP（Internet Control Message Protocol）是 TCP/IP 协议族中用于发送差错报文和控制信息的协议。

**典型应用**：
- **ping**：使用 ICMP 回送请求和回送应答
- **traceroute**：使用 ICMP 超时报文追踪路由

### 5.6 路由协议（★★ 常考）

| 协议 | 类型 | 算法 | 特点 |
|------|------|------|------|
| **RIP** | 内部网关协议 | 距离向量 | 最大跳数 15，每 30 秒更新，简单但收敛慢 |
| **OSPF** | 内部网关协议 | 链路状态 | 收敛快、支持分层、适合大型网络 |
| **BGP** | 外部网关协议 | 路径向量 | 用于自治系统之间 |

> 🎯 **RIP vs OSPF 对比**（★ 常考）：
> - RIP：距离向量、跳数限制 15、适合小型网络、收敛慢
> - OSPF：链路状态、无跳数限制、适合大型网络、收敛快

### 5.7 NAT 网络地址转换

**教材定义**：
> NAT（Network Address Translation）将内部私有 IP 地址转换为公有 IP 地址，使多个内部主机可以共享一个公网 IP 访问 Internet。

### 5.8 IPv6 基础（★ 了解）

| 特点 | 说明 |
|------|------|
| 地址长度 | 128 位（IPv4 为 32 位） |
| 表示方法 | 冒号十六进制，如 2001:0db8::1 |
| 地址空间 | 2¹²⁸（几乎无限） |
| 取消广播 | 用组播和任播代替 |

## 第六章 传输层（★★★ 高频考点）

### 6.1 TCP 协议

**教材定义**：
> TCP（Transmission Control Protocol，传输控制协议）是一种面向连接的、可靠的、基于字节流的传输层通信协议。

**TCP 的特点**：

| 特点 | 说明 |
|------|------|
| **面向连接** | 通信前需要建立连接（三次握手） |
| **可靠传输** | 确认、重传、排序 |
| **全双工** | 双方可以同时发送和接收 |
| **面向字节流** | 将数据看作无结构的字节流 |
| **流量控制** | 滑动窗口机制 |
| **拥塞控制** | 慢开始、拥塞避免等 |

**TCP 三次握手**（★★★ 必考）：
```
客户端 → SYN=1, seq=x           → 服务器   （第一次握手）
客户端 ← SYN=1, ACK=1, seq=y, ack=x+1  ← 服务器   （第二次握手）
客户端 → ACK=1, seq=x+1, ack=y+1      → 服务器   （第三次握手）
```

**三次握手原因**：防止已失效的连接请求报文突然到达服务器，导致服务器错误建立连接。

**TCP 四次挥手**（★★ 常考）：
```
主动方 → FIN=1, seq=u           → 被动方   （第一次挥手）
主动方 ← ACK=1, ack=u+1         ← 被动方   （第二次挥手）
主动方 ← FIN=1, seq=w           ← 被动方   （第三次挥手）
主动方 → ACK=1, ack=w+1         → 被动方   （第四次挥手）
```

**四次挥手原因**：TCP 是全双工的，关闭连接需要双方分别关闭。

**TCP 拥塞控制**（★ 常考）：

| 机制 | 说明 |
|------|------|
| **慢开始** | 拥塞窗口从 1 开始，指数增长 |
| **拥塞避免** | 达到阈值后线性增长 |
| **快重传** | 收到 3 个重复 ACK 立即重传 |
| **快恢复** | 拥塞窗口减半后进入拥塞避免 |

### 6.2 UDP 协议

**教材定义**：
> UDP（User Datagram Protocol，用户数据报协议）是一种无连接的、不可靠的传输层协议。

**UDP 的特点**：

| 特点 | 说明 |
|------|------|
| **无连接** | 不需要建立连接 |
| **不可靠** | 不保证数据到达、不保证顺序 |
| **高效** | 头部开销小（8 字节 vs TCP 20 字节） |
| **支持广播/组播** | TCP 不支持 |
| **面向报文** | 保留报文边界 |

**UDP 应用场景**：DNS、DHCP、TFTP、SNMP、视频/语音通话

### 6.3 TCP 与 UDP 对比（★★★ 必背）

| 对比项 | TCP | UDP |
|--------|-----|-----|
| 连接性 | 面向连接 | 无连接 |
| 可靠性 | 可靠（确认重传） | 不可靠 |
| 有序性 | 保证顺序 | 不保证 |
| 速度 | 较慢 | 较快 |
| 头部开销 | 20 字节 | 8 字节 |
| 广播/组播 | 不支持 | 支持 |
| 应用场景 | HTTP、FTP、SMTP、Telnet | DNS、DHCP、SNMP、视频 |
| 数据单位 | 字节流 | 报文 |

### 6.4 端口号（★ 常考）

| 范围 | 类型 | 说明 |
|------|------|------|
| **0–1023** | 知名端口 | 系统服务使用的标准端口 |
| **1024–49151** | 注册端口 | 应用程序注册使用 |
| **49152–65535** | 动态端口 | 临时分配 |

**常见端口号**（★★★ 必背）：

| 协议 | 端口号 |
|------|--------|
| HTTP | **80** |
| HTTPS | **443** |
| FTP（控制） | **21** |
| FTP（数据） | **20** |
| SMTP | **25** |
| POP3 | **110** |
| IMAP | **143** |
| DNS | **53** |
| Telnet | **23** |
| SSH | **22** |
| DHCP | 服务器 67 / 客户端 68 |
| SNMP | **161** |

## 第七章 应用层（★★★ 高频考点）

### 7.1 DNS 域名系统

**教材定义**：
> DNS（Domain Name System）是一个分布式数据库系统，用于将域名解析为 IP 地址。

**域名结构**（从右到左）：
```
www.example.com.
   ↑      ↑     ↑    ↑
  主机名  三级  二级  顶级（根）
```

**顶级域名分类**：

| 类型 | 示例 |
|------|------|
| **通用顶级域名** | .com、.org、.net、.edu、.gov |
| **国家/地区顶级域名** | .cn、.us、.jp、.uk |
| **新通用顶级域名** | .top、.xyz、.club |

**DNS 查询方式**（★★★ 高频考点）：

| 方式 | 说明 |
|------|------|
| **递归查询** | 被查询服务器必须给出最终答案（本地→根→中介→授权→回传） |
| **迭代查询** | 被查询服务器可以返回下一级服务器的地址（本地→根→返回中介地址→本地→中介→...） |

> 🎯 **考试要点**：
> - 本地域名服务器向根域名服务器：通常为**迭代查询**
> - 主机向本地域名服务器：通常为**递归查询**

### 7.2 HTTP 协议

**教材定义**：
> HTTP（HyperText Transfer Protocol）是 Web 的基础协议，用于在客户端和服务器之间传输超文本文档。

**HTTP 请求方法**：

| 方法 | 功能 |
|------|------|
| **GET** | 获取资源 |
| **POST** | 提交数据 |
| **PUT** | 更新资源 |
| **DELETE** | 删除资源 |
| **HEAD** | 获取报文头部 |

**HTTP 状态码**（★★ 常考）：

| 范围 | 类别 | 说明 | 典型状态码 |
|------|------|------|-----------|
| **1xx** | 信息性 | 收到请求，继续处理 | 100 Continue |
| **2xx** | 成功 | 请求成功 | 200 OK |
| **3xx** | 重定向 | 需要进一步操作 | 301 永久重定向, 302 临时重定向 |
| **4xx** | 客户端错误 | 请求有误 | 400 Bad Request, 403 Forbidden, 404 Not Found |
| **5xx** | 服务器错误 | 服务器处理失败 | 500 Internal Server Error, 503 Service Unavailable |

**HTTPS**：
> HTTPS = HTTP + SSL/TLS，在 HTTP 基础上增加了加密和身份验证。

### 7.3 FTP 协议

**教材定义**：
> FTP（File Transfer Protocol）是用于在网络上进行文件传输的协议。

**FTP 的双连接**（★ 常考）：

| 连接 | 端口 | 功能 |
|------|------|------|
| **控制连接** | 21 | 传输命令和响应（持久连接） |
| **数据连接** | 20（主动模式） | 传输文件数据（临时连接） |

**主动模式 vs 被动模式**：
- **主动模式**：服务器主动连接客户端的数据端口
- **被动模式**：服务器被动等待客户端连接

### 7.4 电子邮件协议

|------|------|------|------|
| **SMTP** | 发送邮件 | 25 | 客户端→服务器 |

| **IMAP** | 接收邮件（服务器上管理） | 143 | 服务器→客户端 |

> 🎯 **POP3 vs IMAP**：
> - POP3：邮件下载到客户端后删除服务器上的
> - IMAP：邮件保留在服务器上，支持多设备同步

### 7.5 DHCP 动态主机配置

**教材定义**：
> DHCP（Dynamic Host Configuration Protocol）自动为网络中的主机分配 IP 地址、子网掩码、默认网关等配置信息。

**DHCP 工作流程**（DORA 过程）：

客户端 ← DHCP Offer ← 服务器
客户端 → DHCP Request（广播） → 服务器
客户端 ← DHCP ACK ← 服务器

**教材定义**：
> SNMP（Simple Network Management Protocol）用于管理 IP 网络上的设备。

- 管理端口：161（代理监听）、162（管理站监听 Trap）

## 第八章 网络安全（★★★ 重点，近年分值增加）

### 8.1 信息安全属性

**教材定义的五个安全属性**（★★★ 必背）：

| 属性 | 定义 |
|------|------|
| **机密性** | 信息不被未授权者获取（加密） |
| **完整性** | 信息不被篡改（数字签名、信息摘要） |
| **可用性** | 合法用户可以正常访问（抗 DoS） |
| **可控性** | 对信息的传播和内容具有控制能力 |
| **不可抵赖性（不可否认性）** | 发送方不能否认其发送行为（数字签名） |

### 8.2 加密技术（★★★ 必考）

#### 8.2.1 对称加密

**原理**：加密和解密使用相同的密钥。

| 算法 | 特点 |
|------|------|
| **DES** | 56 位密钥，已被淘汰 |
| **3DES** | 三次 DES，安全性提高 |
| **AES** | 128/192/256 位密钥，目前主流 |
| **IDEA** | 128 位密钥 |

| 特点 | 说明 |
|------|------|
| 优点 | 速度快、适合大量数据加密 |
| 缺点 | 密钥分发困难 |

#### 8.2.2 非对称加密

**原理**：使用一对密钥——公钥（公开）和私钥（保密）。公钥加密的数据只有私钥能解密。

| 算法 | 特点 |
|------|------|
| **RSA** | 最常用，基于大数分解 |
| **ECC** | 椭圆曲线，同等安全下密钥更短 |

| 特点 | 说明 |
|------|------|
| 优点 | 解决了密钥分发问题 |
| 缺点 | 速度慢，适合小量数据 |

**实际应用**：混合加密——用非对称加密传递对称密钥，用对称密钥加密数据。

#### 8.2.3 信息摘要（★★ 常考）

**原理**：
> 对原文进行 Hash 运算，生成固定长度的摘要值。任何原文的修改都会导致摘要值变化。

| 算法 | 输出长度 | 安全性 |
|------|----------|--------|
| **MD5** | 128 位 | 已不安全 |
| **SHA-1** | 160 位 | 已不安全 |
| **SHA-256** | 256 位 | 目前安全 |

> ⚠️ 摘要不等于加密：摘要是单向的，不能还原原文。

#### 8.2.4 数字签名（★★★ 必考）

**原理**：
> 发送方用私钥对信息摘要进行加密（签名），接收方用发送方的公钥解密验证。

**过程**：
```
发送方：原文 → Hash → 摘要 → 私钥加密 → 数字签名
接收方：数字签名 → 公钥解密 → 摘要1
        原文 → Hash → 摘要2
        比较 摘要1 == 摘要2 → 验证成功
```

**数字签名的作用**：
- ✅ 保证完整性（摘要验证）
- ✅ 保证不可抵赖性（用私钥签名）
- ❌ 不能保证机密性（原文是明文传输）

#### 8.2.5 数字证书与数字信封

**数字证书**：
> 由 CA（证书认证机构）签发，包含持有者的公钥和身份信息，用 CA 的私钥签名。

**数字信封**（★★ 常考）：
```
发送方：
1. 生成对称密钥 K
2. 用 K 加密原文 → 密文
3. 用接收方的公钥加密 K → 数字信封
4. 发送：密文 + 数字信封

接收方：
1. 用自己的私钥解密数字信封 → 得到 K
2. 用 K 解密密文 → 得到原文
```

> 🎯 **数字签名 vs 数字信封**：
> - 数字签名：保证完整性 + 不可否认（私钥签、公钥验）
> - 数字信封：保证机密性（公钥加密对称密钥、私钥解密）

### 8.3 网络安全协议

| 协议 | 功能 | 层次 |
|------|------|------|
| **SSL/TLS** | 传输层加密 | 传输层与应用层之间 |
| **HTTPS** | HTTP + SSL/TLS | 应用层 |
| **IPSec** | IP 层加密和认证 | 网络层 |
| **PGP** | 电子邮件加密和签名 | 应用层 |

### 8.4 防火墙技术

| 类型 | 原理 | 特点 |
|------|------|------|
| **包过滤防火墙** | 根据 IP/端口/协议过滤 | 速度快、粒度粗 |
| **状态检测防火墙** | 跟踪连接状态 | 更安全 |
| **应用代理防火墙** | 在应用层代理转发 | 最安全、速度最慢 |

### 8.5 网络攻击类型

| 攻击类型 | 说明 |
|----------|------|
| **DoS/DDoS** | 拒绝服务/分布式拒绝服务，耗尽目标资源 |
| **SQL 注入** | 在输入中插入恶意 SQL 语句 |
| **XSS** | 跨站脚本攻击 |
| **钓鱼攻击** | 伪造网站或邮件骗取信息 |
| **中间人攻击** | 截获和篡改通信 |
| **ARP 欺骗** | 伪造 ARP 响应，篡改 MAC 地址映射 |

### 10.1 加密体系综合对比

| 对比项 | 对称加密 | 非对称加密 | 信息摘要 |
|--------|----------|-----------|----------|

### 10.2 安全协议层次分布

```
应用层:    PGP（邮件加密）、S-HTTP
表示层:    SSL/TLS（握手和加密）
传输层:    SSL/TLS（记录协议）
网络层:    IPSec（AH 认证头 + ESP 封装安全载荷）
数据链路层: PPTP、L2TP
```

**各层次安全协议详解**：

| 协议 | 层次 | 功能 | 应用场景 |
|------|------|------|----------|
| **SSL 3.0 / TLS** | 传输层与应用层之间 | 加密 HTTP 通信 | HTTPS、网上银行 |
| **IPSec** | 网络层 | IP 包级加密和认证 | VPN |
| **HTTPS** | 应用层 | HTTP + SSL/TLS | 安全 Web 访问 |
| **PGP** | 应用层 | 邮件加密和签名 | 安全电子邮件 |
| **SET** | 应用层 | 安全电子交易 | 网上支付 |
| **Kerberos** | 应用层 | 网络身份认证 | 企业网络认证 |

### 10.3 认证技术

| 技术 | 说明 |
|------|------|
| **用户名/密码** | 最基本的认证方式 |
| **数字证书** | CA 签发，绑定公钥和身份 |
| **生物特征** | 指纹、虹膜、面部识别 |
| **双因素认证** | 结合两种以上认证因素 |
| **Kerberos** | 基于票据的网络认证协议 |
| **RADIUS** | 远程认证拨号用户服务 |

### 10.4 网络安全威胁分类

| 类别 | 威胁 | 说明 |
|------|------|------|
| **窃听** | 嗅探、流量分析 | 信息被未授权获取 |
| **篡改** | 中间人攻击、数据修改 | 信息被非法修改 |
| **伪装** | IP 欺骗、ARP 欺骗 | 冒充他人身份 |
| **重放** | 捕获数据包重发 | 利用旧数据包攻击 |
| **拒绝服务** | DoS、DDoS | 耗尽资源使服务不可用 |
| **社会工程** | 钓鱼、社工 | 利用人性弱点 |

### 10.5 防火墙部署策略

**防火墙的体系结构**：

| 结构 | 说明 |
|------|------|
| **屏蔽路由器** | 在路由器上配置过滤规则 |
| **双宿主主机** | 双网卡主机，禁止 IP 转发 |
| **屏蔽主机** | 路由器+堡垒主机 |
| **屏蔽子网（DMZ）** | 非军事区，放置对外服务器 |

**DMZ（非军事区）**：
> 在内部网络和外部网络之间建立的一个缓冲区域，放置需要对外提供服务的服务器（如 Web 服务器、邮件服务器），即使被攻击也不影响内部网络。

### 10.6 入侵检测系统 IDS

| 类型 | 检测方法 | 特点 |
|------|----------|------|
| **基于特征的 IDS** | 匹配已知攻击特征 | 准确率高，但无法检测未知攻击 |
| **基于异常的 IDS** | 检测与正常行为的偏差 | 能检测未知攻击，误报率高 |

### 10.7 VPN 虚拟专用网络

**教材定义**：
> VPN（Virtual Private Network）是在公用网络上建立专用网络的技术，通过加密和隧道技术保证数据传输的安全性。

**VPN 的关键技术**：
- **隧道技术**：将数据包封装在另一个数据包内传输
- **加密技术**：保证数据在公网上传输的机密性
- **身份认证**：确保只有授权用户可以接入

**VPN 协议**：

| 协议 | 特点 |
|------|------|
| **IPSec VPN** | 网络层 VPN，安全性高 |
| **SSL VPN** | 应用层 VPN，部署简便 |
| **PPTP** | 点对点隧道协议，简单但安全性差 |
| **L2TP** | 第二层隧道协议，常与 IPSec 配合 |

## 附录 E：子网划分专项训练

### 训练题 1
> C 类地址 192.168.10.0/24，划分 8 个子网，求子网掩码、每个子网的网络地址和可用 IP 范围。

**解答**：
- 8 个子网：2^k ≥ 8，k=3
- 新掩码：/27 = 255.255.255.224
- 每子网主机数：2^5 - 2 = 30

| 子网 | 网络地址 | 可用 IP 范围 | 广播地址 |
|------|----------|-------------|----------|
| 1 | 192.168.10.0/27 | .1 – .30 | .31 |
| 2 | 192.168.10.32/27 | .33 – .62 | .63 |
| 3 | 192.168.10.64/27 | .65 – .94 | .95 |
| 4 | 192.168.10.96/27 | .97 – .126 | .127 |
| 5 | 192.168.10.128/27 | .129 – .158 | .159 |
| 6 | 192.168.10.160/27 | .161 – .190 | .191 |
| 7 | 192.168.10.192/27 | .193 – .222 | .223 |
| 8 | 192.168.10.224/27 | .225 – .254 | .255 |

### 训练题 2
> IP 地址 172.16.68.5，子网掩码 255.255.224.0，求所在子网的网络地址和广播地址。

**解答**：
- 掩码 255.255.224.0 = /19
- 224 = 11100000，借了 3 位子网号
- 68 的二进制：01000100
- 子网号位：010（取高3位）= 2
- 网络地址：172.16.64.0（010**00000** = 64）
- 广播地址：172.16.95.255（010**11111** = 95）
- 可用范围：172.16.64.1 – 172.16.95.254

### 训练题 3
> 某公司需要 500 个 IP 地址，申请了一个 B 类地址 172.20.0.0，如何划分？

**解答**：
- 2^k - 2 ≥ 500，k=9（512-2=510）
- 主机位 9 位，网络位 32-9 = 23 位
- 掩码：/23 = 255.255.254.0
- 网络地址：172.20.0.0/23
- 可用 IP：172.20.0.1 – 172.20.1.254（510 个）
- 广播地址：172.20.1.255

|------|-------------|
| 应用层 | 数据/报文（Message） |
| 传输层 | 报文段（Segment，TCP）/ 数据报（Datagram，UDP） |
| 网络层 | 分组/包（Packet） |
| 数据链路层 | 帧（Frame） |
| 物理层 | 比特流（Bit） |

> 🎯 **封装过程**：应用层数据 → 加上 TCP 头 → 加上 IP 头 → 加上帧头帧尾 → 比特流
> 
> **解封装过程**反之：比特流 → 去帧头帧尾 → 去 IP 头 → 去 TCP 头 → 得到应用数据

## 第十二章 数据通信编码与调制（★ 偶考）

### 12.1 数字数据的数字编码

| 编码方式 | 特点 | 波形特征 |
|----------|------|----------|
| **不归零编码（NRZ）** | 用高低电平表示0和1 | 简单，但无同步信号 |
| **曼彻斯特编码** | 每比特中间有一次跳变 | 跳变方向表示0或1（以太网采用） |
| **差分曼彻斯特编码** | 每比特开始有跳变为0，无跳变为1 | 令牌环网采用 |

> 🎯 **曼彻斯特编码规则**（★★ 常考）：
> - 比特中间从高到低跳变 = 0
> - 比特中间从低到高跳变 = 1
> - （也有教材反过来定义，以教材为准）

**曼彻斯特编码计算**：
> 原始数据速率 10Mbps，使用曼彻斯特编码后，信道上的信号速率是多少？
> - 曼彻斯特编码每比特需要两次电平变化 → 信号速率为 20Mbps（波特率 = 2 × 比特率）

### 12.2 数字数据的模拟调制

| 调制方式 | 调制对象 | 说明 |
|----------|----------|------|
| **ASK（幅移键控）** | 振幅 | 用不同幅度表示0和1 |
| **FSK（频移键控）** | 频率 | 用不同频率表示0和1 |
| **PSK（相移键控）** | 相位 | 用不同相位表示0和1 |
| **QAM（正交幅度调制）** | 振幅+相位 | 组合调制，速率更高 |

### 12.3 多路复用技术（★ 常考）

| 方式 | 原理 | 应用 |
|------|------|------|
| **FDM（频分复用）** | 不同用户使用不同频率 | 有线电视、广播 |
| **TDM（时分复用）** | 不同用户使用不同时隙 | 电话网络 |
| **WDM（波分复用）** | 不同用户使用不同波长 | 光纤通信 |
| **CDM/CDMA（码分复用）** | 不同用户使用不同编码 | 3G 移动通信 |
| **OFDM（正交频分复用）** | 将信道分成多个正交子载波 | 4G/5G、Wi-Fi |

## 附录 G：网络综合计算题汇总

### 计算题型 1：奈奎斯特与香农

**例题**：
> 某信道带宽为 4kHz，使用 4 个电平级，信噪比为 20dB。求该信道的最大数据传输速率。

**解答**：
- 奈奎斯特：C₁ = 2 × 4000 × log₂4 = 2 × 4000 × 2 = 16000 bps
- 香农：S/N = 10^(20/10) = 100，C₂ = 4000 × log₂(1+100) ≈ 4000 × 6.66 = 26640 bps
- 取较小值：**16000 bps = 16 kbps**

> 🎯 **当两个定理都给出时，取较小值**。

### 计算题型 2：子网划分与地址计算

**例题**：
> 网络地址 202.117.192.0/20，求可划分的 C 类子网数。

**解答**：
- C 类网络号占 24 位，当前有 20 位
- 可用子网号位数：24 - 20 = 4 位
- 可划分 C 类子网数：2⁴ = **16 个**

### 计算题型 3：CRC 校验计算

**例题**：
> 数据 M = 101001，生成多项式 G = 1101（对应 x³+x²+1），求 CRC 校验码。

**解答**：
1. G 为 4 位，r = 3
2. 在 M 后补 3 个 0：101001**000**
3. 模 2 除法（异或）：
   ```
   101001000 ÷ 1101
   = ... （按位异或逐位除）
   ```
4. 余数即 CRC 校验码

### 计算题型 4：海明码校验位计算

**例题**：
> 数据位 16 位，采用海明码纠错，至少需要多少个校验位？

**解答**：
- 2^r ≥ 16 + r + 1
- r=4: 16 ≥ 21? 不满足
- r=5: 32 ≥ 22? 满足
- 至少需要 **5 个校验位**

### 计算题型 5：曼彻斯特编码

**例题**：
> 数据序列 10110，使用曼彻斯特编码画出波形（假设低→高=1，高→低=0）。

**解答**：
```
数据:    1      0      1      1      0
        ___         ___  ___
       |   |___    |   ||   |___
                      |  |
波形：低-高  高-低  低-高  低-高  高-低
```

## 附录 H：考试策略与备考建议

### 上午选择题答题策略

1. **子网划分题**（每年必考 1-2 题）：
   - 熟练掌握二进制与十进制转换
   - 记住 /8、/16、/24、/27、/28、/30 等常用掩码
   - 快速判断：借 k 位 → 2^k 个子网

2. **协议与端口号题**：
   - 端口号直接背熟（HTTP=80, FTP=21, DNS=53, SMTP=25 等）
   - 协议所在层次要分清

3. **网络安全题**：
   - 对称 vs 非对称加密要分清
   - 数字签名 = 私钥签名 + 公钥验证
   - 数字信封 = 公钥加密对称密钥 + 私钥解密

4. **OSI 模型题**：
   - 七层功能和每层的协议/设备对照要背熟
   - 注意数据单位名称（比特→帧→分组→报文段→数据）

### 常见易错点

| 易错点 | 正确理解 |
|--------|----------|
| ARP 工作在哪个层 | 网络层（虽然涉及 MAC 地址） |
| ICMP 的 ping 是什么协议 | ICMP（不是 TCP 也不是 UDP） |
| DNS 用 TCP 还是 UDP | 查询用 UDP，区域传送用 TCP |
| FTP 是一条连接还是两条 | 两条（控制连接+数据连接） |
| TCP 为什么是三次握手不是两次 | 防止已失效的连接请求建立连接 |
| RIP 最大跳数 | 15（16 表示不可达） |
| A 类地址范围 | 1-126（127 是回环） |
| 私有地址范围 | 10.x.x.x, 172.16-31.x.x, 192.168.x.x |
| 数字签名能否保证机密性 | 不能（原文明文传输） |
| SSL 在 OSI 哪层 | 传输层与应用层之间 |
